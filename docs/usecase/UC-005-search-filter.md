| **名稱** | 搜尋與篩選 |
| **Actor** | Developer |

| **元件** | Flutter Frontend (+ Go Backend 支援) |

---

## 目標

Developer 能透過文字搜尋快速找到特定的 session 或對話內容，
在大量 session 和冗長對話中精準定位所需資訊。

---

## 前置條件

1. Session 列表已載入
2. 至少有一個 session 包含對話歷史

---

## 子功能

### 5.1 Session 列表篩選

1. Developer 在 sidebar 頂部的搜尋欄輸入文字
2. Session 列表即時篩選，只顯示符合條件的 session
3. 篩選依據：摘要、專案名稱、Git branch
4. 清除搜尋文字後恢復完整列表

### 5.2 對話內容搜尋

**[Phase 1-2] 搜尋範圍限於已載入內容**

1. Developer 在對話檢視區域按下搜尋快捷鍵（Ctrl/Cmd + F）
2. 出現搜尋列
3. 輸入搜尋文字
4. 所有符合的文字高亮顯示
5. 搜尋列右側顯示匹配計數，格式為 `{目前位置}/{總匹配數}`（例如 `3/15` 表示共 15 筆匹配，目前焦點在第 3 筆）；無匹配時顯示 `0/0`；搜尋欄為空時不顯示計數
6. 上/下箭頭在搜尋結果間導航，計數中的「目前位置」隨導航同步更新
7. 自動捲動到當前匹配位置

**範圍限制說明**：
- 搜尋僅限於當前已載入的對話內容（由 UC-002 A1「載入更多歷史」決定範圍）
- 若需搜尋更早的內容，開發者須先透過「載入更多歷史」擴展對話視窗範圍

**Phase 3+ 後端全文搜尋（未來擴充方向，不在 Phase 1-2 範圍內）**：
- 目的：支援搜尋完整對話歷史，不受前端已載入範圍限制
- 實現方式：透過 WebSocket 協議擴充，新增 `search_session` action（Client -> Server）和 `search_results` 回應（Server -> Client），詳見 UC-009 訊息協議的 Phase 3+ 預留章節
- 觸發時機：開發者在對話搜尋列中勾選「搜尋完整歷史」選項時，將搜尋請求發送至後端
- 後端職責：Go Backend 讀取完整 JSONL 檔案，執行全文比對，回傳匹配的 SessionEvent 片段及其上下文
- UI 整合：搜尋結果以獨立列表呈現（區別於前端即時高亮），點擊結果項後跳轉並載入對應歷史區段

### 5.3 狀態篩選

1. Developer 點擊狀態群組標題（Active / Idle / Completed）
2. 切換該群組的展開/摺疊
3. 或使用篩選按鈕只顯示特定狀態的 session

---

## 驗收條件

- [ ] Session 列表支援即時文字篩選
- [ ] 對話內容支援全文搜尋
- [ ] 搜尋結果高亮顯示
- [ ] 支援上/下導航搜尋結果
- [ ] 搜尋回應時間 < 200ms（已載入資料，純前端計算，不涉及後端請求）
- [ ] 搜尋不影響即時事件串流
