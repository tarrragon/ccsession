# UC-005: 搜尋與篩選

## 基本資訊

| 項目 | 說明 |
|------|------|
| **ID** | UC-005 |
| **名稱** | 搜尋與篩選 |
| **Actor** | Developer |
| **優先級** | P1 |
| **元件** | Flutter Frontend (+ Go Backend 支援) |
| **依賴** | UC-001 (Session List), UC-002 (Conversation View) |

---

## 目標

Developer 能透過文字搜尋快速找到特定的 session 或對話內容，
在大量 session 和冗長對話中精準定位所需資訊。

---

## 前置條件

1. Session 列表已載入
2. 至少有一個 session 包含對話歷史

---

## 子功能

### 5.1 Session 列表篩選

1. Developer 在 sidebar 頂部的搜尋欄輸入文字
2. Session 列表即時篩選，只顯示符合條件的 session
3. 篩選依據：摘要、專案名稱、Git branch
4. 清除搜尋文字後恢復完整列表

### 5.2 對話內容搜尋

**[Phase 1-2] 搜尋範圍限於已載入內容**

1. Developer 在對話檢視區域按下搜尋快捷鍵（Ctrl/Cmd + F）
2. 出現搜尋列
3. 輸入搜尋文字
4. 所有符合的文字高亮顯示
5. 顯示匹配數量（如 "3/15"）
6. 上/下箭頭在搜尋結果間導航
7. 自動捲動到當前匹配位置

**範圍限制說明**：
- 搜尋僅限於當前已載入的對話內容（由 UC-002 A1「載入更多歷史」決定範圍）
- 若需搜尋更早的內容，開發者須先透過「載入更多歷史」擴展對話視窗範圍
- Phase 3+ 可考慮後端全文搜尋支援（需要 WebSocket 協議擴充，暫不在 Phase 1-2 範圍內）

### 5.3 狀態篩選

1. Developer 點擊狀態群組標題（Active / Idle / Completed）
2. 切換該群組的展開/摺疊
3. 或使用篩選按鈕只顯示特定狀態的 session

---

## 驗收條件

- [ ] Session 列表支援即時文字篩選
- [ ] 對話內容支援全文搜尋
- [ ] 搜尋結果高亮顯示
- [ ] 支援上/下導航搜尋結果
- [ ] 搜尋回應時間 < 200ms（已載入資料，純前端計算，不涉及後端請求）
- [ ] 搜尋不影響即時事件串流
