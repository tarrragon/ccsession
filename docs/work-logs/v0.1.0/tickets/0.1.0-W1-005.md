# Execution Log

## Task Summary

修復 handoff CLI 的 `_verify_handoff_status()` 函式，使其支援 in_progress 獨立任務的 context 刷新交接，與設計文件 askuserquestion-rules.md #11a 保持一致。

---

## Problem Analysis

### Bug 描述
`handoff.py:68` 的 `_verify_handoff_status()` 只允許：
- `completed` 狀態的獨立任務
- `in_progress` 或 `completed` 狀態的**有 children 父任務**

不支援 in_progress 獨立任務的 context 刷新場景。

### 影響
- askuserquestion-rules.md 場景 #11a「Context 刷新 Handoff」無法透過 CLI 執行
- 上個 session 的 handoff 因此失敗，交接記錄遺失

### 安全約束（不可協商）
**handoff 檔案必須且只能透過 `ticket handoff` CLI 命令建立。**
禁止以任何方式手動建立 `.claude/handoff/pending/*.json` 檔案。

理由：
- 手動建立繞過了 CLI 的狀態驗證、依賴檢查、chain 分析等防護
- 手動建立的 JSON 可能缺少必要欄位或格式錯誤，導致 resume 行為異常
- 如果允許手動建立，會讓「CLI 狀態驗證」形同虛設，失去防護意義
- 正確做法是修復 CLI 本身，而非繞過它

### 相關程式碼
- `handoff.py:68-100` — `_verify_handoff_status()`
- `handoff.py:216` — 路徑硬編碼
- `resume.py:35-47` — 路徑硬編碼（應與 handoff 使用共用常數）
