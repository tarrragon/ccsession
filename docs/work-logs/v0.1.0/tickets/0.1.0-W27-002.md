修復 Hook tool_input 第二層 None 防護（or {} 模式統一）

---

## Problem Analysis

Python 的 `dict.get('tool_input', {})` 在 key 存在但值為 JSON `null`（Python `None`）時，回傳 `None` 而非 `{}`。後續的 `.get()` 呼叫會拋 `AttributeError`。

全量掃描發現 25 個 Hook 檔案共 40 處使用此舊模式：
- 30 個檔案中的 `.get("tool_input", {})` 或 `.get('tool_input', {})`
- `.get("tool_response", {})` 或 `.get('tool_response', {})` 亦有相同問題

---

## Solution

使用 Python 的 `or` 運算子，將所有 `.get('tool_input', {})` 改為 `.get('tool_input') or {}`。這種寫法在 key 不存在或值為 `None` 時都會回傳 `{}`。
