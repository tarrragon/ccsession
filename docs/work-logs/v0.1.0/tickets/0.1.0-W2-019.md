# Execution Log

## Task Summary

修正 ticket resume 錯誤訊息語義不正確（找不到 handoff 檔案時誤報為找不到 Ticket）

---

## Problem Analysis

**根因**：`resume.py:_execute_resume()` 行 468-474，當 `load_handoff_file(ticket_id)` 回傳 `None` 時，直接使用 `ErrorMessages.TICKET_NOT_FOUND` 訊息。但此函式只查 `pending/` 目錄的 handoff 檔案，並未驗證 Ticket 本身是否存在。

**影響**：用戶對已存在但無 handoff 的 Ticket 執行 resume 時，收到誤導性訊息「找不到 Ticket」，產生困惑。

**修復方向**：在 handoff 檔案不存在時，額外檢查 Ticket 是否存在，分別顯示：
- Ticket 存在、無 handoff → 「此 Ticket 無待恢復的交接檔案」
- Ticket 不存在 → 「找不到 Ticket」（維持原訊息）
