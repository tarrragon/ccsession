## 修改內容

1. **導入變更**：
   - 新增 `find_ticket_files` 導入自 hook_utils 模組

2. **find_ticket_file() 函式重構**：
   - 刪除自實作的搜尋邏輯（搜尋 .claude/tickets/ 和 docs/work-logs/*/tickets/）
   - 改用 hook_utils.find_ticket_files() 進行全面掃描
   - 根據 ticket_id 篩選出符合的檔案

## 優勢

- 消除重複程式碼：原本分散在兩處的相同功能合併為單一實作
- 提升可維護性：所有 Ticket 檔案搜尋邏輯集中在 hook_utils
- 保持相容性：API 簽名不變（仍接受 ticket_id 和 logger，返回 Optional[Path]）
- 支援擴展性：hook_utils.find_ticket_files() 的改進會自動惠及此 hook
