# Execution Log

## Task Summary

改進 prompt-submit-hook _is_keyword_negated() 遠距否定詞支援

---

## Problem Analysis

目前 `_is_keyword_negated()` 函式實作使用直接字串連接檢查：
```python
if negation + keyword in prompt:
    return True
```

此方法只支援**緊鄰模式**（如「不需要查詢」），無法處理**遠距模式**：
- 「我不是說不用查詢進度」（中間有「說」）
- 「完全不需要去查詢」（中間有「去」）
- 「這個 ticket 不用繼續了」（否定詞和關鍵字間隔多詞）

導致這些否定語境無法被正確過濾，使得 SKILL 檢測邏輯誤判為肯定語境，觸發不必要的提示。

---

## Solution

### 方案概述：滑動窗口方法

使用**遠距窗口搜索**替代直接字串連接，在否定詞出現後的 N 個字符內尋找關鍵字。

### 實作步驟

1. **新增常數** `NEGATION_WINDOW_SIZE = 15`
   - 定義遠距否定詞搜索窗口大小
   - 放置在 `NEGATION_WORDS` 定義之後

2. **改進函式邏輯**
   ```python
   def _is_keyword_negated(prompt: str, keyword: str) -> bool:
       for negation in NEGATION_WORDS:
           negation_idx = prompt.find(negation)
           if negation_idx == -1:
               continue
           # 取否定詞後的窗口文本
           window_start = negation_idx + len(negation)
           window_end = window_start + NEGATION_WINDOW_SIZE
           window_text = prompt[window_start:window_end]
           if keyword in window_text:
               return True
       return False
   ```

3. **核心改進**
   - 使用 `find()` 定位否定詞位置
   - 計算否定詞後的窗口範圍（15 字符內）
   - 在窗口文本內檢查關鍵字
   - 保留原有的緊鄰模式能力（15 字符涵蓋大多數情況）

### 邊界案例處理

- **多個否定詞**：只要任一否定詞在關鍵字 15 字符內就判定為否定
- **否定詞在關鍵字之後**：窗口只往前搜索，不影響此類情況
