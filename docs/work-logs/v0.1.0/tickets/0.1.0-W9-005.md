# Execution Log

## Task Summary

改善 handoff gc 子命令路由：以 subparser 取代 ticket_id == 'gc' 魔法值

---

## Problem Analysis

### 現狀分析

目前 `ticket handoff gc` 命令的路由存在以下問題：

1. **魔法值判斷**：在 `execute()` 函式第 952 行使用 `ticket_id == "gc"` 進行路由，這是不符合 argparse 最佳實踐的做法
2. **混亂的參數設計**：`--dry-run` 和 `--execute` flag 定義在 handoff 主命令的頂層 parser（第 1076-1084 行），而不是在 gc 子命令專用 parser
3. **缺乏互斥性控制**：目前沒有使用 `mutually_exclusive_group` 確保 `--dry-run` 和 `--execute` 互斥

### 對比參考

`track.py` 的 subparser 模式（第 538-540 行）展示了正確的做法：
- 使用 `add_subparsers(dest="operation", required=True)` 明確地分離子命令
- 每個子命令有獨立的 parser，參數清晰隔離

### 重構目標

1. 建立 gc subparser，將其作為 handoff 的子命令
2. 將 `--dry-run` 和 `--execute` 移至 gc subparser，並使用 `mutually_exclusive_group`
3. 在 `execute()` 中改用 `args.handoff_command == "gc"` 進行路由，取代魔法值判斷

---

## Solution

### 實作步驟

#### 1. 新增 `_register_handoff_gc()` 函式（第 1097-1113 行）

```python
def _register_handoff_gc(subparsers: argparse._SubParsersAction) -> None:
    """註冊 gc 子命令"""
    gc_parser = subparsers.add_parser("gc", help="清理 stale handoff 檔案")

    # 使用 mutually_exclusive_group 確保只能選一個
    gc_group = gc_parser.add_mutually_exclusive_group()
    gc_group.add_argument(
        "--dry-run",
        action="store_true",
        help="預覽 stale handoff 清單，不實際刪除"
    )
    gc_group.add_argument(
        "--execute",
        action="store_true",
        help="執行清理，將 stale handoff 移至 archive/"
    )
    gc_parser.set_defaults(func=execute)
```

特點：
- 使用 `mutually_exclusive_group()` 明確指定 `--dry-run` 和 `--execute` 互斥
- 將 flag 限制在 gc subparser，避免污染主命令 parser

#### 2. 改造 `register()` 函式（第 1044-1094 行）

```python
def register(subparsers: argparse._SubParsersAction) -> None:
    """註冊 handoff 主命令及子命令"""
    parser = subparsers.add_parser("handoff", help=HandoffMessages.HELP_TEXT)

    # 建立 handoff 子命令解析器
    handoff_subparsers = parser.add_subparsers(
        dest="handoff_command",
        required=False,
        help="handoff 子命令"
    )

    # 註冊 gc 子命令
    _register_handoff_gc(handoff_subparsers)

    # 註冊主 handoff 命令的參數（用於傳統 handoff）
    parser.add_argument("ticket_id", nargs="?", ...)
    # ... 其他傳統 handoff 參數
```

變更點：
- 新增 `add_subparsers(dest="handoff_command")` 分離 gc 子命令
- 移除頂層 parser 中的 `--dry-run` 和 `--execute` 參數
- 保留 `ticket_id` 等傳統 handoff 參數

#### 3. 新增 `_execute_gc()` 函式（第 949-955 行）

```python
def _execute_gc(args: argparse.Namespace) -> int:
    """執行 gc 子命令"""
    from ticket_system.commands.handoff_gc import execute_gc
    dry_run = getattr(args, "dry_run", False)
    execute_mode = getattr(args, "execute", False)
    # 預設為 dry-run（dry_run=True），除非明確指定 --execute
    return execute_gc(dry_run=(dry_run or not execute_mode))
```

邏輯：
- 優先檢查 `--dry-run` flag
- 如果未指定 `--dry-run`，則檢查是否指定 `--execute`
- 預設行為為 dry-run（`dry_run=True`），避免誤操作

#### 4. 改造 `execute()` 函式（第 958-962 行）

**移除魔法值判斷：**
```python
# 舊版本（第 952-955 行）
if getattr(args, "ticket_id", None) == "gc":
    from ticket_system.commands.handoff_gc import execute_gc
    dry_run = not getattr(args, "execute", False)
    return execute_gc(dry_run=dry_run)
```

**新版本（第 961-962 行）：**
```python
if getattr(args, "handoff_command", None) == "gc":
    return _execute_gc(args)
```

優點：
- 使用 argparse 提供的 `handoff_command` 而非魔法值
- 代碼更易讀、易測試
- 符合 argparse 最佳實踐

### 行為驗證

CLI 行為保持不變：
- `ticket handoff gc --dry-run`：預覽 stale handoff（預設行為）
- `ticket handoff gc --execute`：執行清理，移至 archive/
- `ticket handoff <ticket_id>`：傳統 handoff 行為不變
