# 0.1.0-W2-014：sync-claude-pull.py main() 拆分重構

## 重構目標
降低 `main()` 函式的認知負擔，從 20 降低到 8 以下。

## 重構前分析

### 原始 main() 指標
- **行數**：70 行（行 242-311）
- **變數追蹤數**：8 個（project_root, claude_dir, temp_dir, backup_dir, flutter_md, remote_files, file_count, removed）
- **條件分支數**：5 個（try-except, 2x if flutter_md.exists(), if removed, if src_flutter.exists())
- **巢狀深度**：3 層（try → if → for）
- **依賴模組數**：7 個（find_project_root, check_uncommitted_changes, clone_repo, collect_remote_files, sync_directory, cleanup_stale_files, detect_changed_packages）
- **認知負擔指數**：約 20（變數 8 + 分支 5 + 巢狀 3 + 依賴 7 = 23，實際約 20）

### 複雜度來源
1. try-finally 區塊內部多層邏輯
2. 備份、同步、清理、偵測、更新等多個責任混合
3. 中間變數追蹤（backup_dir, remote_files, file_count, removed）
4. 條件邏輯散落各處

## 重構方案

### 提取的子函式

#### 1. `_sync_with_backup(project_root: Path, temp_dir: Path) -> Path`
**責任**：備份當前配置並執行同步

**邏輯**：

- 同步遠端更新到本地

**行數**：43 行
**變數數**：5 個（claude_dir, backup_dir, flutter_md, remote_files, file_count, removed）
**條件分支**：2 個（if flutter_md.exists, if removed）
**巢狀深度**：2 層
**認知負擔指數**：約 9

#### 2. `_update_project_templates(temp_dir: Path, project_root: Path) -> None`
**責任**：更新專案模板檔案

**邏輯**：
- 檢查遠端是否有 project-templates 目錄
- 更新 FLUTTER.md（不覆蓋 CLAUDE.md）
- 輸出提示訊息

**行數**：9 行
**變數數**：2 個（templates_dir, src_flutter）
**條件分支**：2 個（if templates_dir.is_dir, if src_flutter.exists）
**巢狀深度**：2 層
**認知負擔指數**：約 6

#### 3. `_finalize_sync(backup_dir: Path) -> None`
**責任**：完成同步並輸出提示訊息

**邏輯**：
- 輸出成功訊息
- 輸出備份位置
- 輸出初始化提示

**行數**：11 行
**變數數**：1 個（backup_dir，傳入參數）
**條件分支**：0 個
**巢狀深度**：1 層
**認知負擔指數**：約 2
