# Execution Log

## Task Summary

修復 ticket resume --list 任務鏈 handoff 被誤判為 stale 的 bug（IMP-010 再現）

---

## Problem Analysis

`list_pending_handoffs()` 函式（第 111-158 行）的 stale 過濾邏輯在判斷 handoff 是否為 stale 時，只檢查了來源 ticket 的狀態是否為 completed，未考慮 handoff 的 `direction` 欄位和 `from_status` 上下文。

根本原因：
- 任務鏈 handoff（direction: to-sibling/to-parent/to-child）中，ticket completed 是預期狀態（先 complete 再 handoff 到下一任務），不應被過濾為 stale
- 非任務鏈 handoff（如 context-refresh）若從 completed 狀態創建，也不應被過濾
- 只有「ticket completed 且 from_status != completed 的非任務鏈 handoff」才是真正的 stale

這是 IMP-010（GC 狀態語義衝突）錯誤模式的再現。

---

## Solution

### 1. 新增任務鏈 direction 常數定義（constants.py）

在 `constants.py` 中新增兩個常數：
- `TASK_CHAIN_DIRECTION_TYPES = ("to-sibling", "to-parent", "to-child")`
- `NON_CHAIN_DIRECTION_TYPES = ("context-refresh",)`

### 2. 新增 `_is_task_chain_direction()` 輔助函式（resume.py）

判斷 direction 是否為任務鏈類型，支援帶目標 ID 的格式（如 `to-sibling:target_id`）。

### 3. 修改 `list_pending_handoffs()` 的 stale 過濾邏輯

更新 JSON handoff 處理部分（第 144-170 行）：
```python
# 過濾 stale handoff：
# 條件1：來源 Ticket 已 completed（status: completed）
# 條件2：Handoff 是從非 completed 狀態創建的（from_status != "completed"）
# 特例：任務鏈 handoff，即使 completed 也保留

if ticket_id and _is_ticket_completed(ticket_id):
    from_status = data.get("from_status", "")
    direction = data.get("direction", "")

    # 任務鏈 handoff，completed 是預期
    if _is_task_chain_direction(direction):
        handoffs.append(data)
        continue

    # 非任務鏈：只有 from_status != completed 才過濾為 stale
    if from_status != "completed":
        continue

handoffs.append(data)
```
