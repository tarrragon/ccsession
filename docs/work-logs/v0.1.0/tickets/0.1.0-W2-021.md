# Execution Log

## Task Summary

重構 提取 _resolve_ticket_ref_id() 消除 isinstance 型別分派重複

---

## Problem Analysis

三個檔案中（`chain_analyzer.py`、`handoff.py`、`ticket_chain_index.py`）重複出現相同的 isinstance 型別分派邏輯，用於區分子任務/兄弟任務的字串 ID 和字典格式：

1. **chain_analyzer.py**：8 處 isinstance 分派
   - `_has_pending_children()` 中 2 處（lines 172, 180）
   - `_get_sibling_status()` 中 2 處（lines 238, 249）
   - `_get_to_child_recommendation()` 中 2 處（lines 364, 377）
   - `_get_to_sibling_recommendation()` 中 2 處（lines 427, 442）

2. **handoff.py**：6 處 isinstance 分派
   - `_print_children_status()` 中 2 處（lines 550, 557）
   - `_print_to_child_options()` 中 2 處（lines 604, 607）
   - `_print_to_sibling_options()` 中 2 處（lines 631, 637）

3. **ticket_chain_index.py**：2 處 isinstance 分派 + 1 個私有方法
   - `build_from_tickets()` 中 2 處（lines 113, 115，使用已有的 `_extract_id` 方法）
   - 私有方法 `_extract_id()` 可提升為共用函式

邏輯都相同：
```python
if isinstance(item, str):
    # 字串 ID 邏輯
elif isinstance(item, dict):
    # 字典邏輯
```

此模式重複率為 100%（16 處除以單個邏輯單元），符合重構觸發條件。

---

## Solution

提取共用函式 `resolve_id_from_ref()` 到 `lib/ticket_ops.py`，統一處理字串和字典格式的任務 ID 提取。

**實作步驟：**

1. **新增共用函式**（ticket_ops.py）：
   - 函式簽名：`resolve_id_from_ref(ref: Any) -> str`
   - 邏輯：統一處理 str | dict | None 三種情況
   - 返回值：Ticket ID（無法提取返回空字串）

2. **更新 ticket_chain_index.py**：
   - 導入 `resolve_id_from_ref`
   - 替換 2 處 isinstance 分派（lines 77, 99）
   - 移除私有方法 `_extract_id()`

3. **更新 chain_analyzer.py**：
   - 導入 `resolve_id_from_ref`
   - 替換 8 處 isinstance 分派
   - 優化迴圈變數命名（child_item 而非 child_id）便於區分

4. **更新 handoff.py**：
   - 導入 `resolve_id_from_ref`
   - 替換 6 處 isinstance 分派
   - 簡化邏輯分支（從 if/elif/if 改為 if/elif）

**優化成果：**
- 消除 16 處重複的 isinstance 型別分派
- 提升程式碼可讀性（邏輯集中在單一函式）
- 降低維護成本（型別判斷邏輯變更只需修改一處）
- 無功能變更，完全向後相容
