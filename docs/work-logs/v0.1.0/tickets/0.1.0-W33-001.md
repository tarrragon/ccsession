## Solution

### 修復 1：UNICODE_BOARD_TITLE 版本號動態化
- **位置**: `command_tracking_messages.py:77`
- **變更**: 將硬編碼版本號 `"TICKET BOARD - v0.31.0 (W7)"` 改為模板 `"TICKET BOARD - v{version}"`
- **實作**:
  - 在 messages.py 中改為 `UNICODE_BOARD_TITLE = "TICKET BOARD - v{version}"`
  - render_board_unicode() 函式新增 `version` 參數（預設空字串以保持向後相容）
  - 使用 `format_msg(TrackBoardMessages.UNICODE_BOARD_TITLE, version=version)` 進行動態格式化

### 修復 2：row_idx>0 卡片內容永遠空白
- **位置**: `track_board.py:526-533`
- **原問題**: 條件式 `if row_idx == 0:` 導致僅第一行渲染卡片內容（ID、標題、優先級），其餘行全部為空字串
- **修復**: 移除 if 條件，所有 row_idx 都渲染卡片內容
- **驗證**: 現在所有卡片行都正確顯示內容，第二卡及以後的行不再空白

### 修復 3：中文寬度感知的 ljust 實現
- **問題**: 原代碼使用 Python 內建 `str.ljust(width)`，但中文字符視覺寬度為 2，英文為 1，導致中文內容時欄位對齊錯亂
- **解決方案**: 提取兩個工具函式
  - `calculate_visual_width(text: str) -> int`: 計算文本視覺寬度，中文字符（U+4E00~U+9FFF）計為 2，其他計為 1
  - `ljust_with_chinese_width(text: str, width: int) -> str`: 中文感知的字串填充，正確計算補充空格數
- **應用位置**:
  - 欄標題填充 (line 560): `header.ljust()` → `ljust_with_chinese_width(header, card_width)`
  - 卡片 ID/標題/優先級填充 (lines 580-582): `.ljust()` → `ljust_with_chinese_width()`
  - 卡片行內容填充 (line 593): `.ljust(card_width - 2)` → `ljust_with_chinese_width(col[line_in_card], card_width - 2)`
