### 測試預期分析

測試失敗原因：

1. **test_scenario_1_normal_setup**
   - 原預期：setup_hook_logging() 呼叫後日誌檔案立即建立
   - 實際行為：FileHandler 使用 delay=True 參數，檔案只在首次寫入時才建立（W3-004 設計）
   - 修正方式：調整測試順序，先驗證 handlers 已設定，再寫入日誌後驗證檔案建立

2. **test_log_exception_stderr_format**
   - 原預期：stderr 只包含 [Hook Error] 訊息
   - 實際行為：StreamHandler 輸出 CRITICAL 級別訊息（因為 CRITICAL >= WARNING），所以 stderr 包含：
     - [CRITICAL] Unhandled exception in hook_name
     - [CRITICAL] traceback
     - [Hook Error] hook_name failed unexpectedly...
   - 修正方式：調整預期，驗證所有三條訊息都在 stderr 中（按照實作邏輯）
