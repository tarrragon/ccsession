## Solution

### 執行步驟

1. **建立 lib/handoff_utils.py**
   - 新增共用模組 `ticket_system/lib/handoff_utils.py`
   - 遷移 3 個函式：
     - `is_ticket_completed(ticket_id: str) -> bool`：檢查 Ticket 是否已完成
     - `is_task_chain_direction(direction: str) -> bool`：判斷 handoff 是否為任務鏈類型
     - `is_ticket_in_progress_or_completed(ticket_id: str) -> bool`：檢查目標 Ticket 是否已啟動
   - 完整保留原始函式的 docstring 和邏輯

2. **更新 resume.py**
   - 移除原始 3 個函式定義（第 87-169 行）
   - 新增 import：`from ticket_system.lib.handoff_utils import is_ticket_completed, is_task_chain_direction, is_ticket_in_progress_or_completed`
   - 更新所有引用位置：將 `_is_ticket_completed` → `is_ticket_completed`（去掉 `_` 前綴）
   - 共 4 個調用位置需更新

3. **更新 handoff_gc.py**
   - 修改 import 從 `ticket_system.commands.resume` 改為 `ticket_system.lib.handoff_utils`
   - 更新函式調用：去掉 `_` 前綴
   - 共 3 個調用位置需更新

4. **更新測試**
   - 修改所有 @patch 裝飾器：從 `handoff_gc._is_*` → `handoff_gc.is_*`（patch 位置應為使用位置）
   - 共 9 個 @patch 裝飾器需更新

### 核心改進點

- **消除跨模組私有函式引用**：handoff_gc.py 不再依賴 resume.py 的私有實作細節
- **改進模組封裝**：handoff_utils.py 作為共用基礎設施層，提供公開函式（去掉 `_` 前綴）
- **降低耦合風險**：future changes to resume.py 不會影響 handoff_gc.py
