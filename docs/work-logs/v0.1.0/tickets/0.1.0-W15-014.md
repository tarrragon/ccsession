## Solution

修改了 `.claude/hooks/lib/hook_messages.py` 中的 `POST_TICKET_COMPLETE_CHECKPOINT_REMINDER` 常數，將原本簡單的線性流程改為清晰的雙路徑結構：

**路徑 A（有未提交變更 → commit 後）**:
```
Checkpoint 1 → 有未提交變更 → 執行 /commit-as-prompt → 進入路徑 A
  ↓
Checkpoint 1.5 → AskUserQuestion #16（錯誤學習確認）
  ↓
Checkpoint 2 → commit-handoff-hook 自動觸發（無需手動執行）
```

**路徑 B（無未提交變更 → 直接執行 Checkpoint 2）**:
```
Checkpoint 1 → 無未提交變更 → 進入路徑 B
  ↓
Checkpoint 1.5 → AskUserQuestion #16（錯誤學習確認）
  ↓
Checkpoint 2 → 直接執行：ticket track list --wave {n} --status pending in_progress
  ↓
根據查詢結果路由到對應流程或 AskUserQuestion #13
```

**關鍵改善**:
- 清晰區分兩個觸發路徑（commit 和 non-commit）
- 提供直觀的流程圖（使用 | 和 v）
- 補充「無 commit 時的 Checkpoint 2 執行方式」
- 強調 commit-handoff-hook 的條件觸發
- 確保無未提交變更的情況下也不會跳過 Checkpoint 2
