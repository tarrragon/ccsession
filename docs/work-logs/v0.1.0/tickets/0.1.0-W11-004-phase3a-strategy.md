### 1. `_normalize_version(version: str) -> str`

**目標**：標準化版本號（去除 v 前綴）

```
虛擬碼：
函式 _normalize_version(version):
    如果 version 為空或 None:
        返回空字串

    version = version.strip()

    如果 version 以 'v' 或 'V' 開頭:
        version = version[1:]

    返回 version
```

**關鍵設計**：
- 支援 `v0.1.0` 和 `0.1.0` 兩種格式
- 大小寫不敏感（v 和 V 都去除）
- 返回小寫版本號（便於後續比較）

### 2. `_validate_versions(from_version: str, to_version: str, project_root: Path) -> (bool, str)`

**目標**：驗證版本號有效性和存在狀態

```
虛擬碼：
函式 _validate_versions(from_version, to_version, project_root):
    # 標準化版本號
    from_version = _normalize_version(from_version)
    to_version = _normalize_version(to_version)

    # 驗證格式（N.N.N）
    若 from_version 不符合正規表達式 '\d+\.\d+\.\d+':
        返回 (False, "版本號格式無效：{from_version}")

    若 to_version 不符合正規表達式 '\d+\.\d+\.\d+':
        返回 (False, "版本號格式無效：{to_version}")

    # 驗證不相同
    若 from_version == to_version:
        返回 (True, "SAME_VERSION")  # 特殊訊息，不是錯誤

    # 驗證 from_version 存在
    from_dir = project_root / "docs" / "work-logs" / f"v{from_version}"
    若 from_dir 不存在:
        返回 (False, "版本目錄不存在：{from_dir}")

    # 驗證 to_version 不存在
    to_dir = project_root / "docs" / "work-logs" / f"v{to_version}"
    若 to_dir 存在:
        返回 (False, "目標版本目錄已存在：{to_dir}")

    返回 (True, "OK")
```

**驗證順序**：format → equality → existence

### 4. `_shift_version_in_references(ticket: Dict, from_version: str, to_version: str) -> Dict`

**目標**：更新 Ticket 中所有 from_version 前綴的引用

```
虛擬碼：
函式 _shift_version_in_references(ticket, from_version, to_version):
    版本_前綴_舊 = f"{from_version}-"
    版本_前綴_新 = f"{to_version}-"

    需要_更新的_欄位 = [
        "blockedBy", "parent_id", "children", "relatedTo",
        "spawned_tickets", "source_ticket", "chain.root", "chain.parent"
    ]

    對 needs_to_update_欄位 中的 欄位名 迭代:
        若 欄位名 不在 ticket 中:
            繼續下一個

        值 = ticket[欄位名]
        若 值 為空:
            繼續下一個

        # 特殊處理 chain.root 和 chain.parent（巢狀 dict）
        若 欄位名 == "chain.root" 或 "chain.parent":
            # chain 結構下有 root 和 parent 子欄位
            若 "chain" 在 ticket 中:
                更新 ticket["chain"]["root"] 和 ticket["chain"]["parent"]

        # 列表型欄位
        若 值 是列表:
            對列表中的每個元素:
                若 元素 是字串 且 以 版本_前綴_舊 開頭:
                    替換為 版本_前綴_新 + 元素[len(版本_前綴_舊):]
                若 元素 是 dict 且 "id" 在其中 且 "id" 以 版本_前綴_舊 開頭:
                    替換 元素["id"]

        # 字串型欄位
        若 值 是字串 且 以 版本_前綴_舊 開頭:
            替換 ticket[欄位名]

    返回 ticket
```

**複雜度**：中高（涉及多種資料結構和欄位類型）

### 5. `_shift_single_ticket(ticket_path: Path, from_version: str, to_version: str) -> (bool, Dict)`

**目標**：更新單一 Ticket 的所有版本欄位和內部引用

```
虛擬碼：
函式 _shift_single_ticket(ticket_path, from_version, to_version):
    試試:
        ticket = _load_ticket_from_path(ticket_path)
        若 ticket 為 None:
            返回 (False, None)

        # 提取舊 ID
        舊_id = ticket.get("id")
        若 舊_id 為空:
            返回 (False, None)

        # 更新 ID 和 version 欄位
        components = _extract_id_components(舊_id)
        新_id = f"{to_version}-W{components['wave']}-{components['sequence']}"
        ticket["id"] = 新_id
        ticket["version"] = to_version

        # 更新內部交叉引用
        ticket = _shift_version_in_references(ticket, from_version, to_version)

        # 更新 chain 資訊（如果存在）
        chain_info = _calculate_chain_info(新_id)
        若 chain_info:
            ticket["chain"] = chain_info
            ticket["parent_id"] = chain_info.get("parent")

        返回 (True, ticket)

    捕獲任何異常 as e:
        記錄警告訊息
        返回 (False, None)
```

### 7. `_find_auxiliary_files(from_version: str, project_root: Path) -> List[Path]`

**目標**：識別附屬檔案（以 Ticket ID 開頭但非主 Ticket 的 .md 檔）

```
虛擬碼：
函式 _find_auxiliary_files(from_version, project_root):
    tickets_dir = project_root / "docs" / "work-logs" / f"v{from_version}" / "tickets"

    若 tickets_dir 不存在:
        返回 []

    # 列出所有主 Ticket 的 ID（無後綴）
    主_票數_ids = 新的集合()
    對 tickets_dir 中的每個 .md 檔案:
        # 檔名格式：{version}-W{n}-{seq}.md（主 Ticket）
        #          {version}-W{n}-{seq}-{描述}.md（附屬）
        match = 正規表達式.match(r"(\d+\.\d+\.\d+)-W(\d+)-([0-9.]+)\.md", 檔名)
        若 match 且 檔名 == f"{match.group(1)}-W{match.group(2)}-{match.group(3)}.md":
            主_票數_ids.add(f"{match.group(1)}-W{match.group(2)}-{match.group(3)}")

    # 找出附屬檔案（ID 前綴在 主_票數_ids 中，但本身不在其中）
    附屬_檔案 = []
    對 tickets_dir 中的每個 .md 檔案:
        match = 正規表達式.match(r"(\d+\.\d+\.\d+)-W(\d+)-([0-9.]+)-(.+)\.md", 檔名)
        若 match:
            前綴_id = f"{match.group(1)}-W{match.group(2)}-{match.group(3)}"
            若 前綴_id 在 主_票數_ids 中:
                附屬_檔案.append(檔案路徑)

    返回 附屬_檔案
```

**附屬檔案識別規則**：
- 格式：`{version}-W{n}-{seq}-{描述}.md`
- 對應的主 Ticket：`{version}-W{n}-{seq}.md`

### 9. `_update_cross_version_refs(from_version: str, to_version: str, project_root: Path) -> int`

**目標**：掃描其他版本的 Ticket，更新對 from_version 的引用

```
虛擬碼：
函式 _update_cross_version_refs(from_version, to_version, project_root):
    work_logs_root = project_root / "docs" / "work-logs"
    已_更新_檔案 = 新的集合()
    舊_前綴 = f"{from_version}-"
    新_前綴 = f"{to_version}-"

    對 work_logs_root 下的每個版本目錄:
        若 版本 == from_version 或 版本 == to_version:
            跳過（from_version 檔案已被移除，to_version 檔案已在 Step 2-3 更新）

        tickets_dir = 版本目錄 / "tickets"
        若 tickets_dir 不存在:
            繼續下一個

        對 tickets_dir 中的每個 .md 檔案:
            ticket = _load_ticket_from_path(檔案路徑)
            若 ticket 為 None:
                繼續下一個

            # 檢查是否包含舊版本的引用
            包含_舊_引用 = False
            對 "blockedBy", "relatedTo", "children", "parent_id", "spawned_tickets", "source_ticket" 中的欄位:
                若 欄位 在 ticket 中 且 ticket[欄位] 包含 舊_前綴:
                    包含_舊_引用 = True
                    break

            若 包含_舊_引用:
                # 進行替換
                ticket = _shift_version_in_references(ticket, from_version, to_version)
                儲存 ticket
                已_更新_檔案.add(檔案路徑)

    返回 len(已_更新_檔案)
```

**關鍵設計**：
- 排除 from_version 和 to_version 目錄
- 檢查欄位前先判斷是否包含舊前綴（效能最佳化）

### 11. `_update_todolist_yaml(from_version: str, to_version: str, project_root: Path) -> int`

**目標**：更新 todolist.yaml 中與版本相關的欄位

```
虛擬碼：
函式 _update_todolist_yaml(from_version, to_version, project_root):
    todolist_path = project_root / "docs" / "todolist.yaml"

    若 todolist_path 不存在:
        返回 0

    試試:
        載入 todolist = yaml.safe_load(todolist_path)
    捕獲異常:
        返回 0

    若 todolist 為 None 或不是 dict:
        返回 0

    更新計數 = 0

    # 更新 current_version
    若 todolist.get("current_version") == from_version:
        todolist["current_version"] = to_version
        更新計數 += 1

    # 更新 previous_version
    若 todolist.get("previous_version") == from_version:
        todolist["previous_version"] = to_version
        更新計數 += 1

    # 更新 tech_debt 陣列
    若 "tech_debt" 在 todolist 中 且 todolist["tech_debt"] 是列表:
        對 tech_debt 中的每個項目:
            # source_version 有 v 前綴
            若 項目.get("source_version") == f"v{from_version}":
                項目["source_version"] = f"v{to_version}"
                更新計數 += 1

            # source_ticket 無 v 前綴
            若 項目.get("source_ticket") 以 f"{from_version}-" 開頭:
                項目["source_ticket"] = f"{to_version}-" + 項目["source_ticket"][len(f"{from_version}-"):]
                更新計數 += 1

    試試:
        使用 yaml.safe_dump(todolist, todolist_path)
        返回 更新計數
    捕獲異常:
        返回 0
```

**設計考量**：
- 注意 source_version 和 source_ticket 的版本號格式差異（v 前綴）
- YAML 讀寫使用 safe_load/safe_dump

### 14. `execute(args) -> int`

**目標**：命令主入口

```
虛擬碼：
函式 execute(args):
    # 取得專案根目錄
    project_root = get_project_root()
    若 project_root 為 None:
        輸出錯誤訊息
        返回 1

    # 提取參數
    from_version = _normalize_version(args.from_version)
    to_version = _normalize_version(args.to_version)
    dry_run = args.dry_run
    no_backup = args.no_backup
    skip_todolist = args.skip_todolist

    # [Step 0] 前置驗證
    輸出(VersionShiftMessages.STEP_VALIDATE)
    valid, msg = _validate_versions(from_version, to_version, project_root)

    若 msg == "SAME_VERSION":
        輸出(InfoMessages.SAME_VERSION.format(version=from_version))
        返回 0

    若 不是 valid:
        輸出錯誤(msg)
        返回 1

    # Dry-run 模式
    若 dry_run:
        preview = _generate_dry_run_preview(from_version, to_version, project_root)
        輸出(preview)
        返回 0

    # [Step 1] 備份
    輸出(VersionShiftMessages.STEP_BACKUP)
    若 不是 no_backup:
        成功, backup_path = _backup_version_dir(from_version, project_root)
        若 不是 成功:
            輸出錯誤訊息
            返回 1
        輸出訊息(backup_path)
    否則:
        輸出訊息(VersionShiftMessages.BACKUP_SKIP)
        backup_path = None

    # [Step 2] 更新 Ticket 版本欄位
    輸出(VersionShiftMessages.STEP_UPDATE_TICKETS)
    ticket_count, skipped = _shift_ticket_files(from_version, to_version, project_root)
    輸出訊息(f"處理 {ticket_count} 個 Ticket")

    # [Step 3] 重新命名 Ticket 檔案
    輸出(VersionShiftMessages.STEP_RENAME_TICKETS)
    renamed_count = _rename_ticket_files_in_dir(from_version, to_version, project_root)
    附屬_檔案_list = _find_auxiliary_files(from_version, project_root)
    附屬_count = len(附屬_檔案_list)
    輸出訊息(f"附屬文件更新: {附屬_count} 個")

    # [Step 4] 更新跨版本交叉引用
    輸出(VersionShiftMessages.STEP_CROSS_REFS)
    cross_ref_count = _update_cross_version_refs(from_version, to_version, project_root)
    輸出訊息(f"跨版本引用更新: {cross_ref_count} 個")

    # [Step 5] 重新命名 worklog 目錄
    輸出(VersionShiftMessages.STEP_RENAME_DIR)
    success = _rename_worklog_dir(from_version, to_version, project_root)
    若 不是 success:
        輸出錯誤訊息
        返回 1
    輸出訊息(VersionShiftMessages.DIRECTORY_RENAMED)

    # [Step 6] 更新 todolist.yaml
    輸出(VersionShiftMessages.STEP_UPDATE_TODOLIST)
    若 不是 skip_todolist:
        todolist_count = _update_todolist_yaml(from_version, to_version, project_root)
        輸出訊息(f"todolist.yaml 欄位更新: {todolist_count} 個")
    否則:
        輸出訊息(VersionShiftMessages.STEP_UPDATE_TODOLIST_SKIP)
        todolist_count = 0

    # [Step 7] 輸出摘要
    輸出(VersionShiftMessages.STEP_SUMMARY)
    counts = {
        'tickets': ticket_count,
        'auxiliary': 附屬_count,
        'cross_refs': cross_ref_count,
        'todolist': todolist_count,
        'skipped': len(skipped)
    }
    summary = _generate_summary(from_version, to_version, backup_path or "未備份", counts)
    輸出(summary)

    返回 0
```

### 最複雜的 3 個實作單元

**1. `_shift_version_in_references()` — 複雜度：高**
- 涉及 8 種欄位型別（字串、列表、dict、巢狀 dict）
- chain.root 和 chain.parent 的巢狀處理
- 列表中混合字串和 dict 的元素替換
- **建議**：先實作基礎版本（字串替換），再逐步支援複雜型別

**2. `_update_cross_version_refs()` — 複雜度：中高**
- 目錄掃描時需要排除已更新的版本（避免雙重替換）
- 檢查欄位前先判斷是否包含舊前綴（效能最佳化）
- 多重欄位可能包含同一引用（計數不重複）
- **建議**：測試中使用多版本場景驗證去重邏輯

**3. `_update_todolist_yaml()` — 複雜度：中**
- 版本號格式差異（current_version 無 v，source_version 有 v）
- tech_debt 陣列的條件性更新
- YAML 格式保持完整性
- **建議**：YAML 讀寫使用 safe 版本，讀寫後資料結構而非文本比對
