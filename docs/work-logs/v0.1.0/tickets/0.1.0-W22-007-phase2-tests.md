# W22-007 Phase 2 測試案例設計：acceptance-gate-hook.py 場景 #17

## 測試策略總覽

### 分層測試決策

根據 hybrid-testing-strategy-methodology.md，acceptance-gate-hook.py 屬於 Hook 系統（Infrastructure Layer），測試策略如下：

| 層級 | 測試類型 | 對應函式/邏輯 |
|------|---------|------------|
| **單元測試** | 函式層級 | `get_ticket_created_time()`、`check_error_patterns_changed()` 的單獨邏輯驗證 |
| **整合測試** | hook 層級 | `check_acceptance_status()` 與兩個新函式的協作、訊息生成 |
| **系統測試** | 端對端 | `acceptance-gate-hook.py` 主程式接收輸入→輸出場景 #17 提醒 |

### Sociable Unit Tests 原則應用

- **Mock 策略**：只 Mock 外部依賴（filesystem 的 mtime 讀取、frontmatter 解析），使用真實的 Hook 邏輯
- **測試目標**：Hook API（接收 JSON 輸入、輸出 additionalContext 和 exit code）
- **邊界測試**：重點覆蓋時間比較、目錄遍歷的邊界情況
