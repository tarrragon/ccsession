# Execution Log

## Task Summary

設計並實作 版本計畫整體後挪功能：將指定版本的所有 Ticket（含已完成）、worklog 目錄、todolist.yaml 版本記錄整體遷移至新版本號

---

## Problem Analysis

現有 `ticket migrate` 只能做單一 Ticket 的 ID 遷移，無法處理版本整體後挪所需的三種操作：
1. 批量更新所有 Ticket 的 id/version 欄位及內部交叉引用
2. 重新命名 worklog 目錄本身（`v{from}` → `v{to}`）
3. 更新 `docs/todolist.yaml` 的版本記錄欄位

已閱讀並分析：
- `migrate.py`：工具函式（`_extract_id_components`, `_update_cross_references` 等）可複用
- `ticket.py`：命令註冊方式（`register()` + `execute()`）可遵循
- `docs/todolist.yaml`：確認 `source_version`（有 v 前綴）和 `source_ticket` 欄位格式
- `docs/work-logs/v0.1.0/`：確認目錄結構（tickets/ 子目錄 + 主文件）

---

## Solution

Phase 1 功能設計完成，規格文件：
`docs/work-logs/v0.1.0/tickets/0.1.0-W11-004-phase1-design.md`

主要設計決策：
1. 命令名稱：`ticket version-shift`（獨立子命令，非擴充 migrate）
2. 操作順序：備份 → 更新 Ticket 欄位 → 重新命名 Ticket 檔案 → 更新跨版本引用 → 重新命名目錄 → 更新 todolist.yaml → 輸出摘要
3. todolist.yaml 的 `source_version`、`source_ticket` 一律更新（版本遷移後原 Ticket 已不存在）
4. to_version 衝突時拒絕執行（不提供 --force），保護資料安全
5. worklog 主文件內容不修改（歷史記錄真實性優先）
6. 備份策略：整目錄備份（shutil.copytree），備份失敗則停止
