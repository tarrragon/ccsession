現狀確認（2026-03-12）：

1. TICKET_ID_PATTERN 現為嚴格正則，結尾 $ 不允許任何後綴。
2. ticket_loader.py:list_tickets() 第 97 行用此正則過濾，22 個帶後綴的非標準命名 Ticket 檔案被跳過不載入。
3. ticket-id-validator-hook.py 維護獨立正則副本（TICKET_ID_REGEX），與 constants.py 脫鉤，修改後需手動同步。
4. 22 個非標準命名模式主要為 TDD Phase 標記（phase1-design、phase2-test-design 等），是系統正常工作流產生的合法文件。
5. id_parser.py 的 extract_id_components() 傳入帶後綴 ID 時返回 None，影響鏈式計算和 handoff 追蹤。
## Solution

<!-- To be filled by executing agent -->

---

Phase 1 設計完成 — ID 解析包容性擴充功能規格已定義。

核心設計決策：

1. TICKET_ID_PATTERN 擴充：新增可選後綴群組 (-[a-z0-9][a-z0-9-]{0,59})?，向後相容，現有 22 個非標準檔案全部可匹配。

2. extract_core_ticket_id() 新增函式：接受帶後綴的原始 ID，返回去掉後綴的核心 ID。邊界行為：None/空字串 → 返回 None，標準格式 → 原樣返回，帶後綴 → 去掉後綴。

3. ticket_loader.py 更新：改用 extract_core_ticket_id() 過濾，並新增 loaded_core_ids 集合防止同一 core_id 被重複載入（當主檔和後綴檔同時存在時）。

4. ticket-id-conventions.md 命名字典：建立 .claude/rules/core/ 下的官方命名規範，定義 TDD Phase、分析、3b 拆分等允許的後綴模式。

5. ticket-id-validator-hook.py 寬鬆驗證：Hook 動態導入 constants.py 的 TICKET_ID_PATTERN（解決副本不同步問題），對帶後綴 ID 只提示不阻止，區分已知後綴（提示識別）和未知後綴（建議使用標準後綴）。

完整規格見工作日誌：docs/work-logs/v0.1.0-W44-003-feature-design.md
