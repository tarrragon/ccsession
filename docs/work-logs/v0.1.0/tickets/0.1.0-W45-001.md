## Solution

### 修改方案

#### 1. constants.py
- 在 TICKET_ID_PATTERN 定義後新增預編譯正則：`TICKET_ID_RE = re.compile(TICKET_ID_PATTERN)`
- 保留原字串常數 TICKET_ID_PATTERN 不刪除（向後相容，Hook 和外部可能引用）

#### 2. id_parser.py
- 更新 import：改為匯入 TICKET_ID_RE（移除 TICKET_ID_PATTERN）
- 第 56 行（extract_core_ticket_id）：`re.match(TICKET_ID_PATTERN, raw_id)` → `TICKET_ID_RE.match(raw_id)`
- 第 95 行（has_description_suffix）：`re.match(TICKET_ID_PATTERN, raw_id)` → `TICKET_ID_RE.match(raw_id)`
- 第 129 行（extract_id_components）：`re.match(TICKET_ID_PATTERN, ticket_id)` → `TICKET_ID_RE.match(ticket_id)`
- extract_id_components 簽名：`ticket_id: str` → `ticket_id: Optional[str]`
- 新增 None 防守（與同模組 extract_core_ticket_id 和 has_description_suffix 一致）

#### 3. ticket_validator.py
- 更新 import：改為匯入 TICKET_ID_RE（移除 TICKET_ID_PATTERN）
- 第 101 行：移除 `pattern = re.compile(TICKET_ID_PATTERN)`，直接使用 `TICKET_ID_RE.match()`
- 簡化程式碼，避免每次呼叫時重新編譯正則

#### 4. migrate.py
- 清理第 23 行未使用的 TICKET_ID_PATTERN 匯入

#### 5. ticket_loader.py
- 確認 list_tickets 函式的 docstring 已完整，包含：
  - 演算法詳細步驟（1-8）
  - 去重設計說明（優先級順序、快取機制）
  - 完整 Args/Returns/Examples
- 無需修改
