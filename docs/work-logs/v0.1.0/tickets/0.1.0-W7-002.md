## Solution

### 實作概要

已修改 `_collect_stale_handoffs()` 函式以支援 .md 格式檔案掃描：

1. **同時掃描格式**：第 59 行修改為 `sorted(pending_dir.glob("*.json")) + sorted(pending_dir.glob("*.md"))`，與 resume.py 的 list_pending_handoffs() 一致（line 132）

2. **.md 格式解析邏輯**：
   - 行 92-100：新增 `.md` 後綴檢查分支
   - 利用檔名 stem（無副檔名）作為 ticket_id
   - 由於 Markdown 格式無 direction/from_status 欄位，直接判斷：若 ticket 已 completed，則視為 stale
   - 與 resume.py line 189-191 行為一致

3. **判斷規則維持一致**：
   - JSON 檔：保留原有邏輯（檢查 direction 和 from_status，處理任務鏈特殊情況）
   - Markdown 檔：簡化邏輯（直接以 is_ticket_completed 判斷）
   - 兩種格式都使用共用的判斷函式（handoff_utils.py）

### 關鍵變更

| 項目 | 修改內容 |
|------|---------|
| 掃描邏輯 | 修改 glob 呼叫以同時掃描 *.json 和 *.md |
| 檔案類型判斷 | 新增 `if handoff_file.suffix` 分支判斷 |
| .md 解析 | 提取 stem 作為 ticket_id，依 is_ticket_completed() 判斷 stale |
| 向後相容 | JSON 邏輯無變更，確保現有行為不受影響 |
