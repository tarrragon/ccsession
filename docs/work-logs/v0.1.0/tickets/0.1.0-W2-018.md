修復 ticket resume 完成後自動清除 pending handoff JSON 檔案

---

## Problem Analysis

`_execute_resume()` 在 `.claude/skills/ticket/ticket_system/commands/resume.py` 第 486-496 行中，呼叫 `mark_handoff_as_resumed()` 成功後只更新了 `resumed_at` 時間戳，但沒有將 JSON 檔案從 `pending/` 移到 `archive/`。

**根本原因**：

1. `mark_handoff_as_resumed()` 對 Markdown 格式自動歸檔（第 259-261 行），但對 JSON 格式只更新時間戳，不歸檔
2. `_execute_resume()` 缺少呼叫 `archive_handoff_file()` 的程式碼

**影響**：handoff-reminder-hook 在每次 session 中重複掃描 pending/ 目錄，已恢復的 JSON 檔案仍在，導致重複提醒已恢復的任務。

**Edge Case 分析**：

- Markdown 格式：`mark_handoff_as_resumed()` 返回 `archive_handoff_file()` 結果，檔案已歸檔
- JSON 格式：`mark_handoff_as_resumed()` 只更新 resumed_at，檔案仍在 pending/
- 歸檔失敗處理：`archive_handoff_file()` 在檔案不存在時返回 False，應只發出警告而非視為核心失敗

---

## Solution

**修改策略**：

在 `_execute_resume()` 第 487-490 行，`mark_handoff_as_resumed()` 成功後新增呼叫 `archive_handoff_file()`，將 JSON 檔案從 pending/ 移至 archive/。

**實作重點**：

1. 在 `_execute_resume()` 中新增：
   ```python
   if not archive_handoff_file(ticket_id):
       print(format_warning(WarningMessages.INVALID_OPERATION))
       print(WarningMessages.HANDOFF_ARCHIVE_FAILED)
   ```

2. 在 `WarningMessages` 中新增常數：
   ```python
   HANDOFF_ARCHIVE_FAILED = "無法歸檔 handoff 檔案到 archive/（檔案已不存在或權限問題）"
   ```

3. 歸檔失敗不應視為 resume 失敗，只輸出警告繼續執行

**為什麼安全**：

- 核心功能（載入 handoff、更新 resumed_at）已完成
- 歸檔只是收尾，失敗不阻止恢復流程
- 對 Markdown 格式無影響（已在 `mark_handoff_as_resumed()` 中自動歸檔）
- `archive_handoff_file()` 在檔案不存在時安全返回 False
