## 根本原因分析

### 現有 Hook 覆蓋狀況

**acceptance-gate-hook（PreToolUse, Bash matcher）**：
- 在 `ticket track complete` 執行**前**觸發
- 輸出 COMPLETE_REMINDER（驗收方式確認）
- 輸出 COMPLETE_NEXT_STEP_REMINDER（完成後下一步，僅無阻擋時）

**commit-handoff-hook（PostToolUse, Bash matcher）**：
- 在 git commit 成功**後**觸發
- 輸出 COMMIT_HANDOFF_REMINDER（Checkpoint 2 路由）

**關鍵發現：缺少 ticket track complete 成功後的 PostToolUse 覆蓋**

### Checkpoint 流程 vs Hook 覆蓋對照表

| Checkpoint | 規則要求 | Hook 覆蓋 | 狀態 |
|-----------|---------|----------|------|
| 第七層：驗收方式確認 | AskUserQuestion #1 | acceptance-gate（PreToolUse） | 部分覆蓋（時機過早） |
| Checkpoint 1：git status | 檢查未提交變更 | 無 | **未覆蓋** |
| Checkpoint 1.5：錯誤學習 | AskUserQuestion #16 | commit-handoff（提示文字） | **僅文字提示，不強制** |
| Checkpoint 2：Wave 路由 | AskUserQuestion #11/13 | commit-handoff（PostToolUse） | 已覆蓋（在 commit 後） |

### 三層根因

**第一層（設計邊界）：提醒時機設計問題**

COMPLETE_NEXT_STEP_REMINDER 在 PreToolUse 觸發，與「驗收確認」提醒混合出現。PM 閱讀時關注焦點是「我能 complete 嗎？」，對「complete 後做什麼」的文字提醒注意力低。

**第二層（使用情境）：缺少 PostToolUse 觸發點**

`ticket track complete` 執行成功後，CLI 輸出「已完成 xxxx」，此時無任何 Hook 強制輸出 Checkpoint 提醒。PM 看到成功訊息，認知上完成感觸發，容易停止。

**第三層（說明充分性）：規則與執行點解耦**

決策樹第八層的 Checkpoint 說明詳細（decision-tree.md），但文字說明和 Hook 提醒之間有落差。commit-handoff-hook 的提示文字中已提到「#16」，但格式是補充說明，不夠強制。

## 改善方案

### 方案比較

**方案 A：修改 acceptance-gate-hook 的提醒措辭（低成本）**
- 強化 COMPLETE_NEXT_STEP_REMINDER 的語氣
- 缺點：時機仍是 pre-complete，PM 注意力集中在驗收確認，效果有限

**方案 B：新增 post-ticket-complete-hook（推薦）**
- PostToolUse 偵測 `ticket track complete` 命令成功
- 強制輸出 Checkpoint 1/1.5 流程提醒
- 優點：時機正確（complete 成功後立即觸發），形成「完成感→強制提醒」的正確時序
- 實作：偵測 stdout 含「已完成」或「Ticket.*completed」標記

**方案 C：在 commit-handoff-hook 中加強 Checkpoint 1.5（低成本補充）**
- 在現有 COMMIT_HANDOFF_REMINDER 中將 #16（錯誤學習）放到更顯著位置
- 缺點：只解決 commit 後的問題，不解決 ticket complete → git commit 之間的空窗

### 推薦方案

採用 **B + C 組合**：

**B：新增 post-ticket-complete-checkpoint-hook（IMP Ticket）**

Hook 設計：
- 類型：PostToolUse，matcher: Bash
- 觸發條件：command 含 `ticket track complete`，stdout 含成功標記
- 輸出訊息（additionalContext）：
  ```
  ============================================================
  [強制提醒] Checkpoint 1 — ticket complete 後必須執行
  ============================================================
  
  ticket track complete 已成功。下一步強制流程：
  
  [Checkpoint 1] git status
    → 有未提交變更 → /commit-as-prompt
  
  [Checkpoint 1.5] AskUserQuestion #16（錯誤學習）
    → 本 Ticket 執行期間是否有新發現的錯誤模式？
    → ToolSearch("select:AskUserQuestion") 載入後使用
  
  [Checkpoint 2] 在 commit 後由 commit-handoff-hook 觸發
  
  禁止：直接結束回應或進入下一個 Ticket（跳過 Checkpoint 1/1.5）
  詳見: .claude/rules/core/decision-tree.md（第八層）
  ============================================================
  ```

**C：強化 commit-handoff-hook 中的 #16 提醒位置（ADJ Ticket）**
- 將 #16（錯誤學習確認）移到 COMMIT_HANDOFF_REMINDER 的第一個動作項目
- 強調「先 #16，再 #11」

### 追蹤 Ticket

- IMP Ticket：實作 post-ticket-complete-checkpoint-hook
- ADJ Ticket：調整 commit-handoff-hook 的 #16 提醒順序
