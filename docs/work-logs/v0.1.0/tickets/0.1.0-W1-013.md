# Execution Log

## Task Summary

補充 UC-006 新 project 目錄初始載入行為（M2）

---

## Problem Analysis

W1-006 二次審查發現 M2 問題：UC-006 的 A2（新 Project 目錄建立）僅描述「偵測到新子目錄，開始監控其下的檔案」，但未定義以下關鍵行為：

1. 新 project 目錄被偵測時，目錄內可能已存在多個 JSONL 檔案的掃描策略
2. 多個 JSONL 檔案的載入順序和並行控制
3. 一檔一 session 的對應關係和獨立管理方式
4. 檔案數量過多時的總量限制策略

---

## Solution

擴充 A2 章節，新增兩個子章節：

1. **A2 既存 JSONL 掃描規則** -- 定義掃描範圍（僅 .jsonl、不遞迴）、檔案過濾、載入順序（mtime 由舊到新）、單檔載入方式（同 A3）、並行控制（序列載入避免 I/O 暴增）
2. **A2 多 JSONL 檔案處理方式** -- 定義一檔一 session 對應關係、獨立 goroutine 和 offset、總量限制（預設 50，按 mtime 保留最新）
