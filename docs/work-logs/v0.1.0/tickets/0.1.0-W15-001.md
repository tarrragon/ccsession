### 三、Undertrigger 風險分析

#### HIGH 風險（無觸發條件）

**provider-architecture**
- 當前：`"Riverpod Provider 架構設計規範 - 確保正確的依賴注入、介面隔離和測試可行性"`
- 問題：無 Use for/when，無任何觸發關鍵字，Claude 無法判斷何時自動觸發
- 建議：新增 `Use for: designing provider architecture, implementing Riverpod DI patterns, reviewing testability. Use when: creating new providers, refactoring injection, reviewing architecture design.`

**strategic-compact**
- 當前：`"Suggests manual context compaction at logical intervals to preserve context through task phases rather than arbitrary auto-compaction."`
- 問題：純功能描述，無觸發條件，user 不知道何時使用
- 建議：新增 `Use when: context window is getting large, before starting a new major task phase, after completing analysis to preserve context.`

**manager**
- 當前：`"主線程管理哲學和做事方法。定義 rosemary-project-manager 作為主管的核心職責、並行化優先原則、非同步心態。這是主線程的核心行為準則。"`
- 問題：無觸發條件。注意：此為 reference type skill，可能刻意設計為主線程讀取規範用
- 建議：若需 auto-trigger，改為 `Use when: deciding how to dispatch tasks, evaluating parallelization strategy, coordinating multiple agents.`

#### MEDIUM 風險（描述不足）

**version-release**
- 問題：description 描述內部流程步驟（Pre-flight → Document Updates → Git Operations，三步驟流程）而非「何時使用」
- 建議：改寫首句為 `版本發布整合工具。Use when: releasing a version, publishing to production, creating git tag, preparing release notes.`

**tdd-phase1-split**
- 問題：純中文 description，無 Use for/when，缺少 Phase 1 觸發關鍵字，英文查詢可能不觸發
- 建議：新增 `Use for: TDD Phase 1 design, SOLID-driven feature splitting. Use when: starting Phase 1 functional design, analyzing feature scope before implementation.`

**lsp-first**
- 問題：使用「用於:」而非「Use for:」格式，缺少「Use when」觸發條件
- 建議：補充 `Use when: choosing search tools (Grep vs LSP), setting up language server, deciding between search approaches.`

#### LOW 風險（可優化）

| Skill | 建議 |
|-------|------|
| mermaid-ascii | 新增 `Use when: user asks to render mermaid diagram, show flowchart as text, visualize workflow in CLI` |
| bulk-evaluate | 補充 `Use when: analyzing N files independently, need to evaluate large dataset without flooding main context` |
| dispatch-strategy-review | body 含 `**版本**: v1.1` 時間敏感資訊，建議移除 |
| design-decision-framework | 同上，body 含 `**版本**: v1.1` `**狀態**: 穩定` |
