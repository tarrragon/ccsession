根本問題：askuserquestion-rules.md 列了 17 個具體場景，但缺少通用原則：「任何需要用戶從選項中選擇的場合 = 必須使用 AskUserQuestion 工具」。Claude 知道要提供選項（流程設計已引導），但沒有把「提供選項」這個動作與「使用 AskUserQuestion 工具」的認知連結起來，導致用文字表格取代真正的互動工具。修復方向：在 askuserquestion-rules.md 頂部加入通用原則，不依賴場景枚舉，而是行為觸發規則：只要輸出含選項供用戶選擇，就必須使用 AskUserQuestion。
## Solution

<!-- To be filled by executing agent -->

---

在 askuserquestion-rules.md 頂部（## 背景之前）新增「## 通用觸發原則（行為驅動，優先於場景枚舉）」章節。核心原則：任何時候，只要輸出選項供用戶選擇，就必須使用 AskUserQuestion 工具。判斷方式為單一問題（是/否），不需對照 17 個場景清單。明確說明 17 個場景僅為常見情境範例，非觸發條件完整清單。版本從 2.3.0 升至 2.4.0。
