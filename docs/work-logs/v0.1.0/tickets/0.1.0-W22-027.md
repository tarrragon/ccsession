## Solution

### 修改 1：`.claude/hooks/lib/ask_user_question_reminders.py`

**修改 1a：更新 `COMMIT_HANDOFF_REMINDER` 為二元確認**
- 行 141：`選項：無需記錄 (Recommended) / 記錄錯誤學習 / 稍後記錄`
  → `選項：無需記錄 (Recommended) / 記錄錯誤學習`
- 行 142：`重新確認 #16 直到選擇「無需記錄」或「稍後記錄」`
  → `重新確認 #16 直到選擇「無需記錄」`

**修改 1b：新增 `COMMIT_HANDOFF_SKIP16_REMINDER` 常數**
在 `COMMIT_HANDOFF_REMINDER` 之後（line 168 後）新增常數，供文件/格式/維護類 commit 使用，自動跳過 #16，直接顯示 #11 指引。

### 修改 2：`.claude/hooks/commit-handoff-hook.py`

**修改 2a：更新 import**
```python
from lib.ask_user_question_reminders import AskUserQuestionReminders
```

**修改 2b：新增常數 `SKIP_SCENE16_COMMIT_PREFIXES`**
```python
SKIP_SCENE16_COMMIT_PREFIXES = frozenset({
    "docs", "chore", "style", "revert", "test", "ci", "build"
})
```

**修改 2c：新增函式 `extract_commit_type()`**
從 git commit 命令提取 conventional commit 類型，支援 `-m "type: ..."` 和 heredoc 格式。

**修改 2d：修改 `main()` 輸出邏輯**
- 提取 commit 類型（使用 `extract_commit_type()`）
- 判斷是否應跳過 #16（檢查是否在 `SKIP_SCENE16_COMMIT_PREFIXES` 中）
- 根據判斷結果選擇 reminder：
  - 應跳過：使用 `COMMIT_HANDOFF_SKIP16_REMINDER`
  - 否則：使用 `COMMIT_HANDOFF_REMINDER`
