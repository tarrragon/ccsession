實施修復：

1. 新增 import：unicodedata 標準庫
2. 修改 calculate_visual_width()：
   - 原：手動碼位範圍判斷 (0x4E00 <= char_code <= 0x9FFF)
   - 新：使用 unicodedata.east_asian_width()，覆蓋所有寬字元（W/F = 2 寬）
   - 優點：完整支援 CJK 擴充、全形標點、全形符號等

3. 修改 truncate_title()：
   - 原：邊界條件 'total_width + char_width >= max_length'
   - 新：邊界條件 'total_width + char_width > max_length'
   - 原因：修正邊界判斷（>= 會導致恰好填滿時也截斷）
   - 說明：當累加寬度超過上限時才截斷，相等時不截斷

修改原則：
- 遵循 PEP 8 風格，無硬編碼常數
- 認知負擔評分：函式長度 12 行（< 15），判斷邏輯清晰（< 10）
- 完整的文件字串說明修改邏輯
