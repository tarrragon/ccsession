操作失誤（PC-005、PC-007 等）的根本問題在於只修復表面症狀（禁止行為清單），未分析三層根因（設計邊界/使用情境/說明充分性）。現有 systematic-debugging 面向程式碼層次，incident-response CLI 調查面向工具失敗，操作行為失誤這一層缺乏對應方法論。
## Solution

1. 建立 `.claude/methodologies/operational-error-root-cause-methodology.md`：定義三層框架（設計邊界/使用情境/說明充分性），每層各 3 個分析問題，包含觸發條件、分析模板、與現有文件的區隔說明。
2. 補充 `.claude/rules/flows/incident-response.md` 「操作失誤根因分析」章節，引用方法論並加入強制三層分析規則。
3. 更新 `.claude/rules/guides/methodology-index.md` 加入新方法論條目。
