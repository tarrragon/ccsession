根因確認：creation-acceptance-gate-hook.py 的 check_creation_accepted() 函式（約第 139 行）只檢查 creation_accepted 欄位，不檢查 ticket 的 current status。

矛盾狀態觸發路徑：
1. Ticket 在 creation_accepted gate 加入前就已被 claim → status: in_progress, creation_accepted: false
2. 用戶 Resume 後，CLI 建議執行 ticket track claim <id>
3. 用戶在 Claude Code prompt 輸入 ticket track claim <id>
4. UserPromptSubmit hook 觸發，讀取 ticket → creation_accepted: false
5. Hook 回傳 EXIT_BLOCK (exit code 2) → Claude CLI session 終止

現有行為 vs 期望行為：
- pending + creation_accepted: false → EXIT_BLOCK（正確，防止未審核 ticket 被認領）
- in_progress + creation_accepted: false → EXIT_BLOCK（錯誤！re-claim 是 no-op，不應終止 session）
- in_progress + creation_accepted: false → 應允許執行並輸出警告（期望行為）

修復方案：修改 check_creation_accepted() 加入 status 判斷，若 status == in_progress 且 creation_accepted: false，回傳 True（允許）並輸出警告，不觸發 EXIT_BLOCK。修改範圍：僅 creation-acceptance-gate-hook.py 一個函式。
## Solution

---

修改 creation-acceptance-gate-hook.py 的 check_creation_accepted() 函式，加入 status 判斷。

變更摘要：
1. hook_messages.py：新增 CONTRADICTORY_STATE_WARNING 訊息常數
2. creation-acceptance-gate-hook.py：
   - check_creation_accepted() 加入 status 讀取
   - 若 status == in_progress 且 creation_accepted: false → 回傳 (True, warning_msg)，不觸發 EXIT_BLOCK
   - main() 新增 warning_messages 列表，矛盾狀態警告顯示在 additionalContext 中

行為矩陣（修改後）：
- pending + creation_accepted: false → EXIT_BLOCK (exit 2)（維持原行為）
- in_progress + creation_accepted: false → EXIT_SUCCESS (exit 0) + WARNING（新行為）
- creation_accepted: true（任何 status）→ EXIT_SUCCESS（維持原行為）
