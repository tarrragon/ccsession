# Execution Log

## Task Summary

補充 UC-001 completed-active 狀態轉換和摘要來源描述（H3+M11+L1）

---

## Problem Analysis

W1-006 二次審查發現 UC-001 有 3 項缺漏：

1. **H3（高）**：UC-008 狀態機定義 completed->active 回升路徑，但 UC-001 替代流程 A2 僅描述降級方向（active->idle, idle->completed），未處理 session 從 completed 回到 active 時的 UI 行為。
2. **M11（中）**：UC-001 摘要備用來源寫「history.jsonl 的第一個 user message」，但 UC-008 元資料表寫「第一個 user prompt（前 100 字元）」，來源檔案名稱不一致（history.jsonl vs session JSONL）。
3. **L1（低）**：UC-001 未定義 session 狀態變更時列表項目的動畫過渡行為。

---

## Solution

### H3 修正：新增替代流程 A4（狀態回升）

- 將原 A2 標題更精確為「Session 狀態降級」
- 新增 A4 流程，描述 completed->active 和 idle->active 的回升情境
- A4 包含：Backend 推送 session_status_change、Frontend 將 session 移至 Active 群組頂部、短暫高亮提示使用者

### M11 修正：統一摘要來源描述

- 將「history.jsonl 的第一個 user message」改為「該 session 的 JSONL 檔案中第一個 user prompt」
- 新增說明：此資料由 Backend 解析後透過 metadata 提供，引導讀者參考 UC-008 Session Metadata 來源優先級
- 術語統一使用「user prompt」（與 UC-008 一致）

### L1 修正：新增列表動畫規範章節

- 在 UI 草圖前新增「列表動畫規範」章節
- 定義 4 種事件的動畫類型和時長（新增、降級、回升、移除）
- 補充設計原則：批量變更合併動畫、動畫不阻擋互動
