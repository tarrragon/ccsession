# Execution Log

## Task Summary

補充 UC-008 eventCount 計算定義和摘要來源統一（M3+M11）

---

## Problem Analysis

W1-006 審查發現 UC-008 有 2 項中優先級缺漏：

1. **M3 - eventCount 計算定義缺失**：UC-008 僅寫「JSONL 行數計數」，未明確 eventCount 是計算原始行數還是成功解析的事件數。空行或格式錯誤行的處理方式不明。
2. **M11 - 摘要來源描述不一致**：UC-008 原文引用「history.jsonl」作為摘要備用來源，但 `history.jsonl` 是全域索引檔案，正確的備用來源應為 per-session JSONL 檔案（`[session-uuid].jsonl`）。UC-001 已在其他 Ticket 修正此問題，但 UC-008 作為 canonical 定義源仍有此錯誤。

---

## Solution

### M3 修正

- 表格 eventCount 備用來源欄位：「JSONL 行數計數」改為「session JSONL 成功解析事件數」
- 優先級邏輯說明 Event Count 段落：補充「計算成功解析為合法 JSON 物件的事件行數，空行、格式錯誤行不計入」

### M11 修正

- 表格 summary 備用來源欄位：加上「session JSONL」前綴明確來源
- 優先級邏輯說明 Summary 段落：標記為 canonical 定義，明確來源為 `[session-uuid].jsonl`，加註「非全域 history.jsonl」
- 替代流程 A1 步驟 3：補充來源為「session JSONL 檔案中第一個 type 為 user 的 message」

修改檔案：
- `docs/usecase/UC-008-session-state.md`（主要修改）
- `docs/usecase/UC-001-session-list.md`（已由其他 Ticket 修正，本次無需變更）
