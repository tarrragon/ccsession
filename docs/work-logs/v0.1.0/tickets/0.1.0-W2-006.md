# Execution Log

## Task Summary

統一 subprocess/git 命令包裝到 hook_utils

---

## Problem Analysis

**現狀**：
1. 5 個 hook 重複實作 `get_project_root()` 邏輯，使用 `__import__("subprocess")` 反模式
   - handoff-cleanup-hook.py
   - handoff-prompt-reminder-hook.py
   - handoff-reminder-hook.py
   - handoff-auto-resume-stop-hook.py
   - tech-debt-reminder.py

2. 3 個 hook 使用 git subprocess 呼叫，無統一的錯誤處理和 timeout 規則
   - changelog-update-hook.py：2 個 git 呼叫（git diff-tree, git log）
   - project-init-env-check-hook.py：1 個 git 呼叫（git rev-parse）
   - prompt-submit-hook.py：1 個 git 呼叫（git log）

3. 違反 DRY 原則，導致維護成本高且不一致

**方案**：
將 `_find_project_root()` 公開為 `get_project_root()`，新增 `run_git()` 統一 git 命令包裝，遷移所有 hook。

---

## Solution

**實作步驟**：

1. **hook_utils.py 新增公開 API**：
   - `get_project_root() -> Path`：公開現有的 `_find_project_root()` 邏輯
   - `run_git(args, cwd=None, timeout=5, logger=None) -> Optional[str]`：統一 git 命令包裝
     - 固定 `capture_output=True, text=True`
     - 統一 timeout=5 秒
     - 處理 TimeoutExpired, FileNotFoundError, OSError
     - 失敗時若有 logger 則記錄 warning
     - 成功回傳 `stdout.strip()`，失敗回傳 None

2. **遷移 5 個 __import__ hook**：
   - 改用 `from hook_utils import get_project_root`
   - 刪除各 hook 中重複的 `get_project_root()` 函式定義

3. **遷移 3 個 git 呼叫 hook**：
   - 改用 `from hook_utils import run_git`
   - 替換 `subprocess.run(['git', ...])` 為 `run_git(['git', ...], logger=logger)`
   - 更新函式簽名加入 logger 參數
   - 更新呼叫端傳遞 logger

**遷移清單**：
- ✅ hook_utils.py：新增 get_project_root() + run_git()
- ✅ handoff-cleanup-hook.py：移除 get_project_root()
- ✅ handoff-prompt-reminder-hook.py：移除 get_project_root()
- ✅ handoff-reminder-hook.py：移除 get_project_root()
- ✅ handoff-auto-resume-stop-hook.py：移除 get_project_root()
- ✅ tech-debt-reminder.py：移除 get_project_root()
- ✅ changelog-update-hook.py：改用 run_git()（2 個呼叫）
- ✅ project-init-env-check-hook.py：改用 run_git()（1 個呼叫）
- ✅ prompt-submit-hook.py：改用 run_git()（1 個呼叫）
