# Execution Log

## Task Summary

修復 ticket track list --status 分隔符歧義（nargs=+ 與 batch-complete 相容性）

---

## Problem Analysis

### 根本原因
track.py 行 250 的 --status 參數定義使用 `type=lambda s: s.split(",")`，強制使用逗號分隔，與 decision-tree.md 引導的空格分隔不一致。

### 確認的位置（已確認）
- `.claude/skills/ticket/ticket_system/commands/track.py` 行 250：track list 的 --status 定義
- `.claude/skills/ticket/ticket_system/commands/track.py` 行 365：batch-complete 的 --status 定義（獨立，不受影響）
- `.claude/skills/ticket/ticket_system/lib/track_query.py`：_build_status_filters() 已支援列表迭代

### 設計衝突
- W31+W32 多視角審查建議：track list --status 改用空格分隔（nargs=+）
- 同時需確保 batch-complete 的 --status 不受影響（預設值「in_progress」字串保留）

---

## Solution

### 修改內容
**檔案**：`.claude/skills/ticket/ticket_system/commands/track.py`
**行號**：250
**修改**：
```python
# 原始：
p_list.add_argument("--status", type=lambda s: s.split(","), help=TrackMessages.ARG_STATUS)

# 修改為：
p_list.add_argument("--status", nargs='+', help=TrackMessages.ARG_STATUS)
```

### 相容性驗證
- ✅ batch-complete 的 --status（行 365）為獨立定義，保留 `default="in_progress"` 字串
- ✅ _build_status_filters()（track_query.py 行 408-429）已正確處理列表迭代，無需修改
- ✅ _collect_ids_from_wave() 使用 batch-complete 的 status 字串，不受影響
