# Execution Log

## Task Summary

重構 handoff.py _print_status() 函式拆分（90行→多個子函式）

---

## Problem Analysis

原函式 `_print_status()` 位於 L501-590，90 行程式碼，包含：
- 5 個區域變數（ticket_id, status, title, chain, children）
- 多層巢狀（最深 3 層：if-for-if）
- 複雜分支邏輯（direction 判斷，to-sibling 含特殊邏輯）
- version 重複提取（L530 和 L546 各提取一次）

認知負擔指數：5 變數 + 4 方向分支 + 3 層巢狀 + 多層 isinstance 判斷 = > 15，超過上限

---

## Solution

### 拆分策略

按功能區塊拆分為 7 個子函式：

1. **_extract_version_from_ticket_id()** - 5 行
   - 提取版本號邏輯獨立為函式
   - 消除重複的 version 提取（原 L530, L546）

2. **_print_header()** - 13 行
   - 列印 ticket ID、title、status、depth
   - 列印 chain 資訊（root、parent）
   - 認知負擔指數: 4

3. **_print_children_info()** - 15 行
   - 列印子任務資訊
   - 支援字串和 dict 兩種格式
   - 認知負擔指數: 6

4. **_print_direction_options()** - 11 行
   - 主分發函式（5 種方向）
   - 使用委派避免巢狀，認知負擔指數: 7
   - 呼叫 _print_to_child_options / _print_to_sibling_options

5. **_print_to_child_options()** - 7 行
   - to-child 方向邏輯獨立
   - 認知負擔指數: 3

6. **_print_to_sibling_options()** - 22 行
   - to-sibling 方向邏輯獨立（最複雜的分支）
   - 載入父任務，過濾兄弟
   - 認知負擔指數: 8

7. **_print_status()** （重構後）- 15 行
   - 主協調函式
   - 依序呼叫各子函式
   - 認知負擔指數: 3 ✓

### 關鍵改進

- ✓ 避免重複：版本號在主函式提取一次，傳入各子函式
- ✓ 巢狀深度降低：from 3→1（使用委派）
- ✓ 單一責任：每個子函式只做一件事
- ✓ 型別標註：完整的 Optional 和 dict 型別註記
- ✓ 文件字串：每個函式都有完整的 docstring
