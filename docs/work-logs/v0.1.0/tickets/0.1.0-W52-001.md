# 0.1.0-W52-001: 版本檔檢查邏輯重構

## 背景

專案為 Go + Flutter 混合專案，但版本檢查邏輯硬編碼檢查根目錄的 `pubspec.yaml`。這導致：

- 非純 Flutter 專案（如本專案）無法正確檢查 Flutter 前端版本
- `ui/pubspec.yaml` 子目錄的版本檔被忽視
- 不支援 Node.js (package.json)、Python (pyproject.toml) 等其他語言
- 純規格版本（無程式碼產物）檢查失敗並阻塞版本發布流程

## 設計方案

新增語言感知的動態版本檔偵測機制：

| 組件 | 說明 |
|------|------|
| `VERSION_FILE_CANDIDATES` | 常數清單，定義優先順序：根目錄 pubspec.yaml → ui/ 子目錄 pubspec.yaml → package.json → pyproject.toml |
| `detect_version_files()` | 偵測專案中存在的版本檔案，按優先順序回傳 |
| `extract_version_from_file()` | 根據檔案類型動態解析版本號（YAML/JSON/TOML） |
| `verify_version_files()` | 新函式（替代 `verify_pubspec_version()`），檢查所有偵測到的版本檔 |

### 關鍵改進

1. **動態偵測**：依優先順序掃描 4 種版本檔案位置，適應不同專案結構
2. **格式支援**：
   - YAML：`version: X.Y.Z`（Flutter）
   - JSON：`{ "version": "X.Y.Z" }`（Node.js）
   - TOML：`version = "X.Y.Z"`（Python，使用 regex fallback）
3. **安全降級**：無版本檔時輸出警告但不阻塞檢查，支援純規格版本發布
4. **向後相容**：純 Flutter 專案（根目錄 pubspec.yaml）仍然正常工作

## 修改位置

### 1. `detect_version()` - 自動版本偵測
- 改用 `detect_version_files()` + `extract_version_from_file()` 替代硬編碼的 pubspec.yaml 檢查
- 優先順序保持不變：git 分支 → 版本檔 → git tag

### 2. `check_version_sync()` - 版本同步檢查
- 改用動態偵測代替硬編碼檢查
- 無版本檔時輸出警告但不加入 errors 清單（不阻塞）
- 已存在版本檔時仍然執行完整的版本一致性檢查

### 3. `verify_version_files()` - 版本驗證
- 新函式名稱表示多檔案支援（取代 `verify_pubspec_version()`）
- 無版本檔時回傳 True（不阻塞發布）
- 逐一檢查所有偵測到的版本檔

### 4. `update_documents()` - 文件更新
- 改為呼叫 `verify_version_files()` 而非 `verify_pubspec_version()`
- 提示文本改為「確認版本號」（泛化語言）

## 技術細節

### TOML 解析
腳本已聲明 `requires-python = ">=3.10"`，但 tomllib 在 Python 3.11+ 才提供。使用 regex 是更安全的 fallback：

```python
match = re.search(r'^version\s*=\s*["\']([^"\']+)["\']', content, re.MULTILINE)
```

### 錯誤處理
- 所有解析異常被 try-except 捕獲，回傳 None
- 呼叫端檢查 None 並適當處理（報錯或警告）
