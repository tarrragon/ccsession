# Execution Log

## Task Summary

重構 提取分隔線魔法數字為具名常數並統一寬度

---

## Problem Analysis

整個 ticket_system 中硬編碼了三種分隔線寬度，導致視覺不一致：

- `"=" * 60`：scripts/ticket.py（5處）、lib/messages.py（6處）、大部分 commands/（多處）
- `"=" * 50`：lib/wave_calculator.py（1處）、lib/critical_path.py（2處）
- `"-" * 50`：lib/critical_path.py（1處）
- `"=" * 70`：commands/track_board.py（2處，看板寬表格用）
- `"-" * 70`：commands/track_board.py（1處）

ui_constants.py 已有 `SEPARATOR_CHAR = "-"` 和 `SEPARATOR_WIDTH = 80`，但缺少三層架構常數，各處寬度分散（60/50/70/80）。

---

## Solution

**新增三層分隔線常數架構**（ui_constants.py）：
- `SEPARATOR_PRIMARY = "=" * 60`：命令頂層標題、主要章節分隔
- `SEPARATOR_SECONDARY = "=" * 50`：摘要分析、Wave 建議、內嵌分隔
- `SEPARATOR_WIDE = "=" * 70`：ASCII 看板表格寬數據對齊
- `SEPARATOR_SECONDARY_DASH = "-" * 50`：次級橫線（表格行分隔）
- `SEPARATOR_WIDE_DASH = "-" * 70`：寬橫線（看板表格行分隔）

**並行派發 12 個代理人**各負責 1-2 個檔案，替換計 45+ 處硬編碼。

**特殊情況 — track_board.py `__main__` 塊**：
直接執行模式（`python3 track_board.py`）無法使用套件 import，使用局部常數 `_SEP = "=" * 60` 統一 3 次 print，消除重複但保留單一參考點。

**out-of-scope — command_tracking_messages.py line 267**：
`AGENT_SEPARATOR = "=" * 60` 在 W2-022 where.files 範圍外，已識別但不在本 ticket 修改範圍。
