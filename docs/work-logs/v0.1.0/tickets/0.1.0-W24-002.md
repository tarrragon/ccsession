## 修復 1：移除 blockedBy dead code

### 問題分析
在 `find_parallelizable_tickets` 函式中存在邏輯冗餘：

1. **Line 432**：`unblocked = [t for t in pending_tickets if not t.get("blockedBy")]` 
   - 此行已完整過濾所有有 blockedBy 欄位的 Ticket，只保留無 blockedBy 的

2. **Line 447-448（已刪除）**：
   ```python
   blocked_ids_1 = set(filter(None, re.split(r'[,\s]+', ticket1.get("blockedBy", ""))))
   ```
   - ticket1 已經過篩選確保 blockedBy 為空，此處解析的 blockedBy 永遠為空
   - 結果 blocked_ids_1 恆為空 set

3. **Line 455（已刪除）**：同樣問題存在於 ticket2

4. **Line 457（已刪除）**：
   ```python
   if ticket2["id"] not in blocked_ids_1 and ticket1["id"] not in blocked_ids_2:
   ```
   - 兩個 blocked_ids 都永遠為空，條件恆為 True
   - 邏輯冗餘且無實際作用

### 根本原因
第一層過濾（line 432）已完全確保無 blockedBy，二次檢查是多餘的。這是重構時的遺留產物。
