## 修改摘要

### 1. tdd-flow.md（轉換動作章節）
- 將「轉換動作」拆分為兩類：Phase 1/2/3a（代理人自治）和 Phase 3b/4a/4b/4c（PM 管理）
- Phase 1-3 代理人自行：更新 Ticket Log → git commit → complete Ticket → 回報成功/失敗
- commit message 格式：`feat({ticket-id}): Phase X 完成 - {摘要}`

### 2. parallel-dispatch.md（新增 Commit 責任邊界章節）
- 定義誰 commit、誰 push、commit message 格式的責任矩陣
- Phase 1/2/3a 代理人自行 commit，禁止 push（PM 統一 push）
- Phase 3b+ 由 PM 管理 commit
- Hook 相容性確認：commit-handoff-hook 在 subagent 中自然跳過 AskUserQuestion

### 3. tdd-flow-details.md（新增自治執行規範章節）
- 自治範圍對照表
- 代理人自治執行清單（6 步驟）
- 回報格式（成功/失敗，禁止完整報告）
- PM 收到回報後全自動動作（對應決策樹情境 D1）

### 一致性驗證
- decision-tree.md 情境 D1「Phase 1/2 → 全自動下一 Phase」與新規範一致
- parallel-dispatch.md 代理人回報原則與新增 Commit 責任邊界互補不矛盾
- commit-handoff-hook 在 subagent 中 AskUserQuestion 自然跳過，無相容性問題
