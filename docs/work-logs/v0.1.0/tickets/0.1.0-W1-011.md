# Execution Log

## Task Summary

補充 UC-004 多 Panel 狀態持久化定義（M6）

---

## Problem Analysis

W1-006 二次審查發現 UC-004 的 M6 問題：多 Panel 版面狀態持久化規則未定義。原文件僅在驗收條件中提及「分割狀態在應用重啟後可恢復（Phase 4 配置持久化）」，但缺乏具體的持久化資料模型、行為規則、恢復策略，以及 Phase 1 與 Phase 4 的範圍界定。

---

## Solution

在 UC-004 的「驗收條件」段落前新增「狀態持久化定義」章節，包含以下內容：

1. **持久化資料模型**：定義 layoutMode（分割模式 enum）和 panels 清單（含 panelIndex、sessionId、scrollPosition）
2. **持久化行為規則**：定義切換模式、選擇 session、關閉面板、應用關閉時的寫入時機
3. **恢復行為規則**：定義正常重啟、session 已刪除、檔案損壞、面板數不一致等 4 種情境的恢復策略
4. **Phase 1 vs Phase 4 範圍表**：Phase 1 僅持久化 layoutMode 和 sessionId（最小可用）；Phase 4 擴充捲動位置、面板大小比例、最大化狀態
5. **儲存位置**：明確為 Flutter 端本地儲存，不經 Go Backend

同步更新驗收條件，將原有的模糊描述拆分為 4 個可驗證的具體條件。
