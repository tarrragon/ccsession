## Problem Analysis

<!-- To be filled by executing agent -->

---

list_pending_handoffs() 在處理任務鏈 handoff 時（direction 為 to-sibling:target_id），只檢查來源 ticket 是否已 completed，但未檢查目標 ticket 是否已啟動。導致 W3-008 完成並 handoff 到 W3-009 後，即使 W3-009 已 in_progress/completed，handoff JSON 仍持續顯示於 resume --list。
## Solution

<!-- To be filled by executing agent -->

---

1. 新增 _is_ticket_in_progress_or_completed() 輔助函式（resume.py:116）
2. 在 list_pending_handoffs() 任務鏈判斷後，提取 direction 中的 target_id
3. 若目標已 in_progress/completed，continue 過濾為 stale
4. 無 target_id 的任務鏈（如 to-sibling）保留原有行為
5. 新增 5 個 TestIsTicketInProgressOrCompleted 單元測試 + 4 個整合測試
