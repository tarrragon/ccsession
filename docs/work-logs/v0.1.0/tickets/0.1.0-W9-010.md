## Problem Analysis

<!-- To be filled by executing agent -->

---

extract_version_from_ticket_id 是純字串函式（無 IO、無副作用），但定義在 ticket_ops.py（命令層工具集，結合載入、驗證和錯誤輸出）。per CQ-006，純工具函式應放在 lib/ 的純層模組，而非 IO-heavy 模組。評估目標層：ticket_validator.py（純粹驗證邏輯，無 IO 操作，與純字串函式性質吻合）。無循環 import 風險（ticket_ops 未 import ticket_validator，反之亦然）。附帶發現：extract_wave_from_ticket_id 在 W9-009 中被加入 ticket_ops.py，同樣有層級不匹配問題，但超出本 Ticket 範圍。
## Solution

<!-- To be filled by executing agent -->

---

將 extract_version_from_ticket_id 從 ticket_ops.py 移至 ticket_validator.py（搬移時同步修正 None guard：原版未處理 ticket_id=None 情況，移至後改為 'if not ticket_id or ...'）。更新 6 個 caller 的 import：handoff_utils.py、ticket_builder.py、chain_analyzer.py（lib/）；resume.py、handoff.py、track.py（commands/）。
