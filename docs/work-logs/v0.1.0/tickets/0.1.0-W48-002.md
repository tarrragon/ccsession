## Problem Analysis

### 建立審核結論（2026-03-13）

原 Ticket 包含場景 #8 和 #10，經 acceptance-auditor + SA 並行審核，發現：

1. **場景 #10 概念混亂（高缺陷）**：「開始/收尾確認」定義模糊。「開始執行」已由 `/ticket track claim` 覆蓋，不需額外 Hook；「收尾」應屬場景 #3。**決定：移除 #10，本 Ticket 僅處理 #8。**

2. **場景 #8 vs #7 邊界（中缺陷）**：經分析，兩者有明確區分：
   - #7「派發方式選擇」：用什麼機制（subagent / Agent Teams / 序列）
   - #8「執行方向確認」：任務的執行順序（先做哪個、哪些並行）
   關鍵字不應重疊。#7 的關鍵字偏向「並行+派發」「agent team」，#8 偏向「先做+哪個」「順序+任務」。

3. **實作策略**：在現有 `prompt-submit-hook.py` 的 `DECISION_KEYWORDS` 字典中擴充，無需新建 Hook 檔案。
