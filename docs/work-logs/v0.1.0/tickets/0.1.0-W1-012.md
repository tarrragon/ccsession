# Execution Log

## Task Summary

補充 UC-005 搜尋計數格式和 Phase 3+ 預留描述（M12+L2）

---

## Problem Analysis

W1-006 二次審查發現 UC-005 有 2 項缺漏：

1. **L2（低優先級）**：UC-005 5.2 節步驟 5 的搜尋結果計數格式僅以範例帶過（「如 3/15」），未明確定義格式規格、無匹配時的顯示、搜尋欄為空時的行為
2. **M12（中優先級）**：UC-005 和 UC-009 的 Phase 3+ 搜尋預留描述為半成品，僅說明「可考慮」但缺乏具體的觸發時機、後端職責、UI 整合方式、協議參數定義

---

## Solution

### L2：搜尋計數格式明確化

修改 UC-005 5.2 節步驟 5，補充：
- 計數格式定義：`{目前位置}/{總匹配數}`
- 無匹配時顯示 `0/0`
- 搜尋欄為空時不顯示計數
- 步驟 6 補充計數隨導航同步更新

### M12：Phase 3+ 搜尋預留描述完整化

UC-005 修改：
- 將原本單行的「Phase 3+ 可考慮...」改為獨立段落
- 補充目的、實現方式、觸發時機、後端職責、UI 整合方式

UC-009 修改：
- 將原本 3 點概述擴充為完整的協議預留定義
- 補充 `search_session` action 的具體參數（sessionId, query, limit, before）
- 補充 `search_results` 回應的 data 結構（matches 陣列、highlights 位置標記、hasMore 分頁旗標）
- 明確標註 Phase 1-2 不實作，Phase 3+ 啟動時再定義錯誤處理和效能需求
