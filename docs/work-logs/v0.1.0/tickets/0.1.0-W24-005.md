## Problem Analysis

<!-- To be filled by executing agent -->

---

cleanup_trigger 使用計數器機制（LOG_CLEANUP_TRIGGER_FREQUENCY=10），每次呼叫都需 read_text/write_text，產生不必要的 I/O。觸發頻率與呼叫次數綁定而非時間間隔，清理行為不可預測。
## Solution

<!-- To be filled by executing agent -->

---

改用 mtime 時間觸發：(1) 移除 LOG_CLEANUP_TRIGGER_FREQUENCY，新增 CLEANUP_INTERVAL_SECONDS=300 (2) 用 stat().st_mtime 檢查時間差，超過閾值才觸發清理 (3) touch() 更新 mtime，不再讀寫檔案內容 (4) except 只需 OSError（移除不再需要的 ValueError）
