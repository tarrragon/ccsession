## Problem Analysis

### W38-001 審核發現的具體缺口

**問題 A：tool_input 子欄位驗證重複**
- `ticket-quality-gate-hook.py` 第 117-122 行：在呼叫 `validate_hook_input()` 後仍手動檢查 `tool_input` 的子欄位 `file_path` 和 `content`
- 影響：呼叫端必須記住額外驗證步驟，無法重用邏輯

**問題 B：Ticket 驗證邏輯分散**
- `agent-ticket-validation-hook.py` 第 131-165 行：包含 find + read + validate_decision_tree 的完整流程
- `command-entrance-gate-hook.py` 第 638-643 行：相似邏輯重複
- 影響：缺少統一入口，導致邏輯重複

**問題 C（保留）：快取邏輯**
- `ticket-quality-gate-hook.py` 的記憶體快取機制（YAGNI 原則暫不提取，只有此 Hook 使用）

---

## Solution

### Phase 1 功能設計完成（已交接）

已在 `0.1.0-W39-001-phase1-design.md` 中定義：

**新增 API**：
1. `validate_tool_input(input_data, logger, required_subfields)` in `hook_io.py`
   - 驗證 tool_input 的子欄位，補充 validate_hook_input 的頂層驗證
   - 支援靜默模式（logger=None）

2. `validate_ticket_unified(ticket_id, project_root, logger)` in `hook_ticket.py`
   - 整合 find + read + validate_decision_tree 的完整流程
   - 回傳 Tuple[bool, Optional[str]]，便於錯誤訊息傳遞

**修改 Hook**：
- `ticket-quality-gate-hook.py`：新增 import，簡化 validate_input() 使用新 API
- `agent-ticket-validation-hook.py`：新增 import，委派 validate_ticket() 給新 API

**re-export 更新**：
- `hook_utils/__init__.py`：新增 validate_tool_input 和 validate_ticket_unified
