**對應 Ticket**: 0.1.0-W39-001
**TDD 階段**: Phase 1 功能設計
**設計者**: lavender-interface-designer

## 一、功能需求分析

### 1.1 核心問題

目前 Hook 系統在「輸入驗證」和「Ticket 驗證」兩個面向存在重複邏輯。SA Phase 0 審查（W39-001）識別出以下具體缺口：

**問題 A：tool_input 子欄位驗證重複**

`ticket-quality-gate-hook.py` 在呼叫 `validate_hook_input(input_data, logger, ("tool_name", "tool_input"))` 之後，仍需手動檢查 `tool_input` 的子欄位 `file_path` 和 `content`（第 117-122 行）。這個「兩段式驗證」模式：
- 使呼叫端必須記住額外的驗證步驟
- 無法讓未來的 Hook 重用相同邏輯
- 若其他 Hook 也需驗證 tool_input 子欄位，會再次手刻

**問題 B：Ticket 驗證邏輯分散**

`agent-ticket-validation-hook.py` 的 `validate_ticket()` 函式（第 131-165 行）包含：
1. 呼叫 `find_ticket_file()` 尋找 Ticket 檔案
2. 讀取 Ticket 內容
3. 呼叫 `validate_ticket_has_decision_tree()` 驗證決策樹欄位

此邏輯與 `command-entrance-gate-hook.py` 的 `check_ticket_status()` 部分重疊（第 638-643 行：find + read + validate_decision_tree）。兩個 Hook 各自組合底層 API，缺少一個「完整 Ticket 驗證」的統一入口。

**問題 C（保留現況）：快取邏輯**

SA 建議依 YAGNI 原則暫不提取快取邏輯，因為目前只有 `ticket-quality-gate-hook.py` 使用記憶體快取。驗收條件中的「快取邏輯提取」項目改為記錄保留原因。

### 1.2 解決方案邊界

本次重構：
- **在 `hook_io.py` 新增** `validate_tool_input()` — 處理 tool_input 子欄位驗證
- **在 `hook_ticket.py` 新增** `validate_ticket_unified()` — 統一 Ticket 存在性 + 決策樹驗證
- **修改** `ticket-quality-gate-hook.py` 的 `validate_input()` 使用新 API
- **修改** `agent-ticket-validation-hook.py` 的 `validate_ticket()` 委派給新 API
- **更新** `hook_utils/__init__.py` re-export 新函式
- **不修改** `parallel-suggestion-hook.py`（已正確使用 `validate_hook_input`）
- **不修改** `command-entrance-gate-hook.py` 的 Ticket 驗證（其 `check_ticket_status` 邏輯較複雜，包含三層驗證；`validate_ticket_unified` 僅服務 agent-ticket-validation-hook 的簡單場景）
- **不提取** 記憶體快取邏輯

## 二、API 介面定義

### 2.1 新增：`validate_tool_input()`

**位置**：`.claude/hooks/hook_utils/hook_io.py`

#### 函式簽名

```python
def validate_tool_input(
    input_data: "dict | None",
    logger: "logging.Logger | None",
    required_subfields: "Tuple[str, ...]"
) -> bool:
```

#### 參數規格

| 參數 | 型別 | 說明 |
|------|------|------|
| `input_data` | `dict \| None` | Hook 的頂層輸入資料（從 stdin 讀取的完整 JSON） |
| `logger` | `logging.Logger \| None` | 日誌記錄器，允許 None（靜默模式） |
| `required_subfields` | `Tuple[str, ...]` | tool_input 必須包含的子欄位清單，如 `("file_path", "content")` |

#### 前置條件（呼叫端責任）

- `input_data` 已通過 `validate_hook_input(input_data, logger, ("tool_name", "tool_input"))` 驗證
- 也就是說，呼叫 `validate_tool_input()` 前，`tool_name` 和 `tool_input` 的頂層存在性已確認

注意：`validate_tool_input()` 內部仍會做防禦性檢查（`tool_input` 存在性），確保即使前置條件未滿足也不會 raise。

#### 回傳值

| 情況 | 回傳值 | 記錄行為 |
|------|--------|---------|
| `input_data` 為 None 或非 dict | `False` | `logger.error("輸入資料為 None 或非 dict")` |
| `tool_input` 欄位不存在或為 None | `False` | `logger.error("tool_input 欄位缺失或為 None")` |
| `tool_input` 非 dict 型別 | `False` | `logger.error("tool_input 非 dict 型別: ...")` |
| 任一 `required_subfields` 不存在 | `False` | `logger.error("tool_input 缺少必要子欄位: {field}")` |
| 任一 `required_subfields` 的值為 None | `False` | `logger.error("tool_input 子欄位值為 None: {field}")` |
| 所有驗證通過 | `True` | `logger.debug("tool_input 子欄位驗證通過")` |

#### 使用範例

```python
# 既有呼叫（保留）
if not validate_hook_input(input_data, logger, ("tool_name", "tool_input")):
    return False

# 新增呼叫（取代手動子欄位驗證）
if not validate_tool_input(input_data, logger, ("file_path", "content")):
    return False
```

#### 完整使用場景（ticket-quality-gate-hook.py）

**修改前：**
```python
def validate_input(input_data: Dict[str, Any], logger) -> bool:
    # 基礎驗證：必要頂層欄位
    if not validate_hook_input(input_data, logger, ("tool_name", "tool_input")):
        return False

    # 額外驗證：tool_input 子欄位（手動）
    tool_input = input_data.get("tool_input") or {}
    if "file_path" not in tool_input or "content" not in tool_input:
        logger.error("tool_input 缺少 file_path 或 content")
        return False

    return True
```

**修改後：**
```python
def validate_input(input_data: Dict[str, Any], logger) -> bool:
    # 頂層欄位驗證
    if not validate_hook_input(input_data, logger, ("tool_name", "tool_input")):
        return False

    # tool_input 子欄位驗證（委派給 hook_utils）
    if not validate_tool_input(input_data, logger, ("file_path", "content")):
        return False

    return True
```

### 2.2 新增：`validate_ticket_unified()`

**位置**：`.claude/hooks/hook_utils/hook_ticket.py`

#### 函式簽名

```python
def validate_ticket_unified(
    ticket_id: str,
    project_root: "Optional[Path]",
    logger: "Optional[logging.Logger]"
) -> Tuple[bool, Optional[str]]:
```

#### 參數規格

| 參數 | 型別 | 說明 |
|------|------|------|
| `ticket_id` | `str` | Ticket ID，如 `"0.1.0-W39-001"` |
| `project_root` | `Optional[Path]` | 專案根目錄；傳 `None` 時自動呼叫 `get_project_root()` |
| `logger` | `Optional[logging.Logger]` | 日誌記錄器；允許 `None`（靜默模式） |

#### 回傳值結構

```python
Tuple[bool, Optional[str]]
# (is_valid, error_message)
```

| 回傳 tuple | 條件 |
|-----------|------|
| `(True, None)` | Ticket 存在且包含決策樹欄位 |
| `(False, "找不到 Ticket: {ticket_id}")` | `find_ticket_file()` 回傳 None |
| `(False, "無法讀取 Ticket 檔案: {ticket_id}")` | 檔案存在但讀取失敗 |
| `(False, "Ticket {ticket_id} 缺少決策樹欄位，為無效 Ticket")` | 決策樹驗證失敗 |

#### 驗證流程

```
validate_ticket_unified(ticket_id, project_root, logger)
    |
    v
[1] project_root 為 None 時，呼叫 get_project_root()
    |
    v
[2] find_ticket_file(ticket_id, project_root, logger)
    +-- 未找到 → return (False, "找不到 Ticket: {ticket_id}")
    |
    v
[3] ticket_path.read_text(encoding="utf-8")
    +-- 讀取失敗 → return (False, "無法讀取 Ticket 檔案: {ticket_id}")
    |
    v
[4] validate_ticket_has_decision_tree(content, logger)
    +-- False → return (False, "Ticket {ticket_id} 缺少決策樹欄位，為無效 Ticket")
    |
    v
[5] return (True, None)
```

#### 使用範例（agent-ticket-validation-hook.py）

**修改前：**
```python
def validate_ticket(ticket_id: str, logger) -> Tuple[bool, Optional[str]]:
    # 尋找 Ticket 檔案
    ticket_path = find_ticket_file(ticket_id, logger=logger)
    if not ticket_path:
        msg = f"找不到 Ticket: {ticket_id}"
        logger.error(msg)
        return False, msg

    # 讀取 Ticket 內容
    content = load_ticket_content(ticket_path, logger)
    if not content:
        msg = f"無法讀取 Ticket 檔案: {ticket_id}"
        logger.error(msg)
        return False, msg

    # 驗證決策樹欄位
    if not validate_ticket_has_decision_tree(content, logger):
        msg = f"Ticket {ticket_id} 缺少決策樹欄位，為無效 Ticket"
        logger.error(msg)
        return False, msg

    logger.info(f"Ticket {ticket_id} 驗證通過")
    return True, None
```

**修改後：**
```python
def validate_ticket(ticket_id: str, logger) -> Tuple[bool, Optional[str]]:
    """驗證 Ticket 的完整性（委派給 hook_utils.validate_ticket_unified）"""
    return validate_ticket_unified(ticket_id, project_root=None, logger=logger)
```

### 3.1 `hook_utils/hook_io.py`

**新增函式**（加在 `validate_hook_input` 定義之後）：

```python
def validate_tool_input(
    input_data: "dict | None",
    logger: "logging.Logger | None" = None,
    required_subfields: "Tuple[str, ...] | None" = None
) -> bool:
    """驗證 tool_input 的子欄位

    補充 validate_hook_input 的頂層欄位驗證，進一步檢查 tool_input 內的
    必要子欄位。呼叫前應已通過 validate_hook_input(input_data, logger,
    ("tool_name", "tool_input")) 驗證。

    Args:
        input_data: Hook 輸入資料（同 validate_hook_input 使用的資料）
        logger: 可選 Logger 實例
        required_subfields: tool_input 必須包含的子欄位清單，如 ("file_path", "content")
                           預設為空，表示只做 tool_input 存在性確認

    Returns:
        bool: tool_input 子欄位驗證是否通過

    Examples:
        # PostToolUse Hook 同時驗證頂層欄位和 tool_input 子欄位
        >>> if not validate_hook_input(data, logger, ("tool_name", "tool_input")):
        ...     return False
        >>> if not validate_tool_input(data, logger, ("file_path", "content")):
        ...     return False
    """
    # 防禦性檢查：input_data 本身
    if input_data is None or not isinstance(input_data, dict):
        if logger:
            logger.error("輸入資料為 None 或非 dict，無法驗證 tool_input 子欄位")
        return False

    # 取得 tool_input
    tool_input = input_data.get("tool_input")
    if tool_input is None:
        if logger:
            logger.error("tool_input 欄位缺失或為 None")
        return False

    if not isinstance(tool_input, dict):
        if logger:
            logger.error("tool_input 非 dict 型別: {}".format(type(tool_input)))
        return False

    # 子欄位驗證
    if required_subfields:
        for field in required_subfields:
            if field not in tool_input:
                if logger:
                    logger.error("tool_input 缺少必要子欄位: {}".format(field))
                return False
            if tool_input.get(field) is None:
                if logger:
                    logger.error("tool_input 子欄位值為 None: {}".format(field))
                return False

    if logger:
        logger.debug("tool_input 子欄位驗證通過")
    return True
```

**型別匯入補充**：確認 `Tuple` 已在 `from typing import List, Optional, Tuple, Dict, Any` 中（目前 hook_io.py 只有 `List, Optional, Tuple, Dict, Any`，Tuple 已存在，無需修改匯入）。

### 3.2 `hook_utils/hook_ticket.py`

**新增函式**（加在 `validate_ticket_has_decision_tree` 定義之後）：

```python
def validate_ticket_unified(
    ticket_id: str,
    project_root: Optional[Path] = None,
    logger: Optional[logging.Logger] = None
) -> Tuple[bool, Optional[str]]:
    """統一的 Ticket 完整性驗證

    整合 find_ticket_file + 讀取內容 + validate_ticket_has_decision_tree
    的完整驗證流程，供 agent-ticket-validation-hook 等 Hook 使用。

    Args:
        ticket_id: Ticket ID，如 "0.1.0-W39-001"
        project_root: 專案根目錄；若為 None 則自動呼叫 get_project_root()
        logger: 可選 Logger 實例

    Returns:
        Tuple[bool, Optional[str]]: (is_valid, error_message)
            - (True, None): Ticket 存在且包含決策樹欄位
            - (False, "錯誤訊息"): 驗證失敗及原因

    Examples:
        >>> is_valid, error_msg = validate_ticket_unified("0.1.0-W39-001", logger=logger)
        >>> if not is_valid:
        ...     logger.error(error_msg)
        ...     return False, error_msg
    """
    # 取得 project_root
    if project_root is None:
        project_root = get_project_root()

    # 步驟 1：尋找 Ticket 檔案
    ticket_path = find_ticket_file(ticket_id, project_root=project_root, logger=logger)
    if not ticket_path:
        msg = "找不到 Ticket: {}".format(ticket_id)
        if logger:
            logger.error(msg)
        return False, msg

    # 步驟 2：讀取 Ticket 內容
    try:
        content = ticket_path.read_text(encoding="utf-8")
    except Exception as e:
        msg = "無法讀取 Ticket 檔案: {}".format(ticket_id)
        if logger:
            logger.error("{}: {}".format(msg, e))
        return False, msg

    if not content:
        msg = "無法讀取 Ticket 檔案: {}".format(ticket_id)
        if logger:
            logger.error(msg)
        return False, msg

    # 步驟 3：驗證決策樹欄位
    if not validate_ticket_has_decision_tree(content, logger):
        msg = "Ticket {} 缺少決策樹欄位，為無效 Ticket".format(ticket_id)
        if logger:
            logger.error(msg)
        return False, msg

    if logger:
        logger.info("Ticket {} 驗證通過".format(ticket_id))
    return True, None
```

**型別匯入確認**：`hook_ticket.py` 已匯入 `from typing import Any, List, NamedTuple, Optional, Tuple, Union`，`Tuple` 和 `Optional` 已存在，無需修改匯入。

## 四、向後相容策略

### 4.1 零破壞原則

本次重構採取「僅新增，不移除底層 API」策略：

| 操作 | 現有 API | 新增 API |
|------|---------|---------|
| Hook 頂層驗證 | `validate_hook_input()` 保留 | `validate_tool_input()` 新增 |
| Ticket 底層查詢 | `find_ticket_file()` 保留 | `validate_ticket_unified()` 新增（組合現有 API） |
| 決策樹驗證 | `validate_ticket_has_decision_tree()` 保留 | （被 `validate_ticket_unified` 內部呼叫） |

### 4.2 現有 53 個 Hook 的影響評估

**不受影響的 Hook**：

- 呼叫 `validate_hook_input()` 的 Hook：函式簽名不變，行為不變
- 呼叫 `find_ticket_file()` 的 Hook：函式不動
- 呼叫 `validate_ticket_has_decision_tree()` 的 Hook：函式不動
- `__init__.py` 的 `__all__` 僅增加項目，不移除

**需驗證的 Hook**（修改了 import 和呼叫）：

| Hook 檔案 | 修改類型 | 風險等級 |
|-----------|---------|---------|
| `ticket-quality-gate-hook.py` | 新增 import，簡化 `validate_input` | 低（行為等價） |
| `agent-ticket-validation-hook.py` | 新增 import，委派 `validate_ticket` | 低（行為等價） |

**行為等價驗證**：
- `validate_tool_input(data, logger, ("file_path", "content"))` 的成功/失敗結果與原手動檢查一致
- `validate_ticket_unified(ticket_id, None, logger)` 的成功/失敗結果與原 `validate_ticket()` 一致

## 五、邊界條件分析

### 5.1 `validate_tool_input()` 邊界條件

| 邊界情況 | 輸入 | 預期行為 |
|---------|------|---------|
| input_data 為 None | `None` | 回傳 `False`，記錄 error |
| input_data 非 dict | `"string"` | 回傳 `False`，記錄 error |
| tool_input 為 None | `{"tool_input": None}` | 回傳 `False`，記錄 error |
| tool_input 為空 dict | `{"tool_input": {}}` | 若 required_subfields 非空則 `False` |
| tool_input 中欄位值為空字串 | `{"tool_input": {"file_path": ""}}` | 回傳 `True`（空字串非 None，通過驗證） |
| tool_input 中欄位值為 None | `{"tool_input": {"file_path": None}}` | 回傳 `False` |
| required_subfields 為空 tuple | `("",)` 或 `()` | 只驗證 tool_input 存在性 |
| required_subfields 為 None | `None` | 只驗證 tool_input 存在性 |
| logger 為 None | `None` | 靜默模式，不寫 log，但回傳正確 bool |

### 5.2 `validate_ticket_unified()` 邊界條件

| 邊界情況 | 輸入 | 預期行為 |
|---------|------|---------|
| ticket_id 為空字串 | `""` | `find_ticket_file("")` 回傳 None，回傳 `(False, "找不到 Ticket: ")` |
| ticket_id 格式非標準 | `"invalid"` | fallback 到全量掃描，找不到則 `(False, ...)` |
| Ticket 檔案存在但內容為空 | 空檔案 | `(False, "無法讀取 Ticket 檔案: {ticket_id}")` |
| Ticket 有決策樹欄位 | 正常 Ticket | `(True, None)` |
| Ticket 無決策樹欄位 | 舊格式 Ticket | `(False, "Ticket {id} 缺少決策樹欄位，為無效 Ticket")` |
| project_root 為 None | `None` | 自動呼叫 `get_project_root()`，正常執行 |
| logger 為 None | `None` | 靜默模式，不寫 log，但回傳正確 tuple |
| 讀取檔案時 PermissionError | 無讀取權限 | `(False, "無法讀取 Ticket 檔案: {ticket_id}")` |

```
       required_subfields = ("file_path", "content")
```

```
Given: input_data = {"tool_name": "Write", "tool_input": {"file_path": "x.md"}}
       required_subfields = ("file_path", "content")
When: validate_tool_input(input_data, logger, required_subfields) 被呼叫
```

```
Given: input_data = {"tool_name": "Write"}
```

```
      logger 記錄 error "tool_input 子欄位值為 None: file_path"
```

### 場景 5：validate_ticket_unified — Ticket 有效

```
Given: ticket_id = "0.1.0-W1-001"

       Ticket 內容包含 "decision_tree_path:" 欄位
```
