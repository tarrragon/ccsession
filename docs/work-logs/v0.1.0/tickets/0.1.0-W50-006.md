## Solution

### 方案設計

1. **constants.py 常數定義**：
   - 新增 `STATUS_SUPERSEDED = "superseded"` 和 `STATUS_CLOSED = "closed"` 常數
   - 擴展 `TICKET_STATUS` 字典，新增 "SUPERSEDED" 和 "CLOSED" 對應
   - 更新 `STATUS_LABELS` 字典，加入新狀態的中文描述（"已取代"、"已關閉"）

2. **ticket_formatter.py 常數引用**：
   - 導入新增的狀態常數和 `STATUS_LABELS`
   - `format_status_icon()` 重構：使用 `STATUS_LABELS.get(status)` 查表而非硬編碼字典
   - `format_ticket_stats()` 重構：使用常數變數（`STATUS_COMPLETED` 等）替換硬編碼字串
   - 實現簡潔的 Guard Clause 模式

3. **test_ticket_formatter.py 向後相容性測試**：
   - 新增 `test_format_stats_backward_compatible_concluded()` 測試
   - 驗證舊格式輸入時 `"[已結案]: 0"` 的正確斷言

### 實施細節

- DRY 原則：消除硬編碼狀態字典，單一事實來源（constants.py）
- 向後相容性：系統自動判別 superseded/closed 狀態，無需修改呼叫方
- 命名風格：常數名稱與現有風格一致（大寫蛇形）
