register() 同時定義 add_subparsers(dest='handoff_command') 和 add_argument('ticket_id', nargs='?')，argparse 優先將位置值匹配為子命令，ticket_id 被誤解為 gc 子命令 positional 導致 'invalid choice' 錯誤。
## Solution

<!-- To be filled by executing agent -->

---

移除 subparsers 機制，改在主 parser 加入 --gc/--dry-run/--execute 旗標。execute() 路由從 handoff_command=='gc' 改為 args.gc。CLI 介面：ticket handoff gc → ticket handoff --gc。
