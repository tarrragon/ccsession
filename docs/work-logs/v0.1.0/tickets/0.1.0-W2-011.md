---

## Problem Analysis

`execute_agent` 函式第 349-352 行存在 4 個獨立 list comprehension，各自遍歷 `agent_tickets`：
- `pending_tickets = [t for t in agent_tickets if t.get("status") == STATUS_PENDING]`
- `in_progress_tickets = [t for t in agent_tickets if t.get("status") == STATUS_IN_PROGRESS]`
- `completed_tickets = [t for t in agent_tickets if t.get("status") == STATUS_COMPLETED]`
- `blocked_tickets = [t for t in agent_tickets if t.get("status") == STATUS_BLOCKED]`

I/O 複雜度：O(n*4)，每次查詢代理人 Tickets 都遍歷清單 4 次。

---

## Solution

將 4x list comprehension 替換為單次遍歷（O(n)）：
- 使用 `status_groups: dict[str, list]` 預建 4 個狀態桶（key 使用 STATUS_* 常數）
- 單次遍歷 `agent_tickets`，依 `status` 分配到對應桶
- 未知狀態直接略過（`if status in status_groups`）
- 最後將 4 個桶的值分配給原有變數名稱，後續邏輯不變
