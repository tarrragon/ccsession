## Problem Analysis

### 教訓來源

五項教訓來自外部專案 `book_overview_app` 的實戰經驗：
- 來源檔案：`/Users/mac-eric/project/book_overview_app/.claude/testing-lessons-learned.md`
- 案例背景：多廚房印表機列印功能，33 個測試全過但線上有 2 個 Bug
- 處理方式：直接將教訓內容整合到 phase2-test-design.md，不複製來源檔案到本專案

### 五項教訓摘要

1. **路徑覆蓋**：測試應驅動被測程式碼的真實路徑，而非用手動構造的資料驗證資料搬運
2. **斷言強度**：斷言要驗證「結果的值」，不要只驗證「結果的存在」（containsKey/isNotEmpty 不等於正確）
3. **依賴完整性**：被測程式碼的所有依賴是否都有提供？缺少的依賴是否被 try-catch 靜默吞掉？
4. **繼承鏈**：覆寫子類別時，要測試「上層呼叫者實際會用到的方法」，而非只測「你覆寫了什麼」
5. **反向驗證**：是否有測試「錯誤情境」來確認你理解了 Bug 的根因？
