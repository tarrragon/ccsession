## Problem Analysis

<!-- To be filled by executing agent -->

---

handoff status to-sibling 方向執行時，parent_ticket 和所有 sibling_ticket 各被 load_ticket（磁碟讀取）3 次：
1. ChainAnalyzer._get_sibling_status（determine_direction 內部）
2. _print_to_sibling_options（print_direction_options 內部）
3. ChainAnalyzer._get_to_sibling_recommendation（get_recommendation 內部）
I/O 複雜度：O(n*3) → parent 3 次 + 每個 sibling 2-3 次
## Solution

<!-- To be filled by executing agent -->

---

在 parser.py 加入 process-scoped 記憶體快取 _ticket_cache（Dict[str, Optional[Dict]]）：
- load_ticket：取得 ticket_path 後先查快取，命中直接回傳；成功載入後更新快取；None（不存在）不快取
- save_ticket：try-finally 結束後失效對應 cache entry
快取 key 使用完整路徑字串（str(ticket_path)），避免版本號正規化問題
