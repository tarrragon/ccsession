## Problem Analysis

<!-- To be filled by executing agent -->

---

三視角並行評估結果（Resume 流程、建立 Skill、全局策略）：

Resume/Claim 現況：完全無 PC-007 防護。
- ticket track claim 後的 6 項清單：阻塞依賴/設計文件/驗收條件/Dev環境/Error-patterns/執行日誌
- 零項涉及「獨立驗證 Ticket 描述的數量/範圍」
- resume.py 結束輸出後無驗證提示

建立端現況：Problem Analysis 模板為空白 HTML 注釋，無調查過程記錄引導。
- command_lifecycle_messages.py 建立後清單無 Problem Analysis 品質項
- ticket_builder.py create_ticket_body() 僅有 <!-- To be filled by executing agent -->

全局評估結論：
- 建立時已有 acceptance-auditor 數字審核（Level 1），但無接手者二次驗證機制
- Claim 認領時是最佳防護插入點（接手者決策時間點）
- 兩端都值得改，改動量小（~14 行）
## Solution

<!-- To be filled by executing agent -->

---

實作三處改動（PC-007 雙端防護）：

1. command_lifecycle_messages.py：新增 CHECKLIST_SCOPE_VERIFICATION 常數
   「[ ] 已獨立驗證 Ticket 描述的數量/範圍（描述是草稿，可能有遺漏）」

2. lifecycle.py _print_claim_checklist()：在 CHECKLIST_ERROR_PATTERNS 後加入新清單項目
   claim 每個 ticket 時都會看到此提醒

3. ticket_builder.py create_ticket_body()：Problem Analysis 模板加入 PC-007 引導注釋
   提示建立者記錄「已確認」vs「推測」的位置，並說明接手者應獨立重新驗證
