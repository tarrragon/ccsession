## Solution

### 重構方案總覽

本次重構完整解決 TD-012~016 五項技術債務：

#### 1. YAML 結構化驗證（TD-012）
- 新增 `_parse_yaml_safely()` 函式使用 PyYAML 進行安全的結構化解析
- 升級 `parse_hook_classification()` 為結構化 YAML 解析，同時保留文字模式作為 fallback
- 更新 `check_hook_configurations()` 新增 YAML 格式驗證和錯誤記錄

#### 2. gitignore 精確前綴匹配（TD-013）
- 重寫 `_has_gitignore_rule()` 使用精確前綴匹配取代模糊匹配
- 標準化規則（移除末尾斜線和 /*），實現精確的行對行比較
- 支援多種規則變體（coverage/, coverage, .claude/hook-logs 等）
- 新增 `test_onboard_gitignore_precise_matching()` 驗證

#### 3. 權限詳細記錄（TD-014）
- 新增 `PermissionInfo` dataclass 記錄讀/寫/執行各別權限
- 新增 `_check_file_permissions()` 檢查檔案權限，拆分為：
  - `_check_file_read_permission()`
  - `_check_file_write_permission()`
- `HookConfigurationCheckInfo` 新增 `yaml_permission_info` 和 `json_permission_info` 欄位
- `_print_hook_config_section()` 輸出權限資訊

#### 4. run_onboard() 拆分（TD-015）
- 拆分為 3 個專責函式（各 <= 30 行）：
  - `_run_detection_checks()` (21 行)：語言偵測和 Hook 分類
  - `_run_structure_checks()` (17 行)：目錄結構驗證和自動建立
  - `_run_final_checks()` (9 行)：README 和語言規範檢查
- 新增 `_build_onboard_result()` (7 行) 建立結果物件
- `run_onboard()` 簡化為 19 行

#### 5. _print_*_section() 共用模式提取（TD-016）
- 新增 3 個共用輔助函式：
  - `_print_section_header(title)`：輸出區段標題
  - `_print_status_line(status, message, indent)`：輸出狀態行
  - `_print_hint_line(hint, indent)`：輸出提示行
- 新增 3 個收集函式簡化主函式流程
- 簡化 9 個原有的 `_print_*_section()` 函式

#### 6. 追加改進：待辦項目收集拆分
- 拆分 `_collect_must_items()` 為 5 個專責函式以降低認知負擔
- 進一步拆分 `_collect_core_file_items()` 為 4 個細粒度函式
