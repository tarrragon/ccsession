# Execution Log

## Task Summary

建立 Bash 非零退出碼 PostToolUse Hook（CLI 失敗時提示查 --help）

---

## Problem Analysis

PC-005 錯誤模式：CLI 工具報錯時，PM 未查閱 `--help`，直接基於假設歸因並採取行動。
需要在系統層面自動提醒，防止跳過調查步驟。

---

## Solution

建立 `cli-failure-help-reminder-hook.py` PostToolUse Hook：

- **觸發條件**：Bash 工具 tool_response 含非空 stderr（非零退出指標）
- **排除條件**：
  - 已由 pre-fix-evaluation-hook 處理的命令（flutter test、dart analyze 等）
  - 預期有 stderr 的命令（grep、find、which 等）
  - git 資訊性 stderr（Already up to date 等）
- **輸出**：PC-005 調查 4 步驟提醒（不阻止執行）
- **訊息常數**：新增 `WorkflowMessages.CLI_FAILURE_HELP_REMINDER` 到 hook_messages.py

修改檔案：
1. `.claude/hooks/cli-failure-help-reminder-hook.py`（新建）
2. `.claude/hooks/lib/hook_messages.py`（新增訊息常數）
3. `.claude/settings.json`（PostToolUse/Bash 註冊）
