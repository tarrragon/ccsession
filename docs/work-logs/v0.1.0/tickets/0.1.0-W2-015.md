### 根本原因

違反 implementation-quality.md 1.3.2 規範（禁止魔法數字），影響可讀性和可維護性。當需要調整超時值或顯示限制時，必須搜尋多個位置修改。

---

## Solution

### 重構策略

遵循 implementation-quality.md 1.3 常數管理規範，在每個檔案頂部（import 之後、函式定義之前）建立常數區塊。

**package-version-sync-hook.py：**
```python
# Constants 區塊補充
HOOK_NAME = "package-version-sync-hook"
SHORT_OPERATION_TIMEOUT_SECONDS = 30  # For uv tool list, uninstall, cache clean
INSTALL_OPERATION_TIMEOUT_SECONDS = 120  # For uv tool install --reinstall
SEPARATOR_LINE_WIDTH = 60
```

**sync-claude-pull.py：**
```python
# Constants 區塊新增
GIT_CLONE_TIMEOUT_SECONDS = 120
GIT_HTTP_LOW_SPEED_LIMIT_BYTES = "1000"
GIT_HTTP_LOW_SPEED_TIME_SECONDS = "30"
MAX_CHANGED_FILES_DISPLAY = 3
```

### 實作細節

1. 在兩個檔案中新增常數區塊，遵循既有的常數定義模式
2. 更新所有魔法數字的引用位置，改用常數變數
3. 動態化超時訊息：`f"git clone 超時（{GIT_CLONE_TIMEOUT_SECONDS} 秒）"` 而非硬編碼 `"120 秒"`
4. 遵循 UPPER_SNAKE_CASE 命名規範，常數名稱描述性強
