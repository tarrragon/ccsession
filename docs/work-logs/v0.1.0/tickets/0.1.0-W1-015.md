# Execution Log

## Task Summary

補充 UC-009 WebSocket 架構預期和同時連線數定義（M4）

---

## Problem Analysis

W1-006 Go 視角審查發現 M4：UC-009 缺少 WebSocket 同時連線數的架構預期定義。
原文件的效能需求僅列出「同時連線數 >= 10」一行，未說明：
- 使用場景定位（本地工具 vs SaaS）
- 同時連線數的設計基準和典型使用範圍
- 每連線記憶體預算
- 連線管理策略的完整定義
- 廣播策略和背壓處理

---

## Solution

在 UC-009 的「效能需求」章節前新增「WebSocket 架構預期」章節，涵蓋：

1. **使用場景定位**：明確本地監控工具的定位，典型 1-2 連線
2. **同時連線數預期**：典型 1-2、多裝置 3-5、設計上限 10
3. **每連線記憶體預算**：約 29 KB/連線，10 連線合計約 290 KB
4. **連線管理策略**：心跳 30s、超時 90s、無閒置斷線、連線數上限檢查
5. **廣播策略**：扇出模式，單 Client 失敗不影響其他，佇列滿時丟棄最舊訊息

同時更新原有效能需求表格，增加說明欄位和新增指標（單連線記憶體、廣播延遲）。

注意保留了 W1-012 補充的 Phase 3+ 搜尋協議預留內容。
