## Solution

### 實作方式

使用 `is_subagent_environment(input_data)` 檢測 input_data 中的 `agent_id` 欄位，判斷是否為 subagent 環境。若在 subagent 中執行，則靜默退出（exit 0），不輸出任何提醒。

### 修改詳情

#### 1. askuserquestion-reminder-hook.py
- 導入 `is_subagent_environment` 函式
- 在 main() 開頭讀取 input_data 後，立即檢測 subagent 環境
- subagent 中返回預設 allow 輸出並退出，避免輸出無效的 AskUserQuestion 提醒

#### 2. process-skip-guard-hook.py
- 導入 `is_subagent_environment` 函式
- 在讀取 input_data 後，緊接著檢測 subagent 環境
- subagent 中直接返回 EXIT_SUCCESS，跳過所有流程省略偵測

#### 3. prompt-submit-hook.py
- 導入 `is_subagent_environment` 函式
- 修改 `read_prompt_from_stdin()` 返回 `(prompt, input_data)` tuple，避免重複讀取 stdin
- 在 main() 中讀取 prompt 和 input_data 後，立即檢測 subagent 環境
- subagent 中直接返回 0，跳過所有工作流程檢查和 SKILL 提示

### 設計理由

- **早期退出**：在 main() 函式開頭檢測，避免執行不必要的邏輯
- **統一模式**：參考 commit-handoff-hook.py 的既有實作方式
- **日誌記錄**：記錄 agent_id 便於除錯
- **安全預設**：subagent 環境下 exit(0)，不對用戶端輸出任何提醒
