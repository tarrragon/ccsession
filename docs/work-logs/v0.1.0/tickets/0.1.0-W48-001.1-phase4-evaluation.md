# Phase 4b 重構評估報告

## 重構動機與目標

**當前問題**：
- `main()` 函式 128 行，包含 9 個邏輯步驟，認知負擔指數 16（超過標準 < 10）
- `scan_prohibited_items()` 63 行，三層嵌套迴圈，認知負擔 12
- `is_in_inline_code()` 32 行，反引號查找邏輯冗長

**重構目標**：
1. 拆分 `main()` 為多個單一職責函式，降低認知負擔至 < 10
2. 改進 `scan_prohibited_items()` 複雜度
3. 提升程式碼可維護性和可測試性

**解決方法**：
- 提取 4 個輔助函式：`_validate_input()`、`_load_file_content()`、`_output_error()`、`_output_success()`
- 簡化 `main()` 的流程邏輯，將特定功能委託給輔助函式
- 保留原有行為，不修改功能
