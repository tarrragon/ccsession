## Solution

### 主要實作修改

#### 1. `hook_verifier.py` 新增完整性檢查函式

新增 `HookCompletenessResult` dataclass 和 `check_hook_completeness()` 函式：
- 掃描 `.claude/hooks/` 目錄找到所有 .py 檔案
- 從 `hook-exclude-list.json` 讀取排除清單（支援精確檔名和萬用字元模式）
- 從 `settings.json` 提取已登記的 Hook 檔名（掃描所有事件類型）
- 比對兩者並回傳未登記的 Hook 清單

輔助函式：
- `_load_json_file()` - 安全載入 JSON 檔案
- `_get_exclude_patterns()` - 從 exclude list 提取排除規則
- `_should_exclude_file()` - 判斷檔案是否應被排除
- `_scan_hooks_directory()` - 掃描 Hook 目錄
- `_extract_registered_hooks()` - 從 settings.json 提取已登記 Hook

#### 2. `onboard.py` 新增 Hook 完整性驗證步驟

修改 `run_onboard()` 函式加入 Step 6 Hook 完整性檢查。
修改 `_print_onboard_result()` 新增 Hook completeness 區段顯示。
新增 `_print_hook_completeness_section()` 輸出 Hook 狀態。
修改 `_collect_todo_items()` 當有未登記 Hook 時加入待辦項目。

#### 3. `messages.py` 新增訊息常數

新增 `OnboardMessages` 類別中的 Hook 完整性相關訊息：
- `HOOK_COMPLETENESS_SECTION` - 區段標題
- `HOOK_COMPLETENESS_OK` / `_TODO` - 狀態訊息
- `HOOK_REGISTERED_COUNT` / `_UNREGISTERED_COUNT` / `_EXCLUDED_COUNT` - 數量統計
- `HOOK_UNREGISTERED_LIST` / `_MORE` - 未登記清單
- `HOOK_COMPLETENESS_HINT` - 處理建議

#### 4. `__init__.py` 更新 export

新增 `HookCompletenessResult` 和 `check_hook_completeness` 到 import 和 `__all__`。
