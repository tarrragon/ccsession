## Solution

### 修改內容

1. **commit-handoff-hook.py - main() 函式新增 agent_id 檢查**
   - 位置：第 305-310 行（驗證工具類型之後）
   - 新增邏輯：讀取 input_data 後，若 agent_id 欄位存在（表示在 subagent 中執行）
   - 行為：直接輸出 DEFAULT_OUTPUT，跳過 AskUserQuestion 提醒
   - 日誌：記錄 "偵測到 subagent 環境（agent_id=...），跳過 AskUserQuestion 提醒"

2. **test_commit_handoff_hook.py - 新增兩個測試案例**
   - test_subagent_environment_detection_skips_reminder()：驗證 subagent 環境（含 agent_id）輸出預設輸出
   - test_main_environment_outputs_reminder()：驗證主線程環境（無 agent_id）輸出 AskUserQuestion

### 設計理由

Phase 1-3 代理人的 commit 應該以最少 context overhead 執行，不需要 PM 介入決策。通過檢測 PostToolUse Hook 中的 agent_id 欄位（這個欄位僅在 subagent 中由 Claude Code 系統注入），可以區分：

- subagent 執行：無需 AskUserQuestion 提醒（代理人自治繼續執行）
- 主線程執行：需要 AskUserQuestion 提醒（PM 做決策）

這樣既滿足了代理人的自治要求，也保留了主線程的完整決策流程。
