# Execution Log

## Task Summary

升級 project-init 的 pyproject.toml 解析為 tomllib（消除文字解析脆弱性）

### 多視角分析結論（方案 C）

原始計畫「提取為 hook_utils 共用 utility」不可行：project-init 是獨立 Python 套件，
uv tool install 後位於 ~/.local/share/uv/tools/，無法 import .claude/hooks/ 下的模組。

推薦方案：各自保留，統一使用 tomllib。具體做法：
- Hook 側（已完成）：package-version-sync-hook.py 已用 tomllib
- project-init 側（本 Ticket）：升級 3 個 _extract_* 函式為 tomllib 結構化解析

現有文字解析的 3 個缺陷：
1. IMP-002：魔法假設（只支援雙引號 + 特定空格格式）
2. IMP-003：狀態管理 bug（_extract_cli_name 的 in_scripts_section 未正確重置）
3. IMP-011：跨節點污染（name 欄位可能誤命中非 [project] 節點）

---

## Problem Analysis

package_manager.py 的 3 個 `_extract_*` 函式（`_extract_version_from_pyproject`、`_extract_package_name_from_pyproject`、`_extract_cli_name_from_pyproject`）使用文字逐行搜尋解析 pyproject.toml，存在以下缺陷：

1. **IMP-002（魔法假設）**：只支援特定格式
   - 假設雙引號 `version = "..."` 格式
   - 假設特定空格布局
   - 不支援單引號或三引號等 TOML 合法格式
   - 無法應對不同的格式變體

2. **IMP-003（狀態管理 bug）**：`_extract_cli_name_from_pyproject` 中的 `in_scripts_section` 布林值管理有問題
   - 狀態轉換邏輯複雜，容易出錯
   - 沒有妥善的邊界條件處理

3. **IMP-011（跨節點污染）**：逐行搜尋會誤命中非 [project] 節點的欄位
   - `name` 欄位可能存在多個位置（[build-system]、[project.optional-dependencies] 等）
   - `version` 欄位也可能被誤命中
   - 無法保證只取 [project] 節點的欄位

**測試覆蓋**：目前無任何單元測試覆蓋這些函式

---

## Solution

### 整體策略
升級為 tomllib 結構化解析，消除所有文字解析脆弱性。Python 3.14 標準庫內建 tomllib，無需新增外部依賴。

### 實作細節

1. **新增 `_parse_pyproject` 共用函式**
   ```python
   def _parse_pyproject(pyproject_path: Path) -> Optional[dict]:
       """使用 tomllib 解析 pyproject.toml 檔案（含快取）"""
       # 快取機制避免重複解析同一檔案
   ```
   - 使用 `tomllib.load()` 讀取 TOML 檔案
   - 快取已解析結果，避免同一檔案重複解析
   - 異常處理：解析失敗回傳 None

2. **重寫 `_extract_version_from_pyproject`**
   - 改用 `data["project"]["version"]` 結構化存取
   - 保持 Optional[str] 回傳型別
   - 保留異常防禦設計

3. **重寫 `_extract_package_name_from_pyproject`**
   - 改用 `data["project"]["name"]` 結構化存取
   - 消除跨節點污染（IMP-011）

4. **重寫 `_extract_cli_name_from_pyproject`**
   - 改用 `data["project"]["scripts"]` 取得指令字典
   - 使用 `next(iter(scripts.keys()))` 取第一個 key
   - 消除複雜的狀態管理邏輯（IMP-003）

### pyproject.toml 更新
**不需要更新**：`requires-python = ">=3.14"` 已保證 `tomllib` 在標準庫中可用，無需新增 `tomli` 條件依賴

### 改進對應

| 缺陷 | 舊實作 | 新實作 | 狀態 |
|-----|-------|--------|------|
| IMP-002 | 文字搜尋+魔法假設 | TOML 結構化解析 | 已消除 |
| IMP-003 | 狀態變數管理 | 直接字典提取 | 已消除 |
| IMP-011 | 逐行搜尋誤命中 | [project] 節點隔離 | 已消除 |
