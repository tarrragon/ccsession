## Solution

### 實施步驟

#### Step 1: 新增共用函式 `is_subagent_environment()`

在 `.claude/hooks/hook_utils/hook_io.py` 末尾新增函式，用於檢測是否為 subagent 環境：

```python
def is_subagent_environment(input_data: "dict | None") -> bool:
    """檢查是否為 subagent 環境

    subagent 執行時，Hook 會在 input_data 中包含 agent_id 欄位。
    此函式用於 Hook 的早期跳過邏輯，避免在 subagent 中輸出 AskUserQuestion 提醒。

    Returns:
        bool: 如果 input_data 包含 agent_id（非空），返回 True；否則返回 False
    """
    if input_data is None:
        return False
    return bool(input_data.get("agent_id"))
```

在 `hook_utils/__init__.py` 中新增導出。

#### Step 2: 修改 4 個 Hook，加入 subagent 環境檢測

在以下 4 個 Hook 的主處理邏輯開始前加入檢測邏輯，使用 `is_subagent_environment()` 共用函式：

1. **acceptance-gate-hook.py**
   - 在 main() 函式讀取 input_data 後，_parse_and_validate_input() 前，加入檢測
   - 若為 subagent 環境，直接返回 EXIT_SUCCESS，不輸出任何 AskUserQuestion 提醒

2. **phase-completion-gate-hook.py**
   - 在 main() 函式讀取 input_data 後，validate_hook_input() 前，加入檢測
   - 若為 subagent 環境，輸出空的 PostToolUse 結果並返回 EXIT_SUCCESS

3. **parallel-suggestion-hook.py**
   - 在 main() 函式讀取 input_data 後，validate_input() 前，加入檢測
   - 若為 subagent 環境，輸出空的 UserPromptSubmit 結果並返回 EXIT_SUCCESS

4. **post-ticket-complete-checkpoint-hook.py**
   - 在 main() 函式驗證 tool_name 前，加入檢測
   - 若為 subagent 環境，輸出空的 PostToolUse 結果並返回 EXIT_SUCCESS

#### Step 3: 同步更新 commit-handoff-hook.py

將現有的 inline subagent 檢測（約第 314 行）改為使用 `is_subagent_environment()` 共用函式。
