# Execution Log

## Task Summary

建立 hook 偵測輸出語言切換（非繁體中文/英文的文字出現時發出警告），防止韓文/日文/簡體中文整段輸出

---

## Problem Analysis

PC-008 問題根因：
- Model 在高 context 下，於撰寫 5W1H 分析時，偶爾觸發韓語/日文模板（可能與國際框架聯想有關）
- 現有 `language-constraints.md` 規則的 Hook 只檢查禁用的簡體詞彙，不偵測整段語言切換
- 需要程式化防護補足缺口

技術決策（已由 PM + SA 確認）：
- **Hook 類型**：UserPromptSubmit（非 Stop）
- **觸發時機**：用戶提交新訊息時，執行 Hook 讀取前一輪 Claude 回應
- **偵測目標**：
  - 韓文（Unicode AC00-D7AF）
  - 日文假名（平假名 3040-309F、片假名 30A0-30FF）
- **動作**：輸出警告到 stderr（非阻止，只警告），使用 `[LANG GUARD]` 前綴
- **靜默條件**：
  - 無輸入時靜默通過
  - 無前一輪 assistant 回應時靜默通過
  - 讀取 transcript 失敗時靜默通過

---

## Solution

Phase 1 設計 Hook 規格：
- UserPromptSubmit 事件，讀取 Hook 輸入 JSON 中的 transcript 陣列
- 從 transcript 反向掃描，找最後一個 role=assistant 的訊息
- 遍歷訊息內容，檢查每個字元的 Unicode 碼點
- 若符合韓文或日文範圍，輸出警告到 stderr（使用 hook_utils 日誌系統）
- 異常時靜默通過，不阻止用戶操作

實作完成：
- language-guard-hook.py：使用 hook_utils setup_hook_logging() 和 run_hook_safely()
- 符合 Python 3.9 相容性（禁用 PEP 604 等新語法）
- 無外部依賴（只用標準庫）
- 常數集中管理（Unicode 範圍、警告前綴等）

配置更新：
- `.claude/settings.json` UserPromptSubmit 事件新增 language-guard-hook.py 配置
- `.claude/hooks/hook-registry.json` 新增 language-guard-hook 記錄
