## Solution

### 重構策略

原始 `parse_ticket_frontmatter` 函式存在高度複雜的 while 迴圈（巢狀 7 層、23 分支），導致認知負擔指數 ~52。重構策略為：

1. **提取文件讀取邏輯**：建立 `_read_content()` 函式，統一處理 Path 和 str 輸入，降低主函式複雜度
2. **提取行檢驗邏輯**：保留 `_skip_empty_or_comment()` 函式（判斷是否跳過空行/註解）
3. **提取嵌套行處理**：保留 `_parse_nested_line()` 函式（處理縮排鍵值對和多行文字）
4. **提取頂層鍵值對**：保留 `_parse_top_level_pair()` 函式（解析簡單 key:value）
5. **新增 YAML 解析協調函式**：建立 `_parse_yaml_lines()` 函式，專責逐行解析邏輯

### 實作成果

**parse_ticket_frontmatter 主函式**（行數：17 行，巢狀深度：3 層）：
- 使用 guard clauses（提前返回）移除不必要的巢狀
- 將 YAML 解析邏輯委派給 `_parse_yaml_lines()`
- 保持原有的 try-except 例外處理框架

**五個私有輔助函式**：
1. `_read_content()` - 統一文件讀取
2. `_skip_empty_or_comment()` - 行過濾
3. `_parse_nested_line()` - 嵌套行處理
4. `_parse_yaml_lines()` - YAML 逐行協調
5. `_parse_top_level_pair()` - 頂層鍵值對解析
