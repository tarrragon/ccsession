## Solution

### 實作步驟

1. **移除直接 import**
   - 在 test_hook_utils.py 第 22-34 行，移除 `_log_exception` 的直接 import
   - 在 fallback 虛擬函式清單中也移除 `_log_exception`

2. **重寫測試類別**
   - 將原有 `TestLogException` 類別改為 `TestExceptionHandling`
   - 從 5 個測試擴展為 7 個測試，覆蓋更全面的異常情況
   - 所有測試都透過 `run_hook_safely()` 觸發異常路徑

3. **新測試覆蓋範圍**
   - test_exception_stderr_normal_path: 異常時 stderr 正常輸出異常通知
   - test_exception_logged_to_file: 異常記錄到日誌檔案
   - test_exception_stderr_and_file_both_written: stderr 和日誌同時輸出（雙通道）
   - test_exception_critical_level_logging: CRITICAL 級別記錄
   - test_exception_with_hook_name: 包含正確的 hook 名稱
   - test_exception_stderr_survives_logging_failure: 日誌失敗時 stderr 備援路徑
   - test_exception_traceback_format: 完整的 traceback 格式
