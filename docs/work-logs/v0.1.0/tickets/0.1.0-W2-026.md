# Execution Log

## Task Summary

修復 W2-012 重構後測試 mock 路徑失效（15 個回歸測試失敗）

---

## Problem Analysis

W2-012 重構將 9 個命令模組改為呼叫 `ticket_ops.load_and_validate_ticket()`，但測試仍在 patch 舊路徑 `ticket_system.lib.ticket_loader.load_ticket`。

**根本原因**：Python mock 遵循「patch where it's used」原則。當 `ticket_ops.py` 內部執行 `from ticket_system.lib.ticket_loader import load_ticket` 時，該模組建立了獨立引用。若 patch `ticket_loader.load_ticket`，無法攔截 `ticket_ops` 內部的呼叫，導致 mock 無效。

**受影響的測試**（15 個回歸失敗）：
- test_track_fields.py: 7 個失敗（patch 路徑錯誤）
- test_track_acceptance.py: 3 個失敗（混合使用 load_ticket 和 load_and_validate_ticket）
- test_track_lifecycle.py: 1 個失敗（patch 路徑錯誤）
- test_handoff_context_refresh.py: 3 個失敗（patch 路徑錯誤）
- test_track_query.py: 1 個失敗（patch 路徑錯誤）

---

## Solution

統一修復策略：對於所有使用 `load_and_validate_ticket` 的命令模組，將測試中的 mock patch 路徑改為 `ticket_system.lib.ticket_ops.load_ticket`（即在 `load_and_validate_ticket` 內部被呼叫的位置）。

**修復清單**：
1. test_track_fields.py: 更新所有 patch 為 `ticket_system.lib.ticket_ops.load_ticket`
2. test_track_acceptance.py:
   - execute_check_acceptance 測試改為 patch `load_and_validate_ticket` 並返回 (ticket, error) 元組
   - execute_append_log 測試保持原有路徑（未重構），補充 save_ticket mock
3. test_track_lifecycle.py: 更新 release 測試的 load patch + save patch
4. test_handoff_context_refresh.py: 更新 handoff 測試的 load patch
5. test_track_query.py: 更新 query/summary/list 測試的 load patch
6. 修復異常處理：changed `mock_load.side_effect = FileNotFoundError()` → `mock_load.return_value = None`
