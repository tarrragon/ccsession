## 1. 需求分析摘要

### 搬移目標
將 `get_project_root()` 相關的**基礎設施邏輯**從 `hook_logging.py` 搬移至新的 `hook_base.py`：

| 項目 | 來源 | 目標 |
|------|------|------|
| 公開函式 | `hook_logging.get_project_root()` | `hook_base.get_project_root()` |
| 私有函式 | `hook_logging._find_project_root()` | `hook_base._find_project_root()` |
| 常數 | `hook_logging.ENV_PROJECT_DIR`, `CLAUDE_MD_SEARCH_DEPTH` | `hook_base.ENV_PROJECT_DIR`, `CLAUDE_MD_SEARCH_DEPTH` |

### 架構目標
建立三層清晰的責任劃分：
```
hook_base.py (基礎設施層 - 路徑查詢)
    ↑ re-export
hook_logging.py (日誌層 - 不含路徑查詢邏輯)
hook_io.py (I/O 層 - 透過 hook_base 查詢路徑)
```

### 向後相容約束
- 54+ 個現有 Hook 無需修改 import 語句
- `hook_logging.get_project_root()` 仍可用（re-export）
- `hook_io.py` 第 273 行延遲導入需要特別處理

## 3. 虛擬碼與流程圖

### 3.1 搬移邏輯虛擬碼

#### hook_base.py（新建）
```pseudocode
模組：hook_base.py

常數定義：
  ENV_PROJECT_DIR ← "CLAUDE_PROJECT_DIR"
  CLAUDE_MD_SEARCH_DEPTH ← 5

函式 _find_project_root() -> Path：
  步驟 1：檢查環境變數
    IF os.getenv(ENV_PROJECT_DIR) 存在:
      RETURN Path(環境變數值)

  步驟 2：向上搜尋 CLAUDE.md（最多 5 層）
    current_dir ← Path.cwd()
    FOR i FROM 1 TO CLAUDE_MD_SEARCH_DEPTH:
      IF (current_dir / "CLAUDE.md").exists():
        RETURN current_dir

      parent ← current_dir.parent
      IF parent == current_dir:  // 已達根目錄
        BREAK

      current_dir ← parent

  步驟 3：Fallback
    RETURN Path.cwd()

函式 get_project_root() -> Path：
  RETURN _find_project_root()
```

## 6. 語言特定實作注意事項

### Python 考量

**包裝層級**（pip/wheel）：
- Hook 系統位於 `.claude/hooks/`（非標準 package）
- 使用相對導入（`.hook_base`）
- `sys.path` 已由上層 CLI 配置

**測試環境**：
- 使用 `pytest` 框架
- 執行命令：`(cd /Users/mac-eric/project/ccsession && uv run pytest .claude/hooks/tests/test_hook_base_refactor.py -v)`
- 需要 tmpdir 隔離檔案系統

**Import 順序**：
- 確保 hook_base.py 無任何 hook_utils 子模組 import
- 避免在模組層級執行 I/O（如 Path.cwd()），只在函式內執行

**文件編碼**：
- 檔案頭：`#!/usr/bin/env python3` + `# -*- coding: utf-8 -*-`
- 字串：使用 `.format()` 或 f-string，支援中文

**效能最佳化**：
- hook_io.py 第 273 行延遲導入是為了避免圓形 import，不要變成頂部導入
- _find_project_root() 在 hook_logging.setup_hook_logging() 執行時呼叫，效能可接受

**可能的技術挑戰**：
1. **循環依賴** — hook_base 必須完全獨立
2. **Re-export 斷鏈** — `__init__.py` 指向錯誤来源
3. **延遲導入破壞** — hook_io 第 273 行改 import 後仍需功能正確
4. **常數查詢** — 其他程式碼可能直接 import 常數（grep 檢查）
