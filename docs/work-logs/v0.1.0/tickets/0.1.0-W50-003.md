### sync-claude-pull.py 重構 (⚠️ 受限)

**狀態**: 無法直接編輯（`.claude/scripts/` 不在 main-thread 編輯白名單中）

**根本原因**：
- main-thread-edit-restriction-hook.py 只允許編輯 `.claude/` 下的預定義子目錄
- ALLOWED_PATTERNS 中列出：hooks/, skills/, agents/, rules/, methodologies/ 等
- 不包含 `.claude/scripts/`（它是 Hook 系統的一部分，但位置不被認可）

**識別的改進方案（待 PM 批准後執行）：**

1. **Cognitive Load 拆分方案**
   - main() 目前 32 行（已接近限制，需拆分）
   - 建議拆分為：
     * `_validate_environment()` (10 行) - 驗證專案根目錄和本地狀態
     * `_clone_and_backup()` (12 行) - 克隆遠端 repo + 備份
     * `_complete_sync()` (8 行) - 完成同步和輸出訊息
   - 拆分後 main() 預期 < 25 行

2. **魔法數字提取方案**
   - GIT_CLONE_TIMEOUT_SECONDS = 120 ✓ (已為常數)
   - GIT_HTTP_LOW_SPEED_LIMIT_BYTES = "1000" ✓ (已為常數)
   - GIT_HTTP_LOW_SPEED_TIME_SECONDS = "30" ✓ (已為常數)
   - MAX_CHANGED_FILES_DISPLAY = 3 ✓ (已為常數)
   - json.dump(..., indent=2) → 應提取為常數 JSON_INDENT = 2

3. **集合設計優化方案**

   **目前代碼：**
   ```python
   _REMOTE_REPO_ONLY = frozenset({".git", "project-templates"})
   _LOCAL_ONLY_PATHS = frozenset({...8 項目...})
   SKIP_DURING_SYNC = _REMOTE_REPO_ONLY | _LOCAL_ONLY_PATHS
   ```

   **建議優化：**
   ```python
   # 明確語義：遠端專有
   REMOTE_ONLY = frozenset({".git", "project-templates"})

   # 明確語義：本地專有
   LOCAL_ONLY = frozenset({
       "hook-logs",
       "handoff",
       "PM_INTERVENTION_REQUIRED",
       "ARCHITECTURE_REVIEW_REQUIRED",
       "pm-status.json",
       "__pycache__",
       ".pytest_cache",
       ".venv",
   })

   # 同步時跳過（邏輯清晰）
   SKIP_DURING_SYNC = REMOTE_ONLY | LOCAL_ONLY
   ```
   - 好處：名稱更清晰，避免下劃線造成的混淆
   - 語義更強：REMOTE_ONLY vs LOCAL_ONLY 表達意圖清楚

4. **Docstring 補充方案**
   - `collect_remote_files()` (line 219) - 缺 docstring
   - `_walk()` 內部函式 (line 242) - 缺 docstring
   - 其餘函式都有完整 docstring
