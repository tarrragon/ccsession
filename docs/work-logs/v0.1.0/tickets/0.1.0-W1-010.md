# Execution Log

## Task Summary

補充 UC-002 分頁連續性和自動捲動互動規則（M5+M7+L3）

---

## Problem Analysis

W1-006 二次審查發現 UC-002 有 3 項缺漏：

- **M5**（中優先級）：分頁載入後的連續性保證未定義 — 跨分頁載入時如何確保事件不重複、不遺漏、順序一致，以及分頁結束的信號均未說明。
- **M7**（中優先級）：自動捲動與分頁載入的互動優先級未明確 — 當 active session 同時觸發自動捲動和分頁載入時，缺乏優先級規則定義。
- **L3**（低優先級）：時間戳相同事件的次級排序規則未定義 — 多筆事件具有相同 timestamp 時的顯示順序未明確。

---

## Solution

### M5: 分頁連續性保證

在 A1（大型 Session）流程末尾新增「分頁連續性保證」子章節，定義 4 條規則：
- 無重複：`before` 為排他比較，確保分頁間不出現重複事件
- 無遺漏：同一 timestamp 下的多筆事件必須在同一批次全部回傳
- 順序一致：每次分頁回傳按 timestamp 升序排列
- 結束信號：回傳事件數量 < limit 時表示歷史記錄已載完

### M7: 自動捲動與分頁互動優先級

在 A2（即時更新）流程末尾新增「自動捲動與分頁載入的互動優先級」子章節，以表格定義 4 種情境的行為規則，核心原則為：使用者明確操作優先於系統自動行為，進行中的使用者請求不應被系統行為中斷。

### L3: 時間戳相同事件次級排序規則

在「訊息類型渲染規格」之前新增「事件排序規則」章節，定義：
- 主要排序：timestamp 升序
- 次級排序：JSONL 檔案行號順序（line index），並補充語義配對約束（tool_use 在 tool_result 之前）
- 實作規則：Backend 賦予單調遞增序號，以 (timestamp, line_index) 複合鍵排序
