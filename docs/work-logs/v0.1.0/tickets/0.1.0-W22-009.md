**根本問題分析**

ticket-lifecycle.md 的描述存在兩個錯誤：

1. **時序錯誤**：文件說 Scene #17 在 `ticket track complete` 「執行時」由 CLI 處理，但實際上是 acceptance-gate-hook（PreToolUse）在 complete **前**觸發，由 hook 輸出提醒，complete 仍允許執行（exit 0）。

2. **死鎖場景**：當 PM 看到 #17 提醒後，可能誤以為「必須先處理 #17 才能執行 complete」，導致：
   - 先處理 error-pattern（建立 fix ticket）
   - 再執行 complete → hook 再次觸發 #17（error-pattern 檔案未消失）
   - 無限循環，complete 永遠無法完成

**根本原因**：error-pattern 檔案在 #17 處理後不會自動移除，hook 的判斷基準是 mtime > ticket_created，所以每次執行 complete 都會觸發 #17。
