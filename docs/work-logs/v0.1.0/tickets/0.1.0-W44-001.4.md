## Solution

### 設計邊界定義

已建立 `portable-design-boundary.md`，分類 TDD 流程為兩層：

**Layer 1（通用/可攜式）40+ 功能點**：
- Phase 0-4 階段定義、轉換條件、目標產出
- SOLID 檢查清單、功能拆分策略、複雜度評估
- BDD/GWT 測試設計方法、測試金字塔
- 代碼品質基準、重構分析維度
- 技術債務管理、遷移任務豁免規則
- 通用 commit 格式規範

**Layer 2（框架特定）6 個整合點**：
1. Ticket 系統整合 - `/ticket` CLI、狀態管理
2. Agent 派發規則 - 具體角色定義和時機
3. Hook 系統觸發 - 檢查點自動化
4. 決策樹路由 - PM 決策邏輯
5. 並行派發框架 - 3b 拆分、Phase 4a/4c 多視角
6. Commit 管理 - 自治 vs PM 管理區別

### 邊界模糊功能點分析

| 功能點 | Layer 1（通用） | Layer 2（框架） |
|--------|---------------|---------------|
| Phase 0 SA 審查 | 架構審查原則 | 派發流程和路由 |
| 3b 拆分評估 | 拆分決策邏輯 | 執行時機和 Ticket 建立 |
| Commit 管理 | 規範格式 | 誰執行和何時執行 |
| Phase 4 豁免 | 判斷標準 | 豁免後的流程路由 |

### 可攜式約束定義

**Layer 1 禁止引用**（共 7 項）：
- `/ticket` CLI、具體 agent 名稱、`.claude/hooks/` 系統
- `decision-tree` 路由、`/parallel-evaluation` 工具
- 本專案路徑結構、Wave/Patch 概念

**Layer 1 允許引用**（共 6 項）：
- 軟體工程通用概念（SOLID、Clean Architecture）
- 標準方法論（BDD、TDD、OOAD）
- 通用工具（Git、測試框架、lint）
- 標準流程（代碼審查、版本控制）

### 後續子任務指導

- **.5 子任務**：提取 Layer 1 內容產出可攜式 TDD 標準（`portable-tdd-standard.md`）
- **.6 子任務**：設計 `/tdd` SKILL 架構，實現 Layer 1 + 可選 Layer 2 整合
