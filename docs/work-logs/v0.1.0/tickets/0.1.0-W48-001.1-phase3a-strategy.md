## 1. 實作策略設計（語言無關）

### 1.1 核心演算法設計

**目標**：實作一個 PostToolUse Write Hook，掃描 Layer 1 檔案中的 7 類禁止項，無誤報。

**策略分解**：

#### 步驟 A：檔案識別（早期篩選）

**目標**：快速判斷檔案是否為 Layer 1 規則檔。

**演算法**：
```
輸入：檔案路徑 file_path
輸出：bool - 是否為 Layer 1 檔案

1. 定義 Layer 1 路徑模式清單：
   LAYER1_PATTERNS = [
     ".claude/rules/core/",
     ".claude/rules/flows/",
     ".claude/rules/guides/",
     ".claude/rules/forbidden/",
     ".claude/skills/tdd/references/portable-design-boundary.md",
     ".claude/skills/tdd/references/portable-tdd-standard.md",
   ]

2. FOR EACH pattern IN LAYER1_PATTERNS:
     IF pattern IN file_path AND file_path.endswith(".md"):
       RETURN true
     END IF
   END FOR

3. RETURN false
```

**驗證**：層級識別需 100% 準確，避免掃描 Layer 2 檔案。

---

#### 步驟 B：上下文排除規則構建（防止誤報）

**目標**：定義和應用排除規則，讓合法上下文的內容不被誤判。

**演算法**（預處理階段）：
```
輸入：檔案內容 content（多行字串）
輸出：排除區間清單 exclusions[]（每個排除區間為 [start, end]）

排除規則優先級（先匹配先排除）：
1. HTML 註解（<!-- ... -->）
2. 程式碼區塊（```...```）
3. 行內程式碼（` ... `）
4. blockquote（> 開頭的行）

演算法：
1. PARSE content 按行
2. 跟蹤排除狀態機（state = NORMAL, IN_CODE_BLOCK, IN_HTML_COMMENT）
3. FOR EACH line:
     IF line.startswith("<!--") -> state = IN_HTML_COMMENT
     ELSE IF state == IN_HTML_COMMENT:
       IF line.contains("-->") -> state = NORMAL
       ELSE -> mark_excluded(line)
     END IF

     IF line.contains("```"):
       toggle state between IN_CODE_BLOCK and NORMAL
     END IF

     IF state == IN_CODE_BLOCK -> mark_excluded(line)
     END IF

     IF line.startswith(">") -> mark_excluded(line)  # blockquote
     END IF
   END FOR

4. RETURN exclusions[]（所有排除的行號或字符位置）
```

**關鍵設計決策**：
- 排除規則作為預處理步驟單獨執行，而非在每次正則匹配時重新判斷
- 多層排除時（如 blockquote 內有程式碼），應用最外層優先（HTML 註解 > 程式碼 > blockquote）

---

#### 步驟 C：禁止項掃描（逐行正則匹配）

**目標**：使用預定義的正則表達式掃描 7 大禁止項，並排除合法上下文。

**演算法**：
```
輸入：
  - content：檔案內容
  - exclusions[]：排除的行號清單（來自步驟 B）
  - prohibited_items：禁止項定義清單

輸出：violations[]（違規清單，每個違規含行號、列號、類型、內容）

1. PARSE content 按行，記錄行號

2. FOR EACH line_num, line IN content:
     IF is_excluded(line_num, exclusions[]):
       CONTINUE  # 跳過此行
     END IF

     FOR EACH prohibited_item IN prohibited_items:
       pattern = prohibited_item["pattern"]
       matches = regex_findall(pattern, line)

       FOR EACH match IN matches:
         column_num = match.start_position
         content_snippet = extract_context(match, line)

         violation = {
           "line": line_num,
           "column": column_num,
           "type": prohibited_item["description"],
           "content": content_snippet,
           "replacement": prohibited_item["replacement"]
         }

         violations.append(violation)
       END FOR
     END FOR
   END FOR

3. RETURN violations[]
```

**邊界處理**：
- 多字節字符（中文）的列號計算需用字符計數而非字節計數
- CRLF 和 LF 換行符混合時，正確判斷行號邊界

---

#### 步驟 D：輸出生成（格式化警告訊息）

**目標**：生成清晰、可操作的警告訊息。

**演算法**：
```
輸入：
  - violations[]：違規清單
  - file_path：檔案路徑

輸出：formatted_warnings（字串），以及 exit_code（0 表示非阻塊）

1. IF violations.is_empty():
     RETURN (無警告訊息, exit_code=0)
   END IF

2. output = ""

3. FOR EACH violation IN violations:
     output += "[WARNING] Layer 1/2 邊界驗證警告\n\n"
     output += "檔案: " + file_path + "\n"
     output += "位置: Line " + violation["line"] +
              ", Column " + violation["column"] + "\n"
     output += "禁止項: " + violation["type"] + "\n"
     output += "內容: " + violation["content"] + "\n"
     output += "建議: 改為「" + violation["replacement"][0] + "」\n"
     output += "相關規範: .claude/skills/tdd/references/portable-design-boundary.md\n\n"
   END FOR

4. RETURN (output, exit_code=0)
```

**設計原則**：
- 每個違規單獨輸出，便於快速定位
- 包含行號、列號、原因、替代用語，實現「一站式」修復

## 2. 資料結構選擇

| 資料結構 | 用途 | 考量 |
|---------|------|------|
| 字典列表（PROHIBITED_ITEMS） | 禁止項定義 | 易於迭代、擴充，支援 JSON 序列化 |
| 集合（LAYER1_PATTERNS） | Layer 1 檔案模式 | 快速查找，適合字符串匹配 |
| 字典（violations） | 違規記錄 | 結構化資訊，便於格式化輸出 |
| 集合（exclusions） | 排除的行號 | O(1) 查找性能 |

**性能考量**：
- 檔案識別：O(n) 其中 n=7（LAYER1_PATTERNS 數量），極快
- 排除規則構建：O(m) 其中 m=行數，單遍掃描
- 禁止項掃描：O(m × k) 其中 k=禁止項數量（7），線性時間複雜度
- **總時間複雜度**: O(m) 其中 m=檔案行數，可接受

## 5. 關鍵實作指引

**優先順序**：
1. **檔案識別** → 排除規則構建 → 禁止項掃描（三個核心演算法）

**測試目標**：
- T1-T7 單元測試（禁止項基本檢測）
- T8 排除規則測試（Blockquote / Code / Comment 組合）
- T9 檔案識別測試（Layer 1/2 區分）

### 第二階段整合目標

**集成測試**：
- I1：Hook 完整執行流程（stdin → stdout）
- I2：實際 Layer 1 檔案掃描（23 個檔案，誤報率 < 5%）
- I3：警告訊息格式驗證

**調整策略**：
- 若誤報率 > 5%，改進排除規則或正則表達式
- 若測試失敗，按照失敗類型回溯至對應演算法檢查

## 6. 認知負擔評估

### 複雜度指數計算

| 維度 | 評估 | 數值 |
|------|------|------|
| 變數狀態數 | 4 個主要狀態（NORMAL, IN_CODE, IN_HTML, IN_BLOCKQUOTE） | 4 |
| 分支數 | 檔案識別 + 排除判斷 + 7 個禁止項 + 輸出生成 | 12 |
| 巢狀深度 | 行 → 禁止項 → 排除規則 | 3 |
| 依賴數 | 排除規則 → 正則匹配 → 輸出 | 3 |
| **總計** | | **22** |

### 評估結論

**認知負擔指數 = 22**（> 10，需要拆分）

**拆分策略**：
- **排除規則構建** → 獨立函式 `extract_exclusions()`
- **禁止項掃描** → 獨立函式 `scan_violations()`
- **輸出生成** → 獨立函式 `format_warnings()`
- **主函式** → 協調這三個函式

**拆分後每個模組的認知負擔**：
- `is_layer1_file()`: 2（快速查找）
- `extract_exclusions()`: 8（狀態機，3 層深）
- `scan_violations()`: 6（正則匹配，2 層深）
- `format_warnings()`: 4（字符串格式化）
- `main()`: 5（協調層）

**結論**：通過函式拆分，每個模組的認知負擔都在可管理範圍（< 10）。

| 排除規則不支援嵌套（blockquote 內的程式碼視作 blockquote） | 低（大多數 Layer 1 檔案無複雜嵌套） | Phase 4b 支援完整嵌套優先級 |
| 列號計算用簡單計數（可能在多字節字符時偏差） | 低（告警訊息可接受 ±2 列誤差） | Phase 4b 用 `len()` 和 `encode()` 精確計數 |
| 7 個禁止項的正則表達式手工維護 | 中（易於疏漏） | Phase 4a 評估是否提取為 YAML 或 JSON 配置 |
| 沒有動態載入配置（禁止項寫死在程式碼中） | 中（新增禁止項需修改程式碼） | Phase 4a 設計可載入的配置系統 |

### 已知限制

| 限制 | 描述 | 處理方式 |
|------|------|---------|
| **False Positive（誤報）** | 可能對合法的 Layer 1 自引用報警（如 decision-tree.md 中的「決策樹」） | Phase 2 測試設計中已規劃排除自引用；Phase 4a 根據誤報率決定是否改進 |

## 8. 語言特定實作注意事項

### Python 實作考量

**環境和執行**：
- Python 版本：>= 3.11（使用 type hints）
- 執行方式：Hook standalone（`dependencies = []`），不依賴外部套件
- 使用 `hook_utils` 共用函式進行日誌、I/O、Ticket 操作

**品質規則**（遵循 `.claude/rules/core/quality-python.md`）：

| 規則 | 實作細節 |
|------|---------|
| 型別標註 | 所有函式參數和返回值必須標註 |
| Docstring | 所有公開函式必須有說明文字（參數、返回、例外） |
| 常數管理 | `PROHIBITED_ITEMS` 和 `LAYER1_PATTERNS` 定義在模組頂部 |
| 訊息常數 | 不在程式碼中硬編碼警告訊息，使用 `.format()` |
| 無魔法數字 | 使用具名常數取代硬編碼數值 |

**異常處理**（遵循 `quality-baseline.md` Rule 4）：

```python
try:
    content = read_file(file_path)
except FileNotFoundError as e:
    logger.error(f"檔案不存在: {file_path}", exc_info=True)
    sys.stderr.write(f"ERROR: 無法讀取檔案\n")
    return EXIT_ERROR
except UnicodeDecodeError as e:
    logger.warning(f"編碼錯誤: {file_path}", exc_info=True)
    sys.stderr.write(f"WARNING: 編碼問題\n")
    return EXIT_SUCCESS  # 跳過此檔案，繼續執行
```

**正則表達式最佳實踐**：
- 使用 `re.compile()` 預編譯正則表達式，避免每次匹配時重新編譯
- 使用原始字串（`r'...'`）避免轉義問題
- 支援多字節字符（中文）的位置計算用 `len(string)` 而非 `len(string.encode())`
