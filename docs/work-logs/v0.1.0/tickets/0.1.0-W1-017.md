# Execution Log

## Task Summary

重新設計 套件追蹤機制（installed-packages.json → 自動掃描 + 實際狀態比對）

---

## Problem Analysis

### 根本問題

`installed-packages.json` 是 per-project 的靜態追蹤檔，但 `uv tool install` 安裝的是全域工具。設計層級不匹配導致：

1. **檔案不存在 bug**：沒有自動建立機制，首次 `uv tool install` 時不會產生此檔案
2. **跨專案不一致**：A 專案更新追蹤檔後，B 專案的追蹤檔仍是舊的
3. **sync-pull 失效**：拉取新程式碼後無法知道需要重新安裝哪些套件

### 正確設計方向

- **什麼需要安裝？** → 自動掃描 `.claude/skills/*/pyproject.toml`
- **什麼已經安裝？** → 查詢 `uv tool list`（實際狀態）
- **是否需要更新？** → 比對原始碼版本/hash vs 已安裝版本

### 影響範圍

- `sync-pull` 腳本：需改為自動掃描 + 實際狀態比對
- `package-version-sync-hook`：需移除對 `installed-packages.json` 的依賴
- `sync-pull.md` (SKILL)：需更新流程描述

---

## Solution

### 1. 現有設計問題總結

#### 問題 1：設計層級不匹配（最根本）

**現狀**：
- `installed-packages.json` 是 **per-project** 追蹤檔（位於 `.claude/` 目錄）
- `uv tool install` 安裝的工具是 **全域級** 資源（系統範圍）
- 兩者語義完全不匹配

**為什麼這是問題**：
```
Project A                    Project B
  |                            |
  v                            v
.claude/installed-packages.json  .claude/installed-packages.json
  |                            |
  +-----> ticket-system v1.0 (全域)
  +-----> mermaid-ascii v1.0 (全域)
           |
           +-----> Project B 的追蹤檔是舊版本
                   （A 項目更新不會影響 B）
```

**程式碼位置**：
- `.claude/hooks/package-version-sync-hook.py` 行 242：
  ```python
  tracking_file = project_root / ".claude" / "installed-packages.json"
  ```
  這裡假設檔案存在，但**沒有自動建立機制**

#### 問題 2：檔案不存在 bug（直接表現）

**現狀**：
- `installed-packages.json` 不存在於任何專案中
- Hook 第 248-251 行會自動返回空字典，導致無法追蹤任何套件
- 輸出：「Package Version Sync - No packages configured」

**具體檢驗**：
```python
# hooks/package-version-sync-hook.py, line 248-251
packages = tracking_data.get("packages", {})

if not packages:
    print("Package Version Sync - No packages configured")
    return 0
```

**缺陷**：
- 這不是「沒有套件」，而是「無法發現套件」
- Hook 該輸出「自動掃描 X 個套件」而不是「無套件」

#### 問題 3：sync-pull 流程失效

**流程位置**：`.claude/commands/sync-pull.md` 行 55-64

```
5. 重新安裝自訂套件（強制）
   - 讀取 `.claude/installed-packages.json` 取得追蹤的套件清單  ← 檔案不存在！
   - 對每個套件，檢查是否已安裝為全域 CLI tool
```

**現狀**：
- `.claude/scripts/sync-claude-pull.py` 不包含重新安裝邏輯
- 拉取後無法自動重新安裝 ticket-system 等已更新的套件

**影響**：
- 新增套件後 sync-pull，不會自動安裝
- 更新套件後 sync-pull，舊版本仍在使用

### 2. 新方案設計

#### 核心設計原則

```
Source of Truth（事實之源）
          ↓
  掃描 .claude/skills/*/pyproject.toml
  → 發現需要安裝哪些工具
          ↓
  查詢 uv tool list
  → 知道目前已安裝什麼
          ↓
  比對版本 / hash / git diff
  → 決定是否需要重新安裝
```

#### 架構設計圖

```
Session Start
    ↓
package-version-sync-hook
    ↓
[Step 1] 自動掃描
  ├─ 遍歷 .claude/skills/*/pyproject.toml
  ├─ 解析 [project] name 和 version
  └─ 建立「應該安裝的工具清單」
    │
    ├─ ticket-system 1.0.0
    ├─ mermaid-ascii 1.0.0
    └─ project-init 1.1.0
    ↓
[Step 2] 查詢實際狀態
  ├─ 執行 uv tool list
  ├─ 解析已安裝的工具及版本
  └─ 建立「實際安裝的工具清單」
    │
    ├─ ticket-system v1.0.0 ✓
    ├─ mermaid-ascii ✗ (未安裝)
    └─ project-init ✗ (未安裝)
    ↓
[Step 3] 比對 & 決策
  ├─ Version mismatch? → 重新安裝
  ├─ Not installed? → 安裝
  ├─ Up to date? → 跳過
  └─ 無需存儲 hash 或追蹤檔
    ↓
[Step 4] 執行安裝
  ├─ uv tool uninstall {name}
  ├─ uv cache clean {name}
  └─ uv tool install {path} --reinstall
```

#### 新方案對比（三個選項）

**選項 A：全域 hash 檔（`~/.claude/package-hashes.json`）**

| 面向 | 評估 |
|------|------|
| 實作難度 | 中 |
| 跨專案一致性 | 好（共用一個 hash 檔） |
| sync-pull 偵測 | 可用 git diff `.claude/skills/*/pyproject.toml` 補充 |
| 新專案首次運行 | hash 檔不存在，全量安裝一次，然後記錄 hash |
| 清理機制 | 難（舊套件 hash 何時刪除？） |
| 優缺點 | (+) 精確偵測源檔案變更<br>(-) 全域配置難維護<br>(-) 多人開發會有 hash 衝突 |

**選項 B：無 hash 存儲，純版本比對（推薦）**

| 面向 | 評估 |
|------|------|
| 實作難度 | 低 |
| 跨專案一致性 | 好（對比的是實際工具版本） |
| sync-pull 偵測 | 用 `git diff` 偵測 `pyproject.toml` 變更 |
| 新專案首次運行 | 掃描 → 版本號对比 → 需要则安装 |
| 清理機制 | 自動（工具不在任何 pyproject.toml 中 → 無需保留） |
| 優缺點 | (+) 簡單可靠<br>(+) 易於維護<br>(+) SessionStart 無副作用<br>(-) 無法偵測 Python 檔案非版本號修改（但 hook 可用 hash 補充） |

**選項 C：混合方案（SessionStart 版本比對 + sync-pull git diff）**

| 面向 | 評估 |
|------|------|
| 實作難度 | 中 |
| 跨專案一致性 | 好 |
| sync-pull 偵測 | 最精確（git diff 檔案 hash） |
| 新專案首次運行 | 最順暢 |
| 清理機制 | git diff 已自動過濾 |
| 優缺點 | (+) sync-pull 最精確<br>(+) SessionStart 輕量<br>(-) 雙軌邏輯略複雜 |

### 3. 推薦方案：選項 B（純版本比對）

**理由**：
1. **最簡單**：無需持久化 hash，減少 bug 風險
2. **最可靠**：對比的是「實際需要什麼 vs 實際安裝了什麼」
3. **最易維護**：跨專案無衝突，新人上手快
4. **夠精確**：對於 CLI 工具，版本號變更足以表示需要重新安裝

**設計細節**：

#### SessionStart Hook 流程（package-version-sync-hook.py）

```python
def main():
    project_root = get_project_root()

    # Step 1: 自動掃描 .claude/skills/*/pyproject.toml
    packages_to_install = scan_skill_packages(project_root)
    #  回傳: {
    #    "ticket-system": {"path": ".claude/skills/ticket", "version": "1.0.0"},
    #    "mermaid-ascii": {"path": ".claude/skills/mermaid-ascii", "version": "1.0.0"},
    #  }

    if not packages_to_install:
        print("No packages found in .claude/skills/*/pyproject.toml")
        return 0

    # Step 2: 查詢已安裝工具
    installed_tools = get_installed_uv_tools()
    #  回傳: {"ticket-system": "1.0.0"}

    # Step 3: 比對 & 安裝
    for pkg_name, pkg_info in packages_to_install.items():
        installed_version = installed_tools.get(pkg_name)
        desired_version = pkg_info["version"]

        if installed_version != desired_version:
            # 版本不符或未安裝 → 重新安裝
            print(f"{pkg_name}: {installed_version or 'not installed'} → {desired_version}")
            reinstall_uv_tool(pkg_name, project_root / pkg_info["path"])
```

#### sync-pull 流程（新增邏輯）

```bash
# 在 .claude/scripts/sync-claude-pull.py 中新增
# Step 4.5: 偵測需要重新安裝的套件

# 使用 git diff 快速掃描變更的 pyproject.toml
changed_toml_files = git diff --name-only origin/HEAD -- ".claude/skills/*/pyproject.toml"
```

### 5. 詳細實作偽代碼

#### 新增函式：`scan_skill_packages()`

```python
def scan_skill_packages(project_root: Path) -> Dict[str, Dict]:
    """自動掃描 .claude/skills/ 目錄下的所有 pyproject.toml。

    Returns:
        {
            "ticket-system": {
                "path": ".claude/skills/ticket",
                "version": "1.0.0"
            },
            ...
        }
    """
    packages = {}
    skills_dir = project_root / ".claude" / "skills"

    if not skills_dir.exists():
        return packages

    for skill_dir in skills_dir.iterdir():
        if not skill_dir.is_dir():
            continue

        pyproject = skill_dir / "pyproject.toml"
        if not pyproject.exists():
            continue

        try:
            import tomllib  # Python 3.11+
            with open(pyproject, "rb") as f:
                data = tomllib.load(f)

            project_info = data.get("project", {})
            pkg_name = project_info.get("name", "")
            version = project_info.get("version", "")

            if pkg_name and version:
                packages[pkg_name] = {
                    "path": str(skill_dir.relative_to(project_root)),
                    "version": version
                }
        except Exception as e:
            logger.warning(f"Failed to parse {pyproject}: {e}")

    return packages
```

#### 修改：`main()` 邏輯

```python
def main():
    logger = setup_hook_logging("package-version-sync-hook")
    project_root = get_project_root()

    # 移除舊邏輯：
    # tracking_file = project_root / ".claude" / "installed-packages.json"
    # tracking_data = load_tracking_file(tracking_file)

    # 新邏輯：自動掃描
    packages = scan_skill_packages(project_root)

    if not packages:
        print("Package Version Sync - No packages found in .claude/skills/")
        return 0

    print("=" * 60)
    print("Package Version Sync - Session Startup Check")
    print("=" * 60)

    installed_tools = get_installed_uv_tools()

    for package_name, package_info in packages.items():
        package_path = package_info.get("path", "")
        desired_version = package_info.get("version", "")
        installed_version = installed_tools.get(package_name)

        print(f"{package_name}: desired={desired_version}, installed={installed_version}")

        if installed_version != desired_version:
            package_full_path = project_root / package_path
            if reinstall_uv_tool(package_name, package_full_path):
                print(f"{package_name}: reinstalled")
            else:
                print(f"{package_name}: reinstall failed", file=sys.stderr)

    print("=" * 60)
    return 0
```

### 8. Hash 儲存位置評估結論

根據分析，**方案 B（無 hash 存儲）是最佳選擇**。原因：

| 評估維度 | 結論 |
|---------|------|
| **實作成本** | 方案 B 最低（僅需版本字串比對） |
| **可靠性** | 方案 B 足夠（版本號變更 = 需要重新安裝） |
| **維護負擔** | 方案 B 最輕（無額外檔案、無清理邏輯） |
| **跨專案安全** | 方案 B 無衝突（版本號由 pyproject.toml 提供，唯一事實源） |
| **sync-pull 偵測** | 方案 C 最精確，但方案 B + git diff 足以應對 |

**不選方案 A 的原因**：
- 全域檔案 `~/.claude/package-hashes.json` 在多人開發環境會有同步問題
- Hash 清理邏輯複雜（何時刪除舊工具的 hash？）
- 對於 CLI 工具，版本號已足夠精確

| 層級 | 測試對象 | 技術 | 覆蓋率目標 |
|------|--------|------|-----------|
| **單元測試** | 個別函式邏輯（掃描、解析、版本比對） | pytest | 100% code path |
| **整合測試** | Hook main() 流程 + uv tool list 互動 | pytest-mock | 所有場景組合 |
| **邊界測試** | 特殊環境（Python<3.11、無 git HEAD） | pytest fixture | 所有 fallback 路徑 |

#### 1.2 測試技術決策

| 決策 | 理由 |
|------|------|
| **pytest 框架** | Python 專案標準，便於 fixture 和 parametrize |
| **BDD 風格（Given-When-Then）** | 清楚表達測試意圖，易於維護 |
| **Mock/Patch uv 命令** | 避免實際安裝，加速測試執行 |
| **Fixture 管理 pyproject.toml** | 測試隔離，避免污染實際專案 |
| **Parametrize 版本組合** | 覆蓋多種版本比對場景 |

#### 1.3 Sociable Unit Tests 原則應用

根據項目規範，本測試採用「Sociable Unit Tests」：

| 原則 | 應用方式 |
|------|---------|
| **單元 = 模組** | 測試整個 Hook，不分離子函式 |
| **隔離外部依賴** | Mock `uv tool list` 輸出、檔案系統操作 |
| **測試行為非實作** | 測試「版本不符→重新安裝」，不測試字串拼接細節 |
| **實體物件優先** | 使用真實的 Path、TOML dict，只 Mock I/O 邊界 |

#### 2.1 `scan_skill_packages()` 單元測試

**目標**：驗證自動掃描 `.claude/skills/*/pyproject.toml` 的所有場景

##### Test Case 1.1：正常掃描多個 Skill

```
Given:
  - 存在 .claude/skills/ticket/pyproject.toml (name=ticket-system, version=1.0.0)
  - 存在 .claude/skills/mermaid-ascii/pyproject.toml (name=mermaid-ascii, version=0.5.0)
  - 存在 .claude/skills/project-init/pyproject.toml (name=project-init, version=1.1.0)

When:
  - 執行 scan_skill_packages(project_root)

Then:
  - 回傳 dict 包含 3 個套件
  - 各套件包含 name, path（相對路徑）, version 三個欄位
  - path 正確指向各 Skill 目錄（.claude/skills/ticket 等）
```

**預期結果**：
```python
{
    "ticket-system": {
        "path": ".claude/skills/ticket",
        "version": "1.0.0"
    },
    "mermaid-ascii": {
        "path": ".claude/skills/mermaid-ascii",
        "version": "0.5.0"
    },
    "project-init": {
        "path": ".claude/skills/project-init",
        "version": "1.1.0"
    }
}
```

**測試實作思路**：
- 使用 pytest fixture 建立臨時目錄結構（含 mock pyproject.toml 檔案）

#### 2.2 `get_installed_uv_tools()` 單元測試

**目標**：驗證解析 `uv tool list` 輸出的所有格式變體

```
When:
  - 執行 get_installed_uv_tools()

Then:
  - 回傳 {"ticket-system": "1.0.0", "mermaid-ascii": "0.5.0"}
  - 版本字串精確匹配（含 v 前綴判斷）
```

**預期輸出格式**（uv tool list 實際格式）：
```
Tool Name      Version  Executable Location
─────────────  ───────  ──────────────────────────
ticket-system  1.0.0    ~/.venv/bin/ticket
mermaid-ascii  0.5.0    ~/.venv/bin/mermaid
```

##### Test Case 2.2：無已安裝工具

```
Given:
  - 執行 uv tool list 回傳空結果（無工具）

When:
  - 執行 get_installed_uv_tools()

Then:
  - 回傳空字典 {}
```

##### Test Case 2.3：版本號變體（v 前綴、預發布版本）

```
Given:
  - uv tool list 輸出包含不同版本格式：
    - tool1  1.0.0     (標準)
    - tool2  v1.0.0    (v 前綴)
    - tool3  1.0.0-rc1 (預發布)
    - tool4  1.0.0.0   (四段版本)

When:
  - 執行 get_installed_uv_tools()

Then:
  - tool1 版本為 "1.0.0"
  - tool2 版本為 "1.0.0" (v 前綴已移除)
  - tool3 版本為 "1.0.0-rc1" (預發布保留)
  - tool4 版本為 "1.0.0.0"
```

##### Test Case 2.4：uv tool list 命令執行失敗

```
Given:
  - subprocess.run('uv tool list') 回傳非零 exit code

When:
  - 執行 get_installed_uv_tools()

Then:
  - 捕獲 CalledProcessError 或 subprocess 異常
  - 記錄 error log
  - 回傳空字典 {} (safe fallback)
```

##### Test Case 2.5：uv tool list 輸出包含特殊字元

```
Given:
  - Tool 名稱包含特殊字元：
    my-tool-123  1.0.0
    CamelCaseTool  2.0.0
    tool_with_underscore  1.5.0

When:
  - 執行 get_installed_uv_tools()

Then:
  - 工具名稱正確解析（含 -, _, 大小寫）
  - 版本提取正確
```

#### 2.3 版本比對邏輯測試

**目標**：驗證「需要重新安裝」的決策邏輯

##### Test Case 3.1：版本完全一致

```
Given:
  - scan_skill_packages() 回傳：ticket-system v1.0.0
  - get_installed_uv_tools() 回傳：ticket-system 1.0.0
  - 兩者版本字串相同

When:
  - 執行版本比對邏輯

Then:
  - 判定為「不需要重新安裝」
  - 不呼叫 reinstall_uv_tool()
```

##### Test Case 3.2：版本不符（已安裝更舊版本）

```
Given:
  - 需要版本：1.0.1
  - 已安裝版本：1.0.0

When:
  - 執行版本比對邏輯

Then:
  - 判定為「需要重新安裝」
  - 呼叫 reinstall_uv_tool(package_name, package_path)
```

##### Test Case 3.3：版本不符（已安裝更新版本）

```
Given:
  - 需要版本：1.0.0
  - 已安裝版本：1.0.1

When:
  - 執行版本比對邏輯

Then:
  - 判定為「需要重新安裝」（降級至需要的版本）
  - 呼叫 reinstall_uv_tool()
```

**設計注意**：版本比對應使用語義化版本（Semantic Versioning）規則，而非字串比較。

##### Test Case 3.4：未安裝的套件

```
Given:
  - 需要版本：1.0.0
  - 已安裝版本：None（未安裝）

When:
  - 執行版本比對邏輯

Then:
  - 判定為「需要安裝」
  - 呼叫 reinstall_uv_tool()
```

##### Test Case 3.5：版本字串正規化

```
Given:
  - 需要版本字串可能的變體：
    - "1.0.0"
    - "v1.0.0"
    - "1.0.0.0"

When:
  - 執行版本比對（不同格式組合）

Then:
  - 版本比對應正規化後進行
  - 相同的語義版本被認為相同
  - 例：1.0.0 == v1.0.0 == 1.0.0.0
```
