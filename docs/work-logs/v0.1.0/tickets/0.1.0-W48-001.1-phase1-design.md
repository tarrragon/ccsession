## 核心設計決策

### 決策 1：為什麼選擇 PostToolUse 而不是 PreToolUse？

**原因**：
- PostToolUse 檔案已寫入，可以讀取完整內容進行掃描
- PreToolUse 需要預測檔案內容（基於 Write 工具輸入），無法直接驗證
- 符合 Hook 系統的非阻塞原則（警告而非錯誤）

### 決策 2：為什麼 exit 0 不阻止？

**原因**：
- 邊界驗證的目的是「提醒」而非「禁止」
- Layer 1 定義本身可能有邊界模糊的情況
- 允許人工判斷是否確實違規

### 決策 3：為什麼不使用 AST 分析？

**原因**：
- Markdown 檔案不是程式碼，AST 分析不適用
- 正則表達式 + 排除規則已足夠應對
- 簡化 Hook 複雜度，易於維護
