## Solution

<!-- To be filled by executing agent -->

正確行為：即使 Phase 4b 豁免（跳過 4c），/tech-debt-capture 仍為必要。理由：Phase 4b 重構執行後可能仍有技術債發現，需要記錄追蹤。

修改內容：
1. tdd-flow.md 流程圖（L36）：「豁免時直接到完成」→「豁免時跳過 Phase 4c，直接執行 /tech-debt-capture → 完成」
2. tdd-flow.md 轉換條件表（L201）：「直接到完成（豁免條件）」→「/tech-debt-capture → 完成（豁免條件）」
3. decision-tree.md D3b：拆為標準路徑（全自動 → 4c）和豁免路徑（強制 tech-debt-capture）
4. decision-tree.md Checkpoint 3：新增「Phase 4b 完成（豁免）→ /tech-debt-capture → 收尾」
5. decision-tree.md Checkpoint 2 評估規則：更新「D3b 全自動」→「D3b 標準全自動、豁免時強制 tech-debt-capture」
6. 版本號：tdd-flow.md v2.3.0、decision-tree.md v7.15.0
