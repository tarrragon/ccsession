## Problem Analysis

### 語義重疊問題

現有兩個集合存在語義重疊和設計問題：

```python
EXCLUDE_FROM_SYNC = {
    ".git",
    "project-templates",
    "hook-logs",
    "handoff",
}

LOCAL_ONLY = {
    "hook-logs",
    "handoff",
    "PM_INTERVENTION_REQUIRED",
    "ARCHITECTURE_REVIEW_REQUIRED",
    "pm-status.json",
    "__pycache__",
    ".pytest_cache",
    ".venv",
}
```

**缺陷根源**：

1. `hook-logs` 和 `handoff` 同時出現在兩個集合中 — 重複定義
2. `EXCLUDE_FROM_SYNC` 混合了兩種語義：
   - 「遠端 repo 專有項目」（`.git`、`project-templates`）
   - 「本地專有項目」（`hook-logs`、`handoff`）
3. `cleanup_stale_files` 第 182 行的邏輯 `item.name in LOCAL_ONLY or item.name in EXCLUDE_FROM_SYNC` 暗示應統一為單一常數

### 影響範圍分析

修改影響點：
- line 135: `sync_directory` 函式 `if item.name in EXCLUDE_FROM_SYNC`
- line 146: `sync_directory` 函式 `ignore=shutil.ignore_patterns(*EXCLUDE_FROM_SYNC)`
- line 159: `collect_remote_files` 函式 `if item.name in EXCLUDE_FROM_SYNC`
- line 182: `cleanup_stale_files` 函式 `if item.name in LOCAL_ONLY or item.name in EXCLUDE_FROM_SYNC`

總計 4 個引用點需要替換。

---

## Solution

### 重構方案

按語義清晰化設計，將原來的兩個集合拆分為三層結構：

#### 第一層：基底集合

```python
# 遠端 repo 專有：存在於遠端但不需複製到本地
_REMOTE_REPO_ONLY = frozenset({".git", "project-templates"})

# 本地專有：只存在於本地，同步時不刪除也不覆蓋
_LOCAL_ONLY_PATHS = frozenset({
    "hook-logs",
    "handoff",
    "PM_INTERVENTION_REQUIRED",
    "ARCHITECTURE_REVIEW_REQUIRED",
    "pm-status.json",
    "__pycache__",
    ".pytest_cache",
    ".venv",
})
```

- `_REMOTE_REPO_ONLY`：明確表示「遠端特有」（`.git` 版控目錄、`project-templates` 模板目錄）
- `_LOCAL_ONLY_PATHS`：明確表示「本地特有」（日誌、交接檔案、Python 快取等）
- 使用底線前綴 `_` 標記為內部使用，遵循 Python 模組級私有慣例
- 使用 `frozenset` 表達不可變語義

#### 第二層：統一使用常數

```python
SKIP_DURING_SYNC = _REMOTE_REPO_ONLY | _LOCAL_ONLY_PATHS
```

- 單一統一的公開常數，全模組統一使用
- 語義清晰：「同步時跳過的所有路徑」
- 使用集合的聯集操作直觀地表達「合併兩類」

### 替換所有引用點

1. **line 135 & 146**（`sync_directory`）：`EXCLUDE_FROM_SYNC` → `SKIP_DURING_SYNC`
2. **line 159**（`collect_remote_files`）：`EXCLUDE_FROM_SYNC` → `SKIP_DURING_SYNC`
3. **line 182**（`cleanup_stale_files`）：`item.name in LOCAL_ONLY or item.name in EXCLUDE_FROM_SYNC` → `item.name in SKIP_DURING_SYNC`

### 行為一致性

- 重構前邏輯：`cleanup_stale_files` 檢查的是 `LOCAL_ONLY | EXCLUDE_FROM_SYNC` 的聯集
- 重構後邏輯：檢查 `SKIP_DURING_SYNC`，其值也是同樣的聯集
- **行為完全一致，無修改邏輯**

### 設計優點

1. **語義清晰**：每個集合有明確的業務語義
2. **消除重複**：`hook-logs`、`handoff` 只定義一次
3. **易於維護**：後續新增「遠端專有」或「本地專有」項目時，只需修改對應的基底集合
4. **類型一致性**：統一使用 `frozenset` 表達不可變語義
5. **模組級規約**：遵循 Python 私有約定（底線前綴）
