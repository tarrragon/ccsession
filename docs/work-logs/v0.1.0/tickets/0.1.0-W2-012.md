# Execution Log

## Task Summary

重構 多命令 Ticket 驗證邏輯統一提取為共用函式

---

## Problem Analysis

9 個命令模組（lifecycle、track_acceptance、track_batch、track_relations、fields、track_query、handoff、resume、migrate）中，重複了以下兩種模式：

1. **load_ticket + validation 重複（20+ 處）**：
   ```python
   ticket = load_ticket(version, ticket_id)
   if not ticket:
       print(format_error(ErrorMessages.TICKET_NOT_FOUND, ticket_id=ticket_id))
       return 1
   # 檢查 YAML 錯誤（部分模組）
   if "_yaml_error" in ticket:
       print(format_error(...))
       return 1
   ```

2. **Path 解析重複（12+ 處）**：
   ```python
   ticket_path = Path(ticket.get("_path", get_ticket_path(version, ticket_id)))
   ```

3. **_check_yaml_error 函式重複**：在 track_batch.py 和 handoff.py 各有一份完全相同的實作

違反 DRY 原則，增加維護成本且容易產生不一致。

---

## Solution

**Step 1：建立 ticket_ops.py 共用模組**

在 `.claude/skills/ticket/ticket_system/lib/ticket_ops.py` 建立兩個共用函式：

1. **load_and_validate_ticket(version, ticket_id, auto_print_error=True)**
   - 統合 load_ticket + not_found check + yaml_error check
   - 返回 (ticket_dict, error_msg) 元組
   - auto_print_error=True 時自動輸出錯誤；False 時由呼叫者自訂處理
   - 用途：簡化常見的「三步驟」驗證流程

2. **resolve_ticket_path(ticket, version, ticket_id)**
   - 統合 Path(ticket.get("_path", get_ticket_path(...))) 模式
   - 返回 Path 物件
   - 用途：統一 Ticket 路徑解析邏輯

**Step 2：批次更新 9 個命令模組**

- lifecycle.py：3 個 load_ticket + 3 個 Path 用法
- track_acceptance.py：3 個 load_ticket + 3 個 Path 用法
- track_batch.py：1 個 load_ticket + 1 個 Path 用法 + 移除 _check_yaml_error
- track_relations.py：3 個 Path 用法
- fields.py：1 個 load_ticket + 1 個 Path 用法
- track_query.py：5 個 load_ticket 用法
- handoff.py：2 個 load_ticket + 移除 _check_yaml_error
- resume.py：1 個 load_ticket 用法
- migrate.py：1 個 load_ticket 用法

**Step 3：移除重複的局部函式**

- 刪除 track_batch.py 的 _check_yaml_error（16 行）
- 刪除 handoff.py 的 _check_yaml_error（16 行）

結果：代碼總行數減少，DRY 原則完全遵循，提高可維護性。
