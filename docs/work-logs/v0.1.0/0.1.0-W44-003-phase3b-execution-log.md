## 問題分析（來源：W43-005）

### 問題描述

Ticket 系統發現 309 個檔案中有 22 個（6.7%）使用非標準命名模式，帶有描述性後綴：
- TDD Phase 文件：`0.1.0-W11-004-phase1-design.md`
- 分析報告：`0.1.0-W25-005-analysis.md`
- 測試設計：`0.1.0-W1-005-phase2-test-design.md`

### 根本原因

現行正則表達式 `^(\d+\.\d+\.\d+)-W(\d+)-(\d+(?:\.\d+)*)$` 要求 ID 結尾必須為序號，不允許任何後綴。導致：

1. **掃描漏洞**：`ticket_loader.py` 第 97 行使用正則過濾，非標準檔名被跳過不載入
2. **解析失敗**：`extract_id_components()` 傳入帶後綴 ID 時返回 None，鏈式計算失敗
3. **Hook 不同步**：`ticket-id-validator-hook.py` 維護獨立正則副本，與 constants.py 脫鉤

### 影響範圍

| 模組 | 行為影響 |
|------|---------|
| `ticket_loader.py` | 22 個非標準檔案無法被系統識別和載入 |
| `id_parser.py` | 帶後綴 ID 無法正確解析 |
| `ticket-id-validator-hook.py` | 驗證邏輯與系統不同步 |
| 3b 拆分子任務 | 帶模組描述的子任務檔名無法正確識別 |

---

## 解決方案

### 核心策略

實施「包容性擴充」：支援描述性後綴，同時維持向後相容（所有標準 ID 仍然有效）。

### 技術方案

**方案選擇理由**：
1. **可選後綴**：不是強制要求，既有檔案無需改名
2. **寬鬆驗證**：Hook 對後綴 ID 只提示不阻止，提升使用體驗
3. **去重機制**：同一 core_id 只載入一次，避免重複

### 實作架構

```
Layer 5 (Domain)
├─ constants.TICKET_ID_PATTERN（正則定義）
├─ constants.KNOWN_TICKET_SUFFIXES（後綴清單）
└─ id_parser.extract_core_ticket_id()（ID 提取）
     ↓ 依賴
Layer 2 (Behavior)
└─ ticket_loader.list_tickets()（載入與去重）
     ↓ 驗證
Hook Layer
└─ ticket-id-validator-hook.py（寬鬆驗證）
```

### Step 3: ticket_loader.py 更新去重邏輯（✓ 完成）

改寫 `list_tickets()` 函式以支援帶後綴的檔案載入。

**新增去重機制**：
```python
loaded_core_ids = set()  # 追蹤已載入的核心 ID

    ticket_id_raw = ticket_file.stem
    core_id = extract_core_ticket_id(ticket_id_raw)

    if core_id is None:
        continue  # 無效格式，跳過
```

## 設計決策回顧

### 為什麼選擇可選後綴而非必填後綴？

**答**：向後相容性。既有的所有標準 Ticket 檔案（無後綴）仍然有效，新功能是可選擴充。

### 為什麼限制小寫字母？

**答**：一致性和可搜尋性。避免大小寫變體（如 `-Phase1` vs `-phase1`），確保系統識別準確。

### 為什麼在 Hook 中使用寬鬆驗證？

**答**：用戶體驗。帶後綴的 Ticket 檔案是合法的產出物（TDD Phase 文件等），Hook 應該提示而非阻止。

### 為什麼 list_tickets() 使用 extract_core_ticket_id() 而非直接正則匹配？

**答**：單一責任原則。分離「去後綴」和「載入」的關注點，提高可維護性。

## 已知限制（技術債務）

### TD-001: Hook 正則同步（優先級：中）

當 constants.py 的 TICKET_ID_PATTERN 變更時，Hook 中的 fallback 正則需手動同步。

**建議**：Hook 啟動時驗證版本一致。

### TD-002: 後綴動態註冊（優先級：低）

KNOWN_TICKET_SUFFIXES 目前硬編碼列表，無法自動發現新後綴。

**建議**：掃描 ticket-id-conventions.md 自動提取後綴清單。

### TD-003: 後綴字符集限制（優先級：低）

後綴僅支援小寫字母+數字+連字號，無法支援其他符號或大小寫混用。

**建議**：Phase 4 評估是否擴充字符集。
