### 重構步驟 1：提取值標準化函式

**動作**：建立 `_normalize_ticket_id_list()` 函式

**實現**：
```python
def _normalize_ticket_id_list(value: str | list) -> list[str]:
    """
    標準化 Ticket ID 清單

    將字符串或列表轉換為統一的列表格式。

    Args:
        value: Ticket ID 清單（字符串或列表）
               - 字符串：逗號或空格分隔
               - 列表：直接使用

    Returns:
        list[str]: 標準化的 ID 列表
    """
    if isinstance(value, str):
        # 支援逗號或空格分隔
        return [id_str.strip() for id_str in value.split(",") if id_str.strip()]
    elif isinstance(value, list):
        return value
    else:
        return []
```

**好處**：
- 消除重複邏輯
- 統一標準化行為
- 易於測試

### 重構步驟 2：拆分模式處理函式

**動作**：提取三個模式處理函式

**1. replace 模式**：
```python
def _execute_set_relation_field_replace(
    current_list: list[str],
    referenced_ids: list[str],
) -> list[str]:
    """替換模式：直接替換"""
    return referenced_ids
```

**2. add 模式**：
```python
def _execute_set_relation_field_add(
    current_list: list[str],
    referenced_ids: list[str],
) -> list[str]:
    """追加模式：將 ID 加入列表（去重）"""
    new_value = current_list.copy()
    for ref_id in referenced_ids:
        if ref_id not in new_value:
            new_value.append(ref_id)
    return new_value
```

**3. remove 模式**：
```python
def _execute_set_relation_field_remove(
    current_list: list[str],
    referenced_ids: list[str],
) -> list[str]:
    """移除模式：從列表中移除指定的 ID"""
    return [id_str for id_str in current_list if id_str not in referenced_ids]
```

**好處**：
- 每個函式單一職責（單一模式邏輯）
- 易於測試（小單位測試）
- 易於擴展（新增模式無需改變核心邏輯）

### 重構步驟 3：簡化主函式

**動作**：重構 `_execute_set_relation_field()` 為協調函式

**新結構**：
1. 驗證目標 Ticket 存在
2. 解析被引用的 ID 清單
3. 驗證被引用 Ticket（除非 remove 模式）
4. 取得並標準化現有值 → **使用 `_normalize_ticket_id_list()`**
5. 根據模式選擇處理函式 → **use replace/add/remove 函式**
6. 更新並保存
7. 輸出結果

**預期長度**：< 40 行（相比原 93 行，減少 57%）
