# v0.2.0 - Onboarding Framework

## 版本目標

建立新專案 onboarding 框架，讓 `.claude/` 配置在同步到新專案後，能透過統一入口引導框架定制。

## 背景

`.claude/` 配置透過 subtree-sync 同步到新專案時，包含大量 Hook 和模板，但缺乏統一的定制引導。
新專案開發者需要手動判斷哪些 Hook 適用、哪些需停用，容易遺漏或誤操作。

## 設計策略

三波次執行：
- **Wave 1（DOC）**: 泛化現有文件，移除專案特定引用
- **Wave 2（ADJ/DOC）**: 建立配置基礎設施（Hook 分類、settings 模板）
- **Wave 3（IMP）**: 實作 onboard 子指令，整合所有配置檢查

## 技術決策

| 決策 | 選擇 | 理由 |
|------|------|------|
| Hook 分類格式 | YAML | 結構化、可被程式讀取 |
| 語言偵測方式 | 檔案特徵偵測 | pubspec.yaml → Flutter, go.mod → Go 等 |
| onboard 輸出格式 | 純文字 + 區段式 | CLI 環境下易讀 |
