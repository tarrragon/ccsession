## 5. 設計決策回顧

### 5.1 為什麼採用去重設計？

- **問題**：同一核心 ID 可能有標準檔（`0.31.0-W3-001.md`）和後綴檔（`0.31.0-W3-001-phase1-design.md`）
- **風險**：重複載入會導致 chain index 出現重複節點
- **方案**：使用 `loaded_core_ids` 集合追蹤已載入的核心 ID，標準檔優先，後綴檔被跳過
- **優勢**：
  1. 邏輯簡單，開銷小（集合 O(1) 查詢）
  2. 向後相容（標準檔案優先，後綴檔是補充）
  3. 無需改動現存 Ticket 檔案，直接支援多個副本

### 5.2 為什麼 Hook 保持 standalone？

- **理由**：Hook 設計為獨立 Python 腳本，不依賴外部模組
- **好處**：Hook 可獨立執行、無環境初始化、直接被框架呼叫
- **妥協**：常數複製帶來維護負擔，但通過一致性測試防護
