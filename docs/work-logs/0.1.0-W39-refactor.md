# 0.1.0-W39-refactor: Hook 共用模式統一重構

## 來源與目標

**來源**：0.1.0-W39-001 Phase 4a 多視角分析結論（項目 #1 核心、項目 #6 複雜度）
**目標**：改進 `validate_tool_input()` 介面設計，消除函式混淆和巢狀深度過深問題

## Phase 4a 分析結論概述

### 項目 #1（核心，Good Taste + Complexity）

**問題識別**：
- 文件要求呼叫前先執行 `validate_hook_input()`，但函式又重複做 input_data 的 None/dict 防禦檢查
- 這造成介面矛盾：呼叫者不確定是否需要先呼叫 validate_hook_input
- 巢狀深度 3 層，降低程式碼可讀性

**根本原因**：
- 函式設計的職責邊界不清（頂層驗證 vs 子欄位驗證混淆）
- 呼叫端 (ticket-quality-gate-hook.py) 傳入 `input_data` dict，但應傳入已提取的 `tool_input` dict

**改進方案**（linux 建議）：
改為直接接受 `tool_input: dict` 作為第一個參數（而非 `input_data`）
- 呼叫者已通過 `validate_hook_input(..., ("tool_name", "tool_input"))` 保證 tool_input 存在
- 呼叫端改為：`validate_tool_input(input_data.get("tool_input", {}), logger, ("file_path", "content"))`
- 函式內部只需檢查：tool_input 本身、必要欄位存在、欄位值非 None

### 項目 #6（低嚴重度，Complexity）

**問題識別**：
- hook_ticket.py 行 804-815：讀取異常和空內容分別處理但產生相同錯誤訊息
- 使用者無法區分是讀取失敗還是內容為空

**改進方案**：
合併為一個 try/except 區塊，空內容也視為讀取失敗的子情況，但提供不同的錯誤訊息

## 重構實施詳情

### 1. `validate_tool_input()` 函式重設計（hook_io.py）

**原簽名**（行 330-389）：
```python
def validate_tool_input(
    input_data: "dict | None",              # ← 設計混淆：應為 tool_input
    logger: "logging.Logger | None" = None,
    required_subfields: "Tuple[str, ...] | None" = None
) -> bool:
```

**新簽名**：
```python
def validate_tool_input(
    tool_input: dict,                       # ← 明確：直接接受 tool_input dict
    logger: "logging.Logger | None" = None,
    required_fields: "Tuple[str, ...] | None" = None  # ← 參數名改為 required_fields
) -> bool:
```

**改進重點**：
- 參數 1 從 `input_data: "dict | None"` 改為 `tool_input: dict`
  - 移除 None 防禦檢查（呼叫者責任：先用 validate_hook_input 驗證）
  - 類型簽名更明確（dict 而非 dict | None）
  - 名稱自說明（tool_input vs input_data）

- 參數 3 從 `required_subfields` 改為 `required_fields`
  - 字面更簡潔（欄位指已在 tool_input 內，無需重複 "sub"）
  - 與簽名改變邏輯一致

- 函式內邏輯簡化：
  - 保留 `isinstance(tool_input, dict)` 防禦（以防呼叫者邏輯錯誤）
  - 移除 `input_data.get("tool_input")` 邏輯
  - 直接檢查 required_fields

**程式碼變更**（60 → 45 行，17% 代碼減少）：
```python
# 舊版本：巢狀 3 層（input_data → tool_input → subfield）
if input_data is None or not isinstance(input_data, dict):      # 第 1 層
    ...
tool_input = input_data.get("tool_input")                       # 第 2 層
if tool_input is None:
    ...
if not isinstance(tool_input, dict):
    ...
for field in required_subfields:                                # 第 3 層
    if field not in tool_input:
        ...
    if tool_input.get(field) is None:
        ...

# 新版本：巢狀 2 層（tool_input → field）
if not isinstance(tool_input, dict):  # 防禦性檢查，預期不會觸發
    ...
for field in required_fields:         # 直接迴圈欄位檢查
    if field not in tool_input:
        ...
    if tool_input.get(field) is None:
        ...
```

**介面清晰性改進**：

| 面向 | 舊設計 | 新設計 |
|------|-------|-------|
| 呼叫者視角 | 不確定要不要先呼叫 validate_hook_input | 明確：先驗證頂層（validate_hook_input），再驗證子欄位（validate_tool_input） |
| 函式職責 | 混淆：驗證頂層 + 子欄位 | 清晰：只驗證子欄位 |
| 巢狀深度 | 3 層 | 2 層 |
| 防禦檢查 | 重複多層 | 單層（僅 isinstance check） |

### 2. 呼叫端修改（ticket-quality-gate-hook.py）

**舊程式碼**（行 101-121）：
```python
def validate_input(input_data: Dict[str, Any], logger) -> bool:
    if not validate_hook_input(input_data, logger, ("tool_name", "tool_input")):
        return False

    # ← 直接傳入 input_data，validate_tool_input 再內部提取
    if not validate_tool_input(input_data, logger, ("file_path", "content")):
        return False

    return True
```

**新程式碼**：
```python
def validate_input(input_data: Dict[str, Any], logger) -> bool:
    if not validate_hook_input(input_data, logger, ("tool_name", "tool_input")):
        return False

    # ← 顯式提取 tool_input，傳入新的 validate_tool_input
    tool_input = input_data["tool_input"]
    if not validate_tool_input(tool_input, logger, ("file_path", "content")):
        return False

    return True
```

**改進**：
- 職責分工明確：validate_hook_input 驗證頂層結構，validate_tool_input 驗證子欄位
- 呼叫端代碼 +1 行，但邏輯流清晰（第一步驗證頂層，第二步驗證子欄位）
- 易於除錯：tool_input 顯式可見

### 3. 空內容檢查合併（hook_ticket.py 行 802-815）

**舊程式碼**：
```python
# 步驟 2：讀取 Ticket 內容
try:
    content = ticket_path.read_text(encoding="utf-8")
except Exception as e:
    msg = "無法讀取 Ticket 檔案: {}".format(ticket_id)  # ← 異常訊息
    if logger:
        logger.error("{}: {}".format(msg, e))
    return False, msg

if not content:
    msg = "無法讀取 Ticket 檔案: {}".format(ticket_id)  # ← 空內容訊息（重複）
    if logger:
        logger.error(msg)
    return False, msg
```

**新程式碼**：
```python
# 步驟 2：讀取 Ticket 內容
try:
    content = ticket_path.read_text(encoding="utf-8")
    if not content:
        msg = "Ticket 檔案內容為空: {}".format(ticket_id)  # ← 不同訊息
        if logger:
            logger.error(msg)
        return False, msg
except Exception as e:
    msg = "無法讀取 Ticket 檔案: {}".format(ticket_id)  # ← 異常訊息
    if logger:
        logger.error("{}: {}".format(msg, e))
    return False, msg
```

**改進**：
- 邏輯更清晰：先讀取，再檢查內容是否為空
- 錯誤訊息區分：讀取失敗 vs 內容為空
- 程式流更直觀：try 內檢查成功情況，except 處理異常

### 測試更新清單

1. **validate_tool_input 測試（12 個）**：
   - 修改所有呼叫：從 `validate_tool_input(input_data, ...)` 改為 `validate_tool_input(tool_input, ...)`

   範例修改：
   ```python

       "tool_name": "Write",
       "tool_input": {"file_path": "test.md", "content": "hello"}

   result = validate_tool_input(input_data, logger, ("file_path", "content"))

   # 新
   tool_input = {"file_path": "test.md", "content": "hello"}
   result = validate_tool_input(tool_input, logger, ("file_path", "content"))
   ```

2. **validate_ticket_unified 測試（9 個）**：
   - 修改空內容測試的斷言：從 `"無法讀取 Ticket 檔案"` 改為 `"Ticket 檔案內容為空"`
