修正 框架路徑偵測與版本回退可觀測性

## Problem Analysis

承接 0.2.0-W4-001 SA 分析結論，執行三項實際修正：

1. paths.py：get_project_root() 只搜尋 pubspec.yaml，在 ccsession（Go/Flutter 混合型）中 pubspec.yaml 位於 ui/ 子目錄，導致函式回傳 Path.cwd()（錯誤的根目錄）
2. version.py：except Exception: pass 靜默吞掉所有異常，fallback 到目錄掃描時無任何警告輸出
3. sync-claude-push.sh：rsync 排除清單缺少 Python 暫存目錄（__pycache__、*.pyc、.pytest_cache）

---

## Solution

### 修正一：paths.py get_project_root()

新增 markers 清單，依序搜尋 CLAUDE.md、go.mod、pubspec.yaml：

```python
markers = ["CLAUDE.md", "go.mod", "pubspec.yaml"]
current = Path.cwd()
while current != current.parent:
    for marker in markers:
        if (current / marker).exists():
            return current
    current = current.parent
```

CLAUDE.md 在 ccsession 根目錄存在，搜尋時會正確命中。

### 修正二：version.py 可觀測性

- 新增 `import logging` 和 `logger = logging.getLogger(__name__)`
- `except Exception as e: logger.warning(...)` 取代 `except Exception: pass`
- `_scan_worklog_directories()` 在 fallback 時輸出 warning，說明版本來源

### 修正三：sync-claude-push.sh 排除清單

rsync 命令新增三行排除：
```bash
--exclude='__pycache__' \
--exclude='*.pyc' \
--exclude='.pytest_cache' \
```
