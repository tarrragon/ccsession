### 問題二：get_project_root() 對非 Flutter 專案的支援

#### 現況程式碼（paths.py 第 14-40 行）
```python
def get_project_root() -> Path:
    claude_project_dir = os.environ.get("CLAUDE_PROJECT_DIR")
    if claude_project_dir:
        return Path(claude_project_dir)
    
    current = Path.cwd()
    while current != current.parent:
        if (current / "pubspec.yaml").exists():
            return current
        current = current.parent
    
    return Path.cwd()
```

#### 失效場景：本專案結構（Go + Flutter 混合型）
```
ccsession/（專案根，CLAUDE.md 在此）
├── server/（Go）
│   └── go.mod
├── ui/（Flutter）
│   └── pubspec.yaml  ← 在子目錄！
└── .claude/
    └── skills/ticket/ticket_system/lib/paths.py
```

#### 執行路徑追蹤
```
執行位置：.claude/skills/ticket/
向上搜尋 pubspec.yaml：
  .claude/skills/ticket/    (無)
  .claude/skills/           (無)
  .claude/                  (無)
  ccsession/                (無) ← pubspec.yaml 在 ui/ 子目錄！
  ... 搜尋到根目錄（無）

結果：回傳 Path.cwd() = /Users/tarragon/Projects/ccsession/.claude/skills/ticket
錯誤：指向 ticket_system 目錄而非專案根目錄
```

#### 環境變數可靠性問題
- Hook 在 Claude Code 框架執行環境中運行
- `CLAUDE_PROJECT_DIR` 是否總是可用取決於框架設置
- 風險：環境變數不可靠時無 fallback

#### 判定
- **嚴重度**：高
- **當前表現**：⚠️ 在本專案中會失效，回傳錯誤路徑
- **影響**：所有依賴 `get_project_root()` 的功能（ticket track、version detection）會使用錯誤的根目錄

### 問題三：version fallback 無警告 log

#### 現況程式碼
`version.py` 第 30-33 行：
```python
def get_current_version() -> Optional[str]:
    version = _parse_todolist_active_version()  # 先嘗試解析 todolist.yaml
    if version:
        return version
    return _scan_worklog_directories()  # ← Fallback，完全無日誌輸出
```

`version.py` 第 105-137 行的 `_scan_worklog_directories()`：
- 掃描 work-logs 目錄找最高版本號
- 完全沒有輸出 WARN 或 INFO log

#### 失效場景
1. **todolist.yaml 不存在或格式錯誤**
   - `_parse_todolist_active_version()` 回傳 None
   - 自動 fallback 到目錄掃描
   - 用戶不知道為什麼沒使用 todolist.yaml

2. **yaml 格式異常但被靜默吞掉**
   - 第 98-100 行：`except Exception: pass`
   - 無論什麼異常都靜默 fallback
   - 用戶毫不知情

#### 判定
- **嚴重度**：中
- **現況**：❌ 無任何可觀測性，用戶無法知道版本來源
- **影響**：執行 `/ticket track summary` 時得到舊版本號，用戶無法診斷原因

### 問題二修正：get_project_root()（高優先級，需謹慎）

**修改位置**：`.claude/skills/ticket/ticket_system/lib/paths.py` 第 14-40 行

**修正策略**：調整搜尋優先級，將 CLAUDE.md 作為主要指標

**建議新實作**：
```python
def get_project_root() -> Path:
    """
    取得專案根目錄

    搜尋優先級：
    1. CLAUDE_PROJECT_DIR 環境變數
    2. 向上搜尋 CLAUDE.md（通用框架標準入口）
    3. 向上搜尋 go.mod（Go 專案）
    4. 向上搜尋 pubspec.yaml（Flutter 專案）
    5. fallback: Path.cwd()
    """
    # 1. 環境變數優先
    claude_project_dir = os.environ.get("CLAUDE_PROJECT_DIR")
    if claude_project_dir:
        return Path(claude_project_dir)

    # 2-4. 向上搜尋標記檔案
    markers = ["CLAUDE.md", "go.mod", "pubspec.yaml"]
    current = Path.cwd()
    while current != current.parent:
        for marker in markers:
            if (current / marker).exists():
                return current
        current = current.parent

    # 5. 最後 fallback
    return Path.cwd()
```
