# Execution Log

## Task Summary

改進 ticket track check-acceptance 支援文字搜尋和 0-based 索引

---

## Problem Analysis

目前的 `execute_check_acceptance` 函式（位於 `.claude/skills/ticket/ticket_system/commands/track_acceptance.py`）的 index 驗證邏輯過於嚴格：

1. 第 209-212 行：只接受正整數（> 0），所以輸入 `0` 會觸發 `ACCEPTANCE_CRITERIA_INDEX_NOT_POSITIVE` 錯誤
2. 第 213-215 行：只接受整數，所以輸入文字會觸發 `ACCEPTANCE_CRITERIA_INDEX_NOT_INTEGER` 錯誤

這導致使用者無法使用直觀的 0-based 索引或文字描述來勾選驗收條件。

---

## Solution

建立新的輔助函式 `_parse_acceptance_index(index_input, acceptance_items)`，支援三種輸入方式：

1. **1-based 整數**（現有功能，保持不變）：`1`, `2`, `3` 等
2. **0-based 整數**（新功能）：`0` 自動轉換為第 1 個項目（1-based）
3. **文字搜尋**（新功能）：如 `"任務實作完成"` 進行模糊比對驗收條件文字

實作細節：
- 先嘗試解析為整數，特殊處理 `0` 作為 0-based 索引
- 若無法解析為整數，嘗試文字搜尋
- 文字搜尋時會去除前綴（`[x]` 或 `[ ]`）進行匹配
- 多個匹配時返回錯誤並提示使用索引
- 找不到匹配時返回友善的錯誤訊息

修改的檔案：
- `/Users/tarragon/Projects/ccsession/.claude/skills/ticket/ticket_system/commands/track_acceptance.py`
  - 新增函式：`_parse_acceptance_index()`（91 行）
  - 修改函式：`execute_check_acceptance()`（72 行），使用新的 index 解析邏輯
