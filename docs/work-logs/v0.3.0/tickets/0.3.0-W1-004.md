# Execution Log

## Task Summary

修正 prompt-submit-hook QUERY/HANDOFF/DECISION_KEYWORDS 否定詞過濾缺失

---

## Problem Analysis

目前 prompt-submit-hook.py 中只有 `check_action_keywords()` 函式在循環中呼叫 `_is_keyword_negated()` 進行否定詞過濾。其他三個函式未套用此過濾：

1. `check_skill_suggestion()` 中的 QUERY_KEYWORDS 迴圈（第 297-299 行）
2. `check_handoff_keywords()` 函式（第 218-242 行）
3. `check_decision_keywords()` 函式（第 245-262 行）

導致用戶說「不需要查詢進度」或「不用切換任務」時仍會誤觸發 SKILL 提示。

---

## Solution

在三個函式中套用 `_is_keyword_negated()` 過濾，遵循 `check_action_keywords()` 的既有實作模式：

### 1. check_skill_suggestion() 中的 QUERY_KEYWORDS 迴圈（第 297-302 行）
```python
for keyword, skill_cmd in QUERY_KEYWORDS.items():
    if keyword in prompt_lower:
        if _is_keyword_negated(prompt_lower, keyword):
            continue
        return (skill_cmd, "query")
```

### 2. check_handoff_keywords() 函式（第 218-248 行）
- 單一關鍵字分支：新增 `if _is_keyword_negated(prompt, keywords[0]): continue`
- 組合關鍵字（非空字符串）分支：新增 `if _is_keyword_negated(prompt, keywords[0]) or _is_keyword_negated(prompt, keywords[1]): continue`

### 3. check_decision_keywords() 函式（第 250-270 行）
- 單一關鍵字分支：新增 `if _is_keyword_negated(prompt, keywords[0]): continue`
- 組合關鍵字分支：新增 `if _is_keyword_negated(prompt, keywords[0]) or _is_keyword_negated(prompt, keywords[1]): continue`
