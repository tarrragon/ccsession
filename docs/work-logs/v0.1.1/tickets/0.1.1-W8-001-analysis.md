# 0.1.1-W8-001 分析報告：Git Worktree 強制隔離開發機制

## 多視角分析結論

### 核心發現

| # | 發現 | 嚴重程度 | 來源 |
|---|------|---------|------|
| 1 | `.claude/` 目錄在 worktree 間共享，隔離是假象 | 高 | linux + Impact + Consistency |
| 2 | `get_project_root()` 雙重實作（CLAUDE.md 搜尋 vs git rev-parse）在 worktree 中失效 | 高 | Impact + Consistency |
| 3 | Handoff 機制（gitignored）在 worktree 中完全失效 | 高 | Impact + Consistency |
| 4 | `create_feature_worktree.py` 使用 `checkout -b` 而非 `worktree add -b`（非原子操作） | 中 | linux |
| 5 | 強制方案(B)需要自動恢復路徑，否則造成 deny-loop 死鎖 | 中 | Simplicity + linux |
| 6 | branch-verify-hook 阻止了 PM 在 main 合法編輯 `.claude/` 和 docs/ | 中 | linux |

### 設計約束（不可違反）

1. **`.claude/` 是共享的** — git worktree 只隔離 working directory，不隔離 `.claude/`
2. **gitignored 資料不跨 worktree** — `.claude/handoff/`、`.claude/hook-logs/` 在 worktree 中不存在
3. **worktree 隔離 = 程式碼隔離** — 不是完整環境隔離

## 用戶設計方向

**核心理念**：將「不要在 main 做開發」和「獨立終端機需要獨立 worktree」整合為**標準化 SKILL**，取代每次 session 的被動提醒。

### 現有問題

| 問題 | 現狀 | 理想 |
|------|------|------|
| main 保護 | Hook 每次 session 輸出警告 | SKILL 一次性設定，後續自動 |
| worktree 建立 | 手動輸入 branch + path 兩個參數 | 從 Ticket ID 自動推導，一個指令 |
| 合併回 main | 完全手動 | SKILL 引導標準化流程 |
| 清理 worktree | 容易遺忘 | 完成時自動提示或觸發 |

### 設計目標

整合後的 SKILL 應覆蓋完整生命週期：

```
/worktree setup <ticket-id>     → 自動建立分支 + worktree
/worktree status                → 列出所有 worktree 和對應 Ticket
/worktree merge <ticket-id>     → 引導合併回 main
/worktree cleanup               → 清理已合併的 worktree
```
