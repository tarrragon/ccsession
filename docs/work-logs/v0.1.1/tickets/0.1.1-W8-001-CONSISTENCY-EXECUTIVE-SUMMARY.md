## 二、Consistency 視角的核心貢獻

### 2.1 與決策樹的整合

**發現**: 決策樹缺少 worktree 觸發時機的定義

**解決方案**: 新增 **Checkpoint 0：Worktree 隔離評估**
- 位置: 決策樹第負一層「並行化評估」尾部
- 觸發條件: 4 個（多並行任務 / 獨立終端機 / PM 明確指定 / 無並行需求）
- 與 `/ticket claim` 和 `/tdd Phase 3b 拆分評估` 的時序已定義

**規格文件**: 部分 1.1（決策樹整合點），~15 行新增

---

### 2.2 與 Ticket 生命週期的整合

**發現**: 生命週期缺少 git 分支合併時序的定義

**解決方案**: 補充 3 個子小節
1. **認領後**: Git 分支初始化（若需隔離）
2. **完成前**: Git 分支合併（若使用 worktree）
3. **完成後**: Worktree 清理提示（Hook 自動）

**規格文件**: 部分 1.2（生命週期整合點），~40 行新增

---

### 2.3 Ticket 欄位擴展設計

**發現**: Ticket 無 git 相關欄位，無法追蹤分支狀態

**解決方案**: 新增 **`git:` frontmatter 區塊**
```yaml
git:
  branch: "feat/0.1.1-W8-001-automation"
  worktree: "../ccsession-0.1.1-W8-001-automation"
  status: "setup" | "merged" | "cleanup"
  setup_timestamp: "2026-03-14T10:30:00Z"
  merge_timestamp: null
```

**規格文件**: 部分 4.2 #23（Ticket 欄位規格），~30 行新增

---

### 2.4 Hook 系統的清晰分工

**發現**: 新增 Hook 與現有 branch-verify-hook 的邊界不清

**解決方案**: 三層架構
1. **新增 Hook**（Ticket 視角）:
   - `ticket-setup-guide-hook.py` - claim 後提示
   - `ticket-cleanup-reminder-hook.py` - complete 後提示

2. **改造 Hook**（worktree 感知）:
   - `branch-verify-hook.py` - 偵測 worktree 環境，區分 `.claude/` 路徑

3. **保持 Hook**（基礎設施）:
   - 其他既有 Hook 不動

**規格文件**: 部分 1.3（Hook 系統整合點），部分 2.4（branch-verify-hook 改造）

---

### 2.5 與 branch-worktree-guardian 的分層清晰化

**發現**: linux 視角想「廢棄 branch-worktree-guardian」，CogLoad 視角想「擴充」，Consistency 需要調和

**解決方案**: **分層設計 + 底層復用**

```
層級              責任                           狀態
─────────────────────────────────────────────────────
應用層            Ticket 驅動、自動命名、Hook 整合  新建 /worktree SKILL
基礎層            git 操作、底層工具、手動控制     保留 branch-worktree-guardian
```

復用函式:
- `create_feature_worktree()` ← `/worktree setup`
- `list_worktrees()` ← `/worktree status`
- `cleanup_worktree()` ← `/worktree cleanup`
- `verify_branch()` ← Hook 使用

**效果**: 
- ✅ 避免重複實現（底層復用）
- ✅ 清晰邊界（應用層 vs 基礎層）
- ✅ 向後相容（branch-worktree-guardian 保留，但不是主流）
- ✅ 易維護（分層獨立演進）

**規格文件**: 部分 2（branch-worktree-guardian 處置方案）
