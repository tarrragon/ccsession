## 部分 3：最終子命令規格

### 3.1 /worktree setup <ticket-id>

自動建立隔離 worktree + 分支

**輸入**: 
- `ticket-id`: `v0.x.x-WN-NNN` 格式
- `--no-switch`: 建立後不自動切換目錄

**執行**:
1. 驗證 Ticket 存在、狀態正確、主 worktree 在 main
2. 推導分支名: `feat/{ticket-id}-{title-slug}`
3. 推導 worktree 路徑: `../{project}-{ticket-id}-{slug}`
4. 呼叫 `create_feature_worktree()`
5. 更新 Ticket `git:` frontmatter 欄位
6. 輸出成功訊息

**輸出範例**:
```
✅ Worktree 已建立
Branch: feat/0.1.1-W8-001-automation
Worktree: ../ccsession-0.1.1-W8-001-automation
```

---

### 3.2 /worktree status [ticket-id]

列出所有 worktree 及狀態

**執行**:
1. 呼叫 `list_worktrees()` 取得所有 worktree
2. 反向查詢對應 Ticket ID
3. 查詢 Ticket 狀態（pending/in_progress/completed）
4. 計算分支提前 main 的 commit 數

**輸出範例**:
```
Worktree 狀態

Ticket          Branch                  Status      Changes
0.1.1-W8-001    feat/0.1.1-W8-001...   setup       +3/-1
0.1.1-W7-003    feat/0.1.1-W7-003...   merged      ─
```

---

### 3.3 /worktree merge <ticket-id> [--strategy=squash|merge-commit] [--no-cleanup]

合併分支回 main，清理 worktree

**執行**:
1. 驗證前置條件（worktree 存在、無未提交變更）
2. 拉取 main 最新版本
3. 檢查合併衝突
4. 執行合併（squash 或 merge commit）
5. 建立 merge commit
6. 清理 worktree（除非 `--no-cleanup`）
7. 更新 Ticket `git: status = "merged"`

**衝突情況**:
- 列出衝突檔案
- 提示手動解決步驟

---

### 3.4 /worktree cleanup [ticket-id] [--force]

清理已合併的 worktree

**執行**:
1. 列出所有「已合併」或「孤立」worktree
2. 若指定 ticket-id，只處理該 Ticket
3. 逐個確認（除非 `--force`）
4. 執行 `git worktree remove` + `git branch -d`
5. 更新 Ticket 欄位
