## 2. 核心邏輯流程設計

### 2.1 整體資料流程

```
使用者執行 check 子命令
    |
    v
[Step 1] load_version_release_config(root)
    |
    +-- 讀取 .version-release.yaml
    +-- 若不存在 → 返回 DEFAULT_VERSION_RELEASE_CONFIG
    +-- 若格式錯誤 → 警告 + fallback
    |
    v
[Step 2] get_monorepo_version(root)
    |
    +-- 讀取 docs/todolist.yaml
    +-- 提取 current_version 欄位
    +-- 若不存在 → 返回 None（會導致 check 失敗）
    |
    v
[Step 3] check_monorepo_version_sync(version, config)
    |
    +-- 讀取 L2: ui/pubspec.yaml
    +-- 偵測 L3: server/go.mod（無版本欄位，直接返回 false）
    +-- 執行三層版本比對邏輯
    +-- 依 config 中的 sync_policy 和 conflict_detection 規則
    +-- 生成訊息和狀態判定
    |
    v
[Step 4] print_version_sync_report(sync_result)
    |
    +-- 樹狀輸出 L1/L2/L3 版本資訊
    +-- 輸出警告/資訊訊息
    +-- 輸出結論
    |
    v
[Step 5] check_version_sync() 回傳結果
    |
    +-- Tuple[bool, List[str]]
    +-- passed = 策略符合（非完全版本匹配）
    +-- errors = [] 清單（不匹配不視為錯誤）
```

### 2.2 四個函式的責任邊界

**load_version_release_config(root: Path) -> dict**
- 責任：讀取配置檔，处理例外，提供預設值
- 邊界：本函式只負責檔案讀取，不負責配置驗證
- 返回值保證：永不返回 None，總是返回有效字典

**get_monorepo_version(root: Path) -> Optional[str]**
- 責任：從 todolist.yaml 讀取 L1 版本
- 邊界：只讀取 current_version，不做版本格式驗證
- 返回值可能為 None（呼叫端需要檢查）

**check_monorepo_version_sync(version: str, config: dict) -> dict**
- 責任：執行三層版本比對邏輯，生成檢查結果
- 邊界：接收版本和配置作為輸入，不再負責檔案讀取
- 返回值結構明確（passed, l1_version, l2_version, l3_has_version, messages, summary）

**print_version_sync_report(sync_result: dict)**
- 責任：將檢查結果以樹狀格式輸出
- 邊界：只負責輸出格式，不負責邏輯判斷
- Side Effect：寫入 stdout

### 2.3 流程決策點

#### Step 2a：L1 版本取得失敗

```
get_monorepo_version() 返回 None
    |
    v
check_monorepo_version_sync() 應如何反應？
    |
    +-- 選項 A：拋出 ValueError
    +-- 選項 B：回傳 passed=false，summary="錯誤：找不到 monorepo 版本"
    +-- 選項 C：回傳 passed=false，messages 含 ERROR 級別訊息
    |
決定：採用選項 C
    理由：
    - 保持函式簽名一致（總是返回 dict）
    - 呼叫端（check_version_sync）負責判定是否阻塞
    - 訊息機制統一管理所有狀態信息
```

#### Step 3a：L2 版本讀取失敗

```
ui/pubspec.yaml 不存在或無 version 欄位
    |
    v
check_monorepo_version_sync() 應如何反應？
    |
    +-- 選項 A：拋出 FileNotFoundError
    +-- 選項 B：l2_version=None，生成 info 訊息，passed=true（非阻塞）
    +-- 選項 C：l2_version=""（空字串），表示未偵測到
    |
決定：採用選項 B
    理由：
    - L2 是可選的（sync_policy: optional）
    - 未偵測不應阻塞整個 check
    - info 訊息讓使用者知道發生了什麼
```

#### Step 3b：版本比對邏輯

```
L2 版本含 build number（如 "1.0.0+1"）
    |
    v
比對時應使用完整版本還是去除 +N 後的主版本？
    |
    +-- 選項 A：完整比對 "1.0.0+1" vs "0.1.1"（總是不相等）
    +-- 選項 B：去除 +N 後比對 "1.0.0" vs "0.1.1"（只比主版本）
    +-- 選項 C：比對主版本，但輸出時保留完整版本
    |
決定：採用選項 C
    理由：
    - 語義上 "1.0.0+1" > "0.1.1"（L2 獨立升級）
    - build number 是增量標記，不影響主版本比較
    - Phase 1 規格要求：「去除 + 後綴進行主版本比對，但 CLI 輸出應顯示完整版本」
```

#### Step 3c：版本比對結果和訊息生成

```
給定：L1="0.1.1"，L2="1.0.0+1"
比對：去除 +1 後得 L2主="1.0.0"

比對邏輯：
    if L2主 > L1: level="warning"
    elif L2主 < L1: level="info"
    else: level="success"

疑問：如何定義「大於」和「小於」？
    - 簡單字符串比較？（"1.0.0" > "0.1.1" → true）
    - 語義版本比較？（從左到右按數字比較）
    |
決定：採用語義版本比較（3 位元組方式）
    理由：
    - 更符合專案版本管理的語意
    - Phase 1 的場景 4 中，L1="0.1.1"，L2="2.0.0" 應判定為 warning
    - 簡單字符串比較可能產生反直覺結果（如 "0.9.0" > "0.10.0"）
```

## 4. 虛擬碼與控制流程

### 4.1 load_version_release_config() 虛擬碼

```
函式 load_version_release_config(root: Path) -> dict:
    輸入：root（專案根目錄 Path 物件）

    config_path = root / ".version-release.yaml"

    如果 config_path 不存在：
        返回 DEFAULT_VERSION_RELEASE_CONFIG

    嘗試：
        使用 yaml.safe_load() 讀取檔案
        檢查返回值是否為 None（空檔案）：
            如果是 → 返回 DEFAULT_VERSION_RELEASE_CONFIG
        檢查返回值是否為 dict：
            如果否 → 拋出 TypeError

        對配置執行深層 merge（補充缺漏欄位）：
            為每個 config["versions"][key] 補充 DEFAULT 中的缺漏欄位
            為每個 config["sync_rules"] 補充預設值

        返回 merged_config

    捕獲 yaml.YAMLError：
        輸出警告到 stderr：
            "[WARNING] .version-release.yaml 格式錯誤，使用內建預設配置"
            輸出錯誤詳情
        返回 DEFAULT_VERSION_RELEASE_CONFIG

    捕獲其他 Exception：
        輸出錯誤到 logger
        返回 DEFAULT_VERSION_RELEASE_CONFIG
```

### 4.2 get_monorepo_version() 虛擬碼

```
函式 get_monorepo_version(root: Path) -> Optional[str]:
    輸入：root（專案根目錄 Path 物件）
    輸出：版本字串或 None

    todolist_path = root / "docs" / "todolist.yaml"

    如果 todolist_path 不存在：
        返回 None

    嘗試：
        使用 yaml.safe_load() 讀取 todolist.yaml

        檢查返回值是否為 dict：
            如果否 → 返回 None

        取得 current_version 欄位：
            version = data.get("current_version")

        如果 version 是 None：
            返回 None

        如果 version 不是字串：
            轉換為字串：str(version)

        返回 version

    捕獲 Exception：
        返回 None
```

### 4.3 check_monorepo_version_sync() 虛擬碼（核心邏輯）

```
函式 check_monorepo_version_sync(version: str, config: dict) -> dict:
    輸入：version（L1 版本字串），config（配置字典）
    輸出：檢查結果字典

    初始化：
        messages = []
        l2_version = None
        l3_has_version = False

    [檢查 L1]
    如果 version 為空或 None：
        messages.append({level: ERROR, text: "L1 monorepo 版本為空"})
        返回 {passed: false, l1_version: version, ...}

    [檢查 L2 - 讀取]
    ui_source = config.get("versions", {}).get("ui", {}).get("source")
    l2_path = root / ui_source

    如果 l2_path 不存在：
        l2_version = None
        messages.append({level: INFO, text: "ui/pubspec.yaml 不存在，跳過 L2 檢查"})
    否則：
        嘗試 yaml.safe_load() 讀取 pubspec.yaml
        ui_key = config["versions"]["ui"]["key"]  # 通常為 "version"
        l2_version = data.get(ui_key)

        如果 l2_version 為 None：
            messages.append({level: INFO, text: "ui/pubspec.yaml 不含 version 欄位"})

    [檢查 L2 - 版本比對]
    如果 l2_version 不為 None：
        sync_policy = config["versions"]["ui"]["sync_policy"]

        計算 l2_main = 去除 build number 的主版本（"1.0.0+1" → "1.0.0"）

        比對結果 = 語義版本比較(l2_main vs version)

        如果 比對結果 > 0（L2 > L1）：
            severity = config["sync_rules"]["conflict_detection"]["ui_ahead_of_monorepo"]["severity"]
            messages.append({level: severity, text: "UI 版本大於 monorepo，確認是否故意？"})

        否則如果 比對結果 < 0（L2 < L1）：
            severity = config["sync_rules"]["conflict_detection"]["ui_behind_monorepo"]["severity"]
            messages.append({level: severity, text: "UI 版本低於 monorepo（正常）"})

        否則（L2 = L1）：
            messages.append({level: SUCCESS, text: "UI 版本與 monorepo 版本一致"})

    [檢查 L3]
    server_source = config["versions"]["server"]["source"]  # 應為 None
    如果 server_source 為 None：
        l3_has_version = false
        messages.append({level: INFO, text: "server/go.mod 無版本欄位，由 monorepo 版本決定"})

    [最終判定]
    passed = 檢查 messages 中是否有 ERROR 級別訊息：
        如果有 → passed = false
        否則 → passed = true

    summary = 生成結論文字：
        如果 passed 且無 warning：→ "通過（版本策略符合 monorepo 三層架構）"
        如果 passed 且有 warning：→ "通過（附警告）"
        如果不 passed：→ "失敗（{錯誤原因}）"

    返回 {
        passed: bool,
        l1_version: version,
        l2_version: l2_version,
        l3_has_version: l3_has_version,
        messages: messages,
        summary: summary
    }
```

### 4.4 print_version_sync_report() 虛擬碼

```
函式 print_version_sync_report(sync_result: dict):
    輸入：sync_result（check_monorepo_version_sync 的返回值）
    副作用：寫入 stdout

    [打印標題]
    print("版本同步檢查（Monorepo 三層架構）")
    print("=" * 60)

    [打印 L1]
    l1 = sync_result["l1_version"]
    print(f"L1 monorepo 版本: {l1} (docs/todolist.yaml)")
    print("|")

    [打印 L2]
    l2 = sync_result.get("l2_version")
    如果 l2 為 None：
        print("|-- L2 ui/pubspec.yaml: 未偵測到")
    否則：
        print(f"|-- L2 ui/pubspec.yaml: {l2}")
        對每個 sync_policy = "optional" 的 message：
            print(f"|   +-- 狀態：不匹配（預期行為）")
            print(f"|       理由：{message['text']}")

    print("|")

    [打印 L3]
    l3_has = sync_result.get("l3_has_version", False)
    如果 l3_has 為 False：
        print("+-- L3 server/go.mod: 無版本欄位（正常）")
        print("    理由：Go module 無自身版本，由 monorepo 版本決定")

    [打印訊息]
    對每個 message 在 sync_result["messages":]：
        根據 message["level"] 選擇格式：
            ERROR → "[ERROR] {text}"
            WARNING → "[WARNING] {text}"
            INFO → "[INFO] {text}"
            SUCCESS → "[OK] {text}"
        print 訊息

    [打印結論]
    print()
    print(f"結論：{sync_result['summary']}")
```

### 4.5 版本比較子函式

```
函式 compare_semantic_versions(v1: str, v2: str) -> int:
    輸入：v1, v2（版本字串，格式 "X.Y.Z"）
    輸出：-1 (v1<v2), 0 (v1=v2), 1 (v1>v2)

    // 分解版本字串為整數列表
    parts1 = [int(x) for x in v1.split(".")[:3]]  // 取前 3 位
    parts2 = [int(x) for x in v2.split(".")[:3]]

    // 補齊缺漏部分（如 "0.1" → [0, 1, 0]）
    while len(parts1) < 3: parts1.append(0)
    while len(parts2) < 3: parts2.append(0)

    // 逐位比較
    for i in range(3):
        if parts1[i] > parts2[i]: return 1
        if parts1[i] < parts2[i]: return -1

    return 0  // 相等
```

```
check_version_sync(version)
    ↓
[入點驗證]

  load_version_release_config()
    ↓
[取得 L1 版本]

    ↓
  version 非 None？ → 否 → ERROR，回傳失敗
    ↓ 是
[執行檢查]
  check_monorepo_version_sync(version, config)
    ↓
  [L2 讀取]
    pubspec.yaml 存在？
      ├─ 否 → l2_version=None，info 訊息
      └─ 是 → 讀取 version 欄位
          ├─ 不存在 → l2_version=None，info 訊息
          └─ 存在 → 繼續
    ↓
  [L2 比較]
    l2_version 非 None？
      ├─ 否 → 跳過
      └─ 是 → 比較版本
          ├─ L2 > L1 → warning
          ├─ L2 < L1 → info
          └─ L2 = L1 → success
    ↓
  [L3 檢查]
    l3_has_version = False（始終，go.mod 無版本欄位）
    info 訊息：Server 版本由 monorepo 決定
    ↓
  [判定]
    passed = messages 中無 ERROR
    ↓
[輸出報告]
  print_version_sync_report(sync_result)
    ↓
[回傳結果]
  return (passed, errors)
```

## 11. 語言特定實作注意事項（給 thyme-python-developer）

### Python 3.10+ 特性

**使用**：
- `| Union 語法`（如 `Optional[str]` 等同 `str | None`）但保持向下相容寫法便於理解
- `match` 陳述句（若覆蓋邏輯複雜，如訊息等級判定）
- `dataclass` 或 `TypedDict` 定義結構化返回值（可選，dict 也可以）

### 常數管理

**強制**：所有常數置於檔案頂部，禁止硬編碼字串和數字散落在函式中。

已定義常數（12 個）：
```python
VERSION_RELEASE_CONFIG_FILE
MONOREPO_VERSION_SOURCE, MONOREPO_VERSION_KEY
UI_VERSION_SOURCE
SERVER_VERSION_SOURCE
SYNC_POLICY_REQUIRED, SYNC_POLICY_OPTIONAL, SYNC_POLICY_IMPLICIT
SEVERITY_ERROR, SEVERITY_WARNING, SEVERITY_INFO, SEVERITY_SUCCESS
DEFAULT_VERSION_RELEASE_CONFIG（巨大但必要）
```

### 例外處理（dual-channel 要求）

依 quality-baseline.md 規則 4，所有 except 區塊必須：
1. 輸出到 stderr（用戶端可見）
2. 記錄到 logger（持久化）

**範例**（load_version_release_config）：
```python
except yaml.YAMLError as e:
    import sys
    print(f"[WARNING] .version-release.yaml 格式錯誤，使用內建預設配置", file=sys.stderr)
    print(f"         錯誤：{e}", file=sys.stderr)
    logger.warning(f"YAML 解析失敗: {e}", exc_info=True)
    return DEFAULT_VERSION_RELEASE_CONFIG
```

### 版本比較實作

建議實作 `compare_semantic_versions(v1: str, v2: str) -> int` 作為 helper 函式，返回 -1/0/1。

**注意**：
- 分解為整數列表，補齊至 3 位
- 忽略 build number（"1.0.0+1" → "1.0.0"）
- 不處理 pre-release（"1.0.0-rc1" 直接字符串比較可能有誤，但項目不使用）

### 路徑操作

使用 `pathlib.Path` 物件，確保跨平台相容：
```python
root = Path(os.getcwd())
config_path = root / ".version-release.yaml"
todolist_path = root / "docs" / "todolist.yaml"
```

### 字典操作

使用 `dict.get()` 提供預設值，實現寬鬆解析：
```python
current_version = data.get("current_version")
sync_policy = config.get("versions", {}).get("ui", {}).get("sync_policy", "optional")
```

### YAML 讀取

已有 `import yaml`，直接使用 `yaml.safe_load()`（安全）：
```python
with open(config_path) as f:
    config = yaml.safe_load(f)
```
