## Solution

### Root Cause

`_validate_target_ticket_exists()` 函式（handoff.py 第 341-373 行）在驗證跨版本 handoff 時使用了來源 ticket 的版本號參數，而非從 target_id 中提取的目標 ticket 版本號。導致跨版本查詢（如 v0.1.0 → v0.1.1）時在來源版本目錄中找不到目標 ticket。

### Fix Implementation

1. **修改 `_validate_target_ticket_exists()` 函式**（handoff.py）：
   - 使用 `extract_version_from_ticket_id(target_id)` 從目標 ticket ID 提取版本號
   - 將提取的版本號傳給 `load_ticket(target_version, target_id)`，替代原有的 `load_ticket(version, target_id)`
   - 新增版本提取失敗時拋出例外的防守機制

2. **新增跨版本 handoff 測試案例**（test_handoff_target_validation.py）：
   - `test_cross_version_handoff_uses_target_version`：驗證使用目標版本（v0.1.1）而非來源版本（v0.1.0）
   - `test_cross_version_handoff_missing_target_raises_error`：驗證跨版本查詢失敗時正確拋出例外

### 參考已有模式

使用了 handoff_utils.py 中已有的跨版本查詢成功模式（第 48-72 行 `is_ticket_completed()` 和第 101-127 行 `is_ticket_in_progress_or_completed()`），確保程式碼一致性。
